# 归零者｜语言版本策略

> 文件定位：本文件用于记录 `归零者 / ZERO ACCESS` 的语言版本策略。
>
> 本文件不替代正式规则书、正式卡面文案或正式发行策略。

---

## 一、结论

正式销售版不建议采用中英文完整混排一版。

推荐方案：

```text
中文版：中文为主。
英文版：英文为主。
中英文各出一版。
```

核心语言原则：

```text
中英文不是逐词直译关系，而是各自语言环境下的本地化意译。
英文版要像英文桌游卡牌，而不是中文直译。
中文版要像中文玩家会自然叫出口的代号，而不是英文术语翻译腔。
```

中文正式术语更新：

```text
游侠 = 人物牌类型 / 行动个体。
英文对应 Operative。
“执行者”翻译腔太重，废弃，不再作为中文正式术语。
“队员”身份太弱，废弃，不再作为中文正式术语。
```

说明：

```text
中文使用“游侠”，强调脱离体制、自由行动、城市边缘生存。
英文使用“Operative”，强调行动人员、秘密行动者、任务单位。
中英文是本地化意译关系，不是逐词直译。
```

## 二、游侠代号名规范

游侠中文卡名采用代号名体系。

```text
游侠名称不参考英文名称。
游侠名称不参考星级。
游侠名称不绑定装备等级。
游侠名称根据技能气质、人物外号、装备绰号和行动代号综合命名。
```

关键判断：

```text
技能是命名依据，但卡名不是技能说明。
卡名要像小队外号、街头绰号、装备绰号或行动代号。
```

允许来源：

```text
人物外号：大兵、黄毛、独眼、老炮、捞仔、中间人、扒手、赌徒。
工具 / 装备绰号：扳手、喇叭、算盘、铁钩、弹簧、雾弹。
动物 / 感官代号：鹰眼、猎豹、白狐、毒牙、乌鸦。
行动气质代号：回魂、续火、逆鳞、沉石、清道夫。
```

禁止方向：

```text
不要所有卡都动物化。
不要所有卡都叫“XX客”。
不要所有卡都叫“XX手”。
不要把技能直接写成卡名。
不要用星级、装备等级或强弱印象命名。
不要为了英文名对应而硬译。
不要使用论文术语、软件术语或说明书式名字。
不要使用过长的名词短语。
不要把游侠命名成正规军军衔或官方探员职位。
```

合格标准：

```text
短。
有识别度。
有街头感。
有行动气质。
能被玩家直接喊出来。
能从技能气质上解释。
不依赖英文名。
不依赖星级。
不依赖装备等级。
不像规则说明。
不像职业说明。
不像翻译腔。
```

当前详细规范来源：

```text
13-执行者本地化命名.md
```

## 三、卡面信息保留原则

普通游侠卡、物品卡保持极简四类信息：

```text
1. 卡名
2. 技能描述
3. 底边附近卡号
4. 底边附近Logo
```

节点卡保留五类世界观内正式信息：

```text
1. 卡名
2. 系统域标签
3. 节点技能描述
4. 唯一卡号
5. 底边附近Logo
```

节点技能文案原则：

```text
“控制此节点”作为规则前提写入规则说明。
卡面技能区不再重复固定前缀，只写具体效果。
```

卡号与系统域标签原则：

```text
卡号不是后台管理编号，而是世界观内真实存储序列。
系统域标签不是辅助说明，而是该封存卡的正式域标签。
版本记录归属到卡号下面，但版本号不拼进卡号。
```

## 四、中英文共用边框

中英文各出一版，但必须共用同一套卡框和技能区版式。

```text
同一套卡框
同一套插画区
同一套标题区
同一套技能区高度
同一套系统域标签位置
同一套卡号位置
同一套底边Logo位置
只替换语言文本
```

技能区空间标准：

```text
中文版：留够 1 行中文规则。
英文版：留够 2 行英文规则。
```

图标语言不再作为技能区核心方案。

## 五、为什么不建议完整双语混排

实体卡尺寸有限，双语混排会带来以下问题：

```text
1. 规则区拥挤，降低可读性。
2. 小字号在CMYK印刷后容易糊。
3. 中文玩家不需要完整英文规则。
4. 英文玩家不需要完整中文规则。
5. 双语规则会让卡牌更像说明卡，而不是商品级卡牌。
6. 后续国际版、众筹页、规则书本地化都会更难管理。
```

## 六、卡面保留信息

### 游侠卡 / 物品卡

| 信息 | 位置 | 说明 |
|---|---|---|
| 卡名 | 顶部或画面主标题区 | 中文版用中文代号，英文版用英文意译名 |
| 技能描述 | 底部技能区 | 中文1行，英文2行 |
| 卡号 | 底边附近 | 如 R-001、G-004 |
| Logo | 底边附近 | 归零者 / ZERO ACCESS |

### 节点卡

| 信息 | 位置 | 说明 |
|---|---|---|
| 卡名 | 顶部或画面主标题区 | 中文版用中文节点代号，英文版用英文意译名 |
| 系统域标签 | 卡名附近或左上小徽章区 | 如：行政域、安防域、生化域、科研域；属于世界观内正式信息 |
| 节点技能描述 | 底部技能区 | 中文1行，英文2行；只写具体效果 |
| 唯一卡号 | 右上或底边附近 | 如 N-007 / BIO-B1；属于世界观内正式信息 |
| Logo | 底边附近 | 归零者 / ZERO ACCESS |
