# 归零者｜语言版本策略

> 文件定位：本文件用于记录 `归零者 / ZERO ACCESS` 的语言版本策略。
>
> 本文件不替代正式规则书、正式卡面文案或正式发行策略。

---

## 一、结论

正式销售版不建议采用中英文完整混排一版。

推荐方案：

```text
中文版：中文为主。
英文版：英文为主。
中英文各出一版。
```

核心语言原则：

```text
中英文不是逐词直译关系，而是各自语言环境下的本地化意译。
英文版要像英文桌游卡牌，而不是中文直译。
中文版要像中文玩家会自然叫出口的代号，而不是英文术语翻译腔。
```

中文正式术语更新：

```text
游侠 = 人物牌类型 / 行动个体。
英文对应 Operative。
“执行者”翻译腔太重，废弃，不再作为中文正式术语。
“队员”身份太弱，废弃，不再作为中文正式术语。
```

说明：

```text
中文使用“游侠”，强调脱离体制、自由行动、城市边缘生存。
英文使用“Operative”，强调行动人员、秘密行动者、任务单位。
中英文是本地化意译关系，不是逐词直译。
```

中文命名原则：

```text
中文卡名优先使用“代号感 / 行动称呼 / 街头黑话 / 系统暗号”。
尤其是游侠与技能相关命名，要接地气、短、好记、能喊出来。
不要为了对应英文而硬译。
不要像论文术语、软件说明书或机器翻译。
```

英文命名原则：

```text
英文卡名优先使用英语玩家能理解的职业感、装备感和地点感。
可以更清楚、更类型化。
不要求和中文逐词对应。
不使用 Ranger 作为类型名，避免传统游侠、巡林者、弓箭手职业误解。
```

## 二、卡面信息保留原则

普通游侠卡、越权模块卡保持极简四类信息：

```text
1. 卡名
2. 技能描述
3. 底边附近卡号
4. 底边附近Logo
```

权限节点卡保留五类世界观内正式信息：

```text
1. 卡名
2. 系统域标签
3. 节点技能描述
4. 唯一卡号
5. 底边附近Logo
```

权限节点技能文案原则：

```text
“控制此节点”作为规则前提写入规则说明。
卡面技能区不再重复固定前缀，只写具体效果。
```

卡号与系统域标签原则：

```text
卡号不是后台管理编号，而是世界观内真实存储序列。
系统域标签不是辅助说明，而是该封存卡的正式域标签。
版本记录归属到卡号下面，但版本号不拼进卡号。
```

---

## 三、中英文共用边框

中英文各出一版，但必须共用同一套卡框和技能区版式。

```text
同一套卡框
同一套插画区
同一套标题区
同一套技能区高度
同一套系统域标签位置
同一套卡号位置
同一套底边Logo位置
只替换语言文本
```

技能区空间标准：

```text
中文版：留够 1 行中文规则。
英文版：留够 2 行英文规则。
```

图标语言不再作为技能区核心方案。

---

## 四、为什么不建议完整双语混排

实体卡尺寸有限，双语混排会带来以下问题：

```text
1. 规则区拥挤，降低可读性。
2. 小字号在CMYK印刷后容易糊。
3. 中文玩家不需要完整英文规则。
4. 英文玩家不需要完整中文规则。
5. 双语规则会让卡牌更像说明卡，而不是商品级卡牌。
6. 后续国际版、众筹页、规则书本地化都会更难管理。
```

## 五、卡面保留信息

### 游侠卡 / 越权模块卡

| 信息 | 位置 | 说明 |
|---|---|---|
| 卡名 | 顶部或画面主标题区 | 中文版用中文代号，英文版用英文意译名 |
| 技能描述 | 底部技能区 | 中文1行，英文2行 |
| 卡号 | 底边附近 | 如 R-001、G-004 |
| Logo | 底边附近 | 归零者 / ZERO ACCESS |

### 权限节点卡

| 信息 | 位置 | 说明 |
|---|---|---|
| 卡名 | 顶部或画面主标题区 | 中文版用中文节点代号，英文版用英文意译名 |
| 系统域标签 | 卡名附近或左上小徽章区 | 如：无人机域、门禁域、医疗域、信用域；属于世界观内正式信息 |
| 节点技能描述 | 底部技能区 | 中文1行，英文2行；只写具体效果 |
| 唯一卡号 | 右上或底边附近 | 如 N-007 / DN-D1；属于世界观内正式信息 |
| Logo | 底边附近 | 归零者 / ZERO ACCESS |

## 六、中文代号策略

中文不是英文的直译稿。

中文卡名应像玩家在桌上会说的代号：

```text
短
狠
好念
有行动感
有地下小队感
有权限系统味道
```

允许：

```text
跑线侦察员
门禁撬手
数据小偷
管线盗权者
黑钱包
信用铸机
债务库
蜂鸟巢
```

这些名字不必逐词对应英文，但要在中文环境下自然、好记、有画面。

技能相关代号要求：

```text
技能相关命名要接地气。
优先使用玩家能理解的动作词：看、抽、弃、移、吹回、回满、放回、重洗、保护、接管。
少用抽象词、长术语、翻译腔。
```

避免：

```text
访问反冲游侠
逆向路由操作员身份实体
权限资源重新分配模块
节点控制状态增益器
```

## 七、英文意译策略

英文版不是中文版逐词翻译。

英文类型使用：

```text
Operative
```

英文卡名应符合英文桌游和科幻卡牌的习惯：

```text
Trace Runner
Street Courier
Gatejack
Data Rat
Rootbreaker
Offline Bunker
Hummingbird Hive
Debt Vault
```

英文可以更类型化、更清楚。

中文可以更黑话、更代号。

两者只需要保持同一张卡的功能和幻想一致，不要求字面一致。

不使用：

```text
Ranger
Executor
Agent
Unit
Soldier
```

## 八、游侠等级与装备语言

游侠星级不是军衔，也不是职业等级，而是装备和资源层级。

```text
1星游侠：冷兵器、匕首、简易工具。
2星游侠：手枪、自制武器、低质弹药。
3星游侠：低配制式装备，弹药不足，品质不高。
4星游侠：全套制式装备，头盔、防弹衣齐全，弹药充足。
5星游侠：轻型动力机甲或高度集成外骨骼。
```

详细设定来源：

```text
14-游侠等级与装备设定.md
```

## 九、系统域标签写法

系统域标签必须短，不能写成长说明。

中文版建议：

```text
无人机域
门禁域
医疗域
信用域
```

英文版建议：

```text
DRONE DOMAIN
ACCESS DOMAIN
MEDICAL DOMAIN
CREDIT DOMAIN
```

允许形式：

```text
系统域短词 + 小型系统域徽章
```

禁止形式：

```text
无人机域 / DRONE DOMAIN 大标签
控制无人机域3个节点以获得ROOT ACCESS
系统域说明长句
区域背景解释文字
把系统域标签做成旁白说明感
```

## 十、卡面删除信息

不再在卡面上保留：

```text
职业标签长文
类型说明长文
系统域说明长文
防御型 / 游侠 / 越权模块等重复说明文字
连续图标技能链
右上角归零者品牌图标
过多英文装饰句
说明性副标题
权限节点固定前缀“控制此节点：”
制作版本号作为正式卡面信息
直译式中英文对照
中文正式术语“执行者”
中文正式术语“队员”
```

说明：

```text
权限节点卡需要保留系统域标签，但不能写系统域长说明。
卡号应表现为真实存储卡信息，而不是后台备注。
卡面文字只服务于“认牌”和“执行规则”。
```

## 十二、图标与语言的关系

当前决策：图标语言不再作为卡面技能表达核心。

```text
文字表达规则
图标表达类型、系统域、状态和氛围
```

允许保留的图标用途：

```text
类型图标
系统域图标
状态图标
稀有度 / 星级图标
品牌与编号辅助
```

不再使用：

```text
[入场] [节点牌库顶1] → [查看] → [置顶/置底]
```

这种连续技能解释链。

## 十三、当前决策

```text
语言策略：中英文各出一版。
翻译策略：各自语言环境下本地化意译，不做逐词直译。
中文术语：游侠，不再使用执行者或队员。
英文术语：Operative，不使用 Ranger 作为类型名。
中文策略：使用代号感、行动称呼和接地气技能词。
英文策略：使用英文桌游玩家自然理解的类型化表达。
等级策略：游侠星级体现装备和资源层级，不是军衔。
边框策略：中英文使用同一套边框和技能区尺寸。
当前样卡：先按中文主版制作。
游侠 / 越权模块：只保留卡名、技能描述、底边附近卡号、底边附近Logo。
权限节点：保留卡名、系统域标签、节点技能、唯一卡号、Logo。
权限节点技能：只写具体效果，不重复“控制此节点”。
系统域标签：世界观内正式信息，不是辅助说明。
卡号：世界观内真实存储序列，不是后台编号。
技能区：中文1行，英文2行。
图标策略：不再使用图标技能链，只保留类型/状态/域识别符号。
规则区：不做完整双语。
```
