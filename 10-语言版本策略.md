# 归零者｜语言版本策略

> 文件定位：本文件用于记录 `归零者 / ZERO ACCESS` 的语言版本策略。
>
> 本文件不替代正式规则书、正式卡面文案或正式发行策略。

---

## 一、结论

正式销售版不建议采用中英文完整混排一版。

推荐方案：

```text
中文版：中文为主。
英文版：英文为主。
中英文各出一版。
```

当前修正：权限节点卡必须补充“区域 / 系统域信息”。

普通执行者卡、越权模块卡保持极简四类信息：

```text
1. 卡名
2. 技能描述
3. 底边附近编号
4. 底边附近Logo
```

权限节点卡保留五类信息：

```text
1. 卡名
2. 区域信息 / 系统域信息
3. 节点技能描述
4. 底边附近编号
5. 底边附近Logo
```

权限节点技能文案原则：

```text
“控制此节点”作为规则前提写入规则说明。
卡面技能区不再重复固定前缀，只写具体效果。
```

---

## 二、中英文共用边框

中英文各出一版，但必须共用同一套卡框和技能区版式。

```text
同一套卡框
同一套插画区
同一套标题区
同一套技能区高度
同一套区域信息位置
同一套底边编号位置
同一套底边Logo位置
只替换语言文本
```

技能区空间标准：

```text
中文版：留够 1 行中文规则。
英文版：留够 2 行英文规则。
```

图标语言不再作为技能区核心方案。

---

## 三、为什么不建议完整双语混排

实体卡尺寸有限，双语混排会带来以下问题：

```text
1. 规则区拥挤，降低可读性。
2. 小字号在CMYK印刷后容易糊。
3. 中文玩家不需要完整英文规则。
4. 英文玩家不需要完整中文规则。
5. 双语规则会让卡牌更像说明卡，而不是商品级卡牌。
6. 后续国际版、众筹页、规则书本地化都会更难管理。
```

## 四、卡面保留信息

### 执行者卡 / 越权模块卡

| 信息 | 位置 | 说明 |
|---|---|---|
| 卡名 | 顶部或画面主标题区 | 中文版用中文卡名，英文版用英文卡名 |
| 技能描述 | 底部技能区 | 中文1行，英文2行 |
| 编号 | 底边附近 | 如 TR-01、OM-01 |
| Logo | 底边附近 | 归零者 / ZERO ACCESS |

### 权限节点卡

| 信息 | 位置 | 说明 |
|---|---|---|
| 卡名 | 顶部或画面主标题区 | 中文版用中文卡名，英文版用英文卡名 |
| 区域信息 | 卡名附近或左上小徽章区 | 如：无人机域、门禁域、医疗域、信用域 |
| 节点技能描述 | 底部技能区 | 中文1行，英文2行；只写具体效果 |
| 节点编号 | 右上或底边附近 | 如 D1、DN-D1 |
| Logo | 底边附近 | 归零者 / ZERO ACCESS |

## 五、区域信息写法

区域信息必须短，不能写成长说明。

中文版建议：

```text
无人机域
门禁域
医疗域
信用域
```

英文版建议：

```text
DRONE DOMAIN
ACCESS DOMAIN
MEDICAL DOMAIN
CREDIT DOMAIN
```

允许形式：

```text
区域短词 + 小型系统域徽章
```

禁止形式：

```text
无人机域 / DRONE DOMAIN 大标签
控制无人机域3个节点以获得ROOT ACCESS
系统域说明长句
区域背景解释文字
```

## 六、卡面删除信息

不再在卡面上保留：

```text
职业标签长文
类型说明长文
系统域说明长文
防御型 / 执行者 / 越权模块等重复说明文字
连续图标技能链
右上角归零者品牌图标
过多英文装饰句
说明性副标题
权限节点固定前缀“控制此节点：”
```

说明：

```text
权限节点卡需要保留区域短词，但不能写系统域长说明。
卡牌类型、星级等信息仍主要通过卡框、颜色、编号、符号和视觉结构识别。
卡面文字只服务于“认牌”和“执行规则”。
```

## 八、图标与语言的关系

当前决策：图标语言不再作为卡面技能表达核心。

```text
文字表达规则
图标表达类型、系统域、状态和氛围
```

允许保留的图标用途：

```text
类型图标
系统域图标
状态图标
稀有度 / 星级图标
品牌与编号辅助
```

不再使用：

```text
[入场] [节点牌库顶1] → [查看] → [置顶/置底]
```

这种连续技能解释链。

---

## 九、当前决策

```text
语言策略：中英文各出一版。
边框策略：中英文使用同一套边框和技能区尺寸。
当前样卡：先按中文主版制作。
执行者 / 越权模块：只保留卡名、技能描述、底边附近编号、底边附近Logo。
权限节点：增加区域信息，保留卡名、区域信息、节点技能、编号、Logo。
权限节点技能：只写具体效果，不重复“控制此节点”。
技能区：中文1行，英文2行。
图标策略：不再使用图标技能链，只保留类型/状态/域识别符号。
规则区：不做完整双语。
```
