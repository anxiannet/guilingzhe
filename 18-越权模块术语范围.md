# 归零者｜越权模块术语范围

> 文件定位：本文件定义《归零者 / ZERO ACCESS》中“越权模块 / Override Gear”的正式术语范围。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、当前结论

```text
越权模块 / Override Gear，是所有非游侠卡的统称。
```

它不是狭义的电子模块，也不只代表软件、芯片或设备。

```text
凡不是游侠的手牌卡，都归入越权模块。
```

---

## 二、正式定义

越权模块，是归零者小队用于越权接入、部署、侦察、防御、干扰、破坏、恢复、移动、反制或临时获取权限的非游侠物品。

它可以是实体，也可以是一次性物品；可以是硬件，也可以是权限载体；可以是武器，也可以是工具。

---

## 三、包含范围

越权模块包括但不限于：

```text
装备
设备
工具
武器
权限卡
纳米机器人
电磁手雷
程序芯片
数据钥匙
临时权限载体
防御工事
侦察无人机
一次性越权物品
```

示例：

```text
蜂鸟无人机：越权模块 / 设备 / 锚点。
防御地堡：越权模块 / 设备 / 防御工事。
零日芯片：越权模块 / 程序芯片 / 一次性反制。
电磁手雷：越权模块 / 武器 / 一次性干扰。
权限卡：越权模块 / 权限载体。
纳米机器人：越权模块 / 纳米设备。
```

---

## 四、与游侠的边界

游侠：

```text
人物牌。
行动个体。
有身份、外号、技能、费用、攻击力和生命值。
```

越权模块：

```text
非游侠物品牌。
可以是装备、设备、工具、武器、权限卡、纳米机器人、电磁手雷或其他越权物品。
可以有费用、攻击力、生命值和技能。
也可以是一次性效果牌。
```

---

## 五、与可部署实体的关系

不是所有越权模块都必须部署。

越权模块分为两大类：

```text
1. 实体模块：可以部署在权限节点上，拥有攻击力和生命值。
2. 一次性模块：只结算效果，不留在权限节点上，可以没有攻击力和生命值。
```

实体模块例子：

```text
蜂鸟无人机。
防御地堡。
部署式工具。
固定设备。
自动武器。
```

一次性模块例子：

```text
零日芯片。
强制回滚扇区。
电磁手雷。
一次性权限卡。
临时越权指令。
```

## 六、卡面类型写法

卡面主类型统一写：

```text
越权模块
```

卡面副类型可以写：

```text
设备
装备
工具
武器
权限卡
纳米设备
程序芯片
电磁手雷
防御工事
锚点
一次性
```

示例卡面类型：

```text
越权模块 / 设备 / 锚点
越权模块 / 设备 / 防御工事
越权模块 / 程序芯片 / 一次性
越权模块 / 武器 / 电磁手雷
越权模块 / 权限卡 / 一次性
```

---

## 七、禁止误解

```text
不要把“越权模块”理解成只是一块芯片。
不要把“越权模块”限制为电子设备。
不要把装备、工具、武器、权限卡另立为与越权模块并列的大类。
不要为每种非游侠物品创建新的一级卡牌类型。
```

统一原则：

```text
一级类型只有：游侠、越权模块、权限节点。
越权模块内部再区分装备、设备、工具、武器、权限卡、纳米机器人、电磁手雷等副类型。
```
