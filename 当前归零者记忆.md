## 三、牌库模式

### 4人共用牌库模式

```text
玩家共用1个手牌牌库。
玩家共用1个节点牌库。
蜂鸟无人机查看任意手牌牌库顶1张。
蜂鸟巢查看节点牌库顶1张。
```

### 双人对战模式

```text
双方各自使用1个手牌牌库。
双人对战模式不使用节点牌库。
蜂鸟无人机查看任意手牌牌库顶1张。
蜂鸟巢若在双人对战规则中启用，需要另行定义；当前默认按4人模式节点牌库口径记录。
```

非节点卡卡面原则：

```text
只有节点卡可以在卡面直接提及节点牌库。
非节点卡不得在卡面直接提及节点牌库。
```

G-006 算法口径：

```text
输入：game_mode、actor_player_id、target_deck_id。
规则引擎根据 game_mode 生成 available_deck_ids。
target_deck_id 必须属于 available_deck_ids。
效果：查看 target_deck_id 顶1张牌。
双人对战模式：available_deck_ids = [actor_hand_deck, opponent_hand_deck]。
```

## 六、标准版基础动作池

详细来源：

```text
22-标准版基础动作池.md
```

当前标准版游侠与物品卡基础动作主池：

```text
1. 抽牌。
2. 查看牌库顶。
3. 调整牌库顶。
4. 返回牌库。
5. 查看手牌。
6. 返回手牌。
7. 消灭。
```

备选动作：

```text
重洗牌库。
```

当前明确移除：

```text
弃牌。
公开手牌。
隔离。
禁用。
费用变化。
攻击变化。
生命变化。
伤害。
治疗。
获得电荷。
己方返回牌库。
己方消灭。
```

按作用对象展开：

```text
手牌牌库：抽牌、查看牌库顶、调整牌库顶、返回牌库；重洗牌库为备选。
对手手牌：查看手牌、返回牌库。
对手场上游侠：返回手牌、返回牌库、消灭。
对手场上装备：返回手牌、返回牌库、消灭。
己方场上游侠：返回手牌。
己方场上装备：返回手牌。
```

## 十一、节点在线 / 离线 / 保护状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
竖置：保护状态。
```

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

保护状态：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被接管。
被保护节点不能被技能变为离线。
```

胜利句：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

## 十二、12座固定节点技能草案

节点技能默认费用：

```text
花费1电荷。
只有已控制且在线的节点可以发动节点技能。
离线节点不能发动节点技能。
```

12座节点按4组设计，每组3个节点技能。

| 系统域 | 节点 | 技能组 | 节点功能 | 节点技能草案 |
|---|---|---|---|---|
| 行政域 | 档案馆 | 节点状态组 | 查看节点信息 | 查看1个节点的信息。 |
| 行政域 | 中继站 | 节点状态组 | 使节点在线 | 使1个节点在线。 |
| 行政域 | 屏蔽塔 | 节点状态组 | 使节点离线 | 使1个节点离线。 |
| 安防域 | 防火墙 | 节点控制组 | 保护节点 | 保护1个节点1回合。 |
| 安防域 | 传送站 | 节点控制组 | 移动己方游侠 | 移动1个己方游侠到1个节点。 |
| 安防域 | 货运站 | 节点控制组 | 移动己方装备 | 移动1个己方装备到1个节点。 |
| 生化域 | 蜂鸟巢 | 游侠强化组 | 强化游侠攻击 | 选择1个己方游侠，本回合攻击力+1。 |
