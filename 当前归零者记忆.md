## 七、回合流程与资源

当前测试回合流程：

```text
1. 回合开始：清除由当前玩家提供、持续到当前玩家回合开始的保护效果。
2. 获得权限电荷：本回合电荷 = min(当前回合数, 10)。
3. 抽牌：按当前牌库模式抽1张牌。
4. 节点技能窗口：可以使用1次已控制且在线节点的节点技能。
5. 行动阶段：可以打出任意张手牌，也可以攻击节点，只要有足够电荷支付费用。
6. 结束阶段：清除本回合临时状态，未使用电荷清空。
```

当前测试行动规则：

```text
每回合不限制打出手牌张数。
每回合不限制攻击节点次数。
每打出1张游侠或物品卡，立即支付其费用。
每派出1个实体攻击节点，立即支付1电荷。
节点技能每回合最多使用1次，消耗电荷。
刚上场的游侠或装备，本回合不能参与攻击节点。
```

费用口径：

```text
游侠打出费用以卡表标注为准。
物品卡费用以卡表标注为准。
节点技能费用以节点卡面费用图标 + 数字标注为准。
节点技能句不重复写“电荷”或“控制此节点”。
游侠、装备的入场效果包含在打出费用中，不额外支付技能费用。
一次性物品结算自身效果，不额外支付技能费用。
攻击节点时，每派出1个己方实体，支付1电荷。
```

## 八、节点状态与保护

在线节点：

```text
可以部署可部署实体。
可以被攻击。
可以被占领。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
可以部署可部署实体。
可以被攻击。
可以被占领。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

保护节点：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能成为攻击节点的目标。
被保护节点不能被技能变为离线。
被保护节点仍可以由控制者部署实体增援。
无主节点不能被保护。
```

## 九、部署、驻守上限与节点归属

可部署实体：

```text
游侠默认是可部署实体。
装备也是可部署实体。
一次性物品不是可部署实体。
每个可部署实体都有攻击力和生命值。
```

驻守与归属：

```text
每座节点最多有3个实体。
一个节点上的实体必须全部属于该节点控制者。
一个节点上不能同时存在多个玩家的实体。
无主节点上不能存在任何实体。
一座节点最多只属于一个玩家。
节点也可以处于无主状态。
控制节点的必要条件：该节点上至少有1个该玩家的己方实体。
若节点控制者在该节点上没有任何己方实体，该节点立即变为无主。
```

部署定义：

```text
打出游侠或装备时，选择1座无主节点或你已控制的节点，将该实体部署到该节点上。
不能从手牌直接部署到其他玩家控制的节点。
若目标节点已有3个实体，不能选择该节点。
若将实体部署到无主节点，你获得该节点控制权。
若将实体部署到你已控制的节点，视为增援节点。
```

入场限制：

```text
刚上场的实体，指本回合打出并部署到节点上的游侠或装备。
刚上场的实体可以留在节点上提供驻守压力。
刚上场的实体本回合不能被选择为攻击节点的攻方。
从下一次其控制者回合开始起，该实体才可以参与攻击节点。
移动不是上场；通过节点技能移动到新节点的旧实体，如果本回合尚未攻击，可以参与攻击节点。
```

## 十、节点技能动作边界

```text
基础动作池只限制游侠与物品卡技能。
节点技能使用独立节点动作池，包括查看节点、上线、离线、保护、移动、临时强化、从弃牌区回收。
节点技能不得反向扩展为标准版手牌技能。
```

移动类节点技能：

```text
移动类节点技能只能移动己方场上实体。
移动目标必须是你已控制且未满3个实体的节点。
不能移动到无主节点。
不能移动到其他玩家控制的节点。
移动不是上场；被移动的旧实体若本回合尚未攻击，仍可参与攻击节点。
```

## 十一、攻击节点与占领结算

攻击节点条件：

```text
攻击方可以从自己控制的任意节点中派出任意数量的己方游侠或装备作为攻方。
攻方不能包含本回合刚上场的实体。
攻方不能包含本回合已攻击过的实体。
攻方不能包含 ATK 为0的实体。
每派出1个攻方实体，支付1电荷。
攻击目标必须是1座由其他玩家控制的节点。
无主节点不能成为攻击目标；无主节点只能通过部署获得节点控制权。
被保护节点不能成为攻击目标。
攻击方不因目标节点最多3个实体而被提前限制为最多派出3个实体。
```

目标节点容量：

```text
若攻击成功，存活攻方实体进入目标节点。
目标节点最多只能保留3个实体。
若存活攻方实体超过3个，攻击方选择其中3个进入目标节点，其余存活攻方实体返回各自来源节点。
```

攻击节点结算流程：

```text
1. 攻击方选择1座由其他玩家控制且未被保护的目标节点。
2. 攻击方从自己控制的任意节点中选择任意数量的己方游侠或装备作为攻方。
3. 攻击方为每个攻方实体支付1电荷。
4. 攻击方决定攻方实体的攻击顺序。
5. 目标节点控制者作为守方。
6. 守方每次选择1个守方实体迎战。
7. 当前攻方实体与当前守方实体同时造成等于各自 ATK 的伤害。
8. 受到伤害大于等于 HP 的实体被消灭，进入对应弃牌区。
9. 每个攻方实体最多攻击1次。
10. 重复结算，直到攻方实体都攻击过、攻方全部被消灭，或目标节点没有守方实体。
```

占领节点结算：

```text
若攻击结束时，目标节点没有守方实体，且至少有1个攻方实体存活：存活攻方实体进入目标节点，攻击方获得节点控制权；目标节点最多保留3个存活攻方实体，超出的返回各自来源节点。
若攻击结束时，目标节点仍有守方实体：存活攻方实体返回各自来源节点，目标节点控制权不变。
若攻击结束时，目标节点没有守方实体，且没有攻方实体存活：目标节点变为无主，目标节点上没有任何实体。
攻击节点结算完成后，检查所有来源节点；若某来源节点的控制者已没有己方实体留在该节点，该来源节点变为无主。
```

说明：

```text
一个节点上不能同时存在多个玩家实体。
获得节点控制权只在部署到无主节点或攻击节点结算成功时发生。
不再使用“节点上 ATK 总和最高者自动获得控制权”的旧测试算法。
```

## 十二、胜利条件与测试结算

正式胜利条件：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

Root Access 检查时点：

```text
玩家部署到无主节点并获得节点控制权后。
攻击节点成功，占领目标节点后。
节点在线 / 离线状态改变后。
节点变为无主后。
```

若达到模拟最大回合仍无人获得 Root Access，测试结算顺序：

```text
1. 控制在线节点数量多者胜。
2. 若相同，比较全部在线节点上己方可部署实体 ATK 总和，较高者胜。
3. 若仍相同，比较自己在单一系统域中控制的最多在线节点数，较高者胜。
4. 若仍相同，比较自己场上可部署实体数量，较多者胜。
5. 若仍相同，视为平局。
```

## 十三、临时状态

```text
本回合 ATK / HP 增益在当前玩家回合结束时清除。
实体离场、返回手牌、返回牌库或进入弃牌区时，清除该实体所有临时状态。

```
