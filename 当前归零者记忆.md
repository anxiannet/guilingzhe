## 五、牌库模式

### 4人共用牌库模式

```text
玩家共用1个手牌牌库。

玩家共用1个节点牌库。
蜂鸟无人机查看任意手牌牌库顶1张。
蜂鸟巢查看节点牌库顶1张。
```

### 3人共用牌库测试模式

```text
3人测试时，默认沿用4人共用牌库模式。
玩家共用1个手牌牌库。
玩家共用1个公共弃牌区。
玩家共用1个节点牌库。
```

### 双人对战模式

```text
双方各自使用1个手牌牌库。
双方各自拥有自己的弃牌区。
双人对战模式不使用节点牌库。
蜂鸟无人机查看任意手牌牌库顶1张。
```

非节点卡卡面原则：

```text
只有节点卡可以在卡面直接提及节点牌库。
非节点卡不得在卡面直接提及节点牌库。
```

## 七、回合流程与资源

当前测试回合流程：

```text
1. 回合开始：清除由当前玩家提供、持续到当前玩家回合开始的保护效果。
2. 获得权限电荷：本回合电荷 = min(当前回合数, 10)。
3. 抽牌：按当前牌库模式抽1张牌。
4. 节点技能窗口：可以使用1次已控制且在线节点的节点技能。节点技能不占打出手牌张数。
5. 打牌阶段：可以打出任意张手牌，只要有足够电荷支付费用。
6. 结束阶段：未使用电荷清空。
```

当前测试打牌规则：

```text
每回合不限制打出手牌张数。
每打出1张游侠或物品卡，立即支付其费用。
没有足够电荷时，不能打出该牌。
玩家可以在仍有电荷时继续打出手牌。
玩家可以主动停止打牌。
节点技能每回合最多使用1次，不占打出手牌张数，但消耗电荷。
```

费用口径：

```text
游侠打出费用以卡表标注为准。
物品卡费用按物品强度单独标注，以卡表标注为准。
节点技能费用以节点卡面标注为准，不再全部默认1电荷。
游侠、装备的入场效果包含在打出费用中，不额外支付技能费用。
一次性物品结算自身效果，不额外支付技能费用。
```

## 八、节点状态与保护

在线节点：

```text
可以部署可部署实体。
可以被攻击。
可以被占领。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
可以部署可部署实体。
可以被攻击。
可以被占领。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

保护节点：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被占领。
被保护节点不能被技能变为离线。
被保护节点仍可以部署实体。
多人局中，不在整轮统一清除保护。
```

## 九、部署、入场技能与控制权算法

可部署实体：

```text
游侠默认是可部署实体。
装备也是可部署实体。
一次性物品不是可部署实体。
每个可部署实体都有攻击力和生命值。
每个可部署实体都可以部署在节点上。
在线节点和离线节点都可以部署可部署实体。
```

入场技能时点：

```text
当前测试规则采用“先部署，后结算入场技能”。
打出游侠或装备时，先选择节点并部署该实体，再结算该牌的卡面技能。
一次性物品不部署，直接结算卡面技能。
```

当前测试控制权算法：

```text
每当有可部署实体进入或离开节点、节点状态变化、实体 ATK 变化时，立即重新检查节点控制权。
对每座节点，分别计算每名玩家在该节点上的可部署实体 ATK 总和。
ATK 总和最高且大于0的玩家获得该节点控制权。
若最高 ATK 并列，节点控制权不变。
若节点无控制者且最高 ATK 并列，则该节点保持无主。
被保护节点不能因控制权检查改变控制者。
离线节点可以被攻击、占领并保留控制归属，但不计入 Root Access 胜利条件。
```

控制权算法审核结论：

```text
当前自动控制权算法继续保留。
它适合自动测试，因为简单、稳定、可复现。
因为当前不限制打出手牌张数，Root Access 必须在每次控制权检查后立即检查，而不是只在回合结束检查。
如果连续打牌导致胜利过快，优先调整费用曲线、复制数和起手牌，而不是恢复每回合1张手牌限制。
```

## 十、胜利条件与测试结算

正式胜利条件：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

牌库耗尽与测试结算：

```text
牌库耗尽后，不自动重洗弃牌区。
需要抽牌时，若对应牌库为空，则跳过抽牌。
玩家仍可使用手牌中的卡牌。
玩家仍可通过场上实体和节点技能改变节点控制。
```

若达到模拟最大回合仍无人获得 Root Access，测试结算顺序：

```text
1. 控制在线节点数量多者胜。
2. 若相同，比较全部在线节点上己方可部署实体 ATK 总和，较高者胜。
3. 若仍相同，比较自己在单一系统域中控制的最多在线节点数，较高者胜。
4. 若仍相同，比较自己场上可部署实体数量，较多者胜。
5. 若仍相同，视为平局。
```

## 十二、技能审核原则

技能必须同时满足：

```text
1. AI 友好。
2. 算法友好。
3. 可自动测试。
4. 放在符合世界观的载体上。
5. 具备明确规则意义。
6. 标准版游侠与物品卡技能只能由基础动作池中的动作变体或组合形成。
7. 卡名必须让玩家能直觉联想到技能。
8. 卡名、技能和载体必须互相支持。
```
