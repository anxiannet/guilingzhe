## 三、权限节点在线 / 离线状态

权限节点有两种状态：

```text
正面朝上：在线状态。
背面朝上：离线状态。
```

在线权限节点：

```text
节点信息公开。
可以部署实体。
可以争夺和被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线权限节点：

```text
节点信息不公开。
可以部署实体。
可以争夺和被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

离线节点隐藏信息包括：

```text
节点正面卡名。
系统域。
节点技能。
当前控制者。
该节点上的部署实体。
```

核心区别：

```text
离线只关闭节点技能与公开信息，不关闭部署和争夺。
```

当前胜利句：

```text
控制同一系统域的3座在线权限节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Access Nodes in one System Domain to gain Root Access and win.
```

## 四、权限节点三类技能结构

每个系统域固定3座权限节点。

每个系统域内，3座权限节点分别承担：

```text
侦察类
进攻类
防守类
```

当前三类节点技能：

```text
侦察类：查看任意1座离线权限节点的隐藏信息。
进攻类：选择1座权限节点，使其离线。
防守类：选择1座权限节点，使其在线。
```

说明：

```text
当前主规则不使用节点牌库。
因此侦察类技能只查看场上固定离线权限节点的隐藏信息，不重新引入节点牌库。
查看后，该节点仍保持离线，不自动公开给所有玩家。
```

## 六、可部署实体与攻防生命

```text
游侠、装备、设备、武器都视为可部署实体。
每个可部署实体都有攻击力和生命值。
每个可部署实体都可以部署在权限节点上。
在线节点和离线节点都可以部署可部署实体。
```

部署规则：

```text
打出一个可部署实体时，选择1座权限节点，将该实体部署到该权限节点上。
部署在权限节点上的实体仍由其控制者控制。
部署实体不必然等于控制该节点。
```

数值：

```text
攻击力 / ATK：用于接管权限节点、压制敌方实体或参与节点争夺。
生命值 / HP：用于承受攻击、清理、回滚或系统压制。
生命值降到0的实体被处理，置入来源手牌弃牌区。
```

## 七、权限电荷资源系统

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
没有足够电荷时，不能打出对应卡牌，也不能发动对应技能。
```

费用口径：

```text
游侠打出费用默认等于当前星级。
越权模块费用按模块强度单独标注。
权限节点技能默认花费1电荷，且节点必须在线。
游侠入场效果包含在该游侠的打出费用中，不额外支付技能费用。
```
