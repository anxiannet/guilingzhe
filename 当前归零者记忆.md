## 四、技能审核原则

技能必须同时满足：

```text
1. AI 友好。
2. 算法友好。
3. 可自动测试。
4. 放在符合世界观的载体上。
```

禁止：

```text
喊一名玩家。
喊一名玩家名字。
对方答应后。
如果对方听见。
如果对方回应。
口头确认。
现实语音触发。
玩家主观判断触发。
```

载体判断：

```text
游侠技能：必须像“人能做的事”。
物品卡技能：必须像“物品产生的效果”。
节点技能：必须像“城市基础设施提供的系统能力”。
```


## 九、节点在线 / 离线 / 保护状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
竖置：保护状态。
```

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

保护状态：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被接管。
被保护节点不能被技能变为离线。
```

胜利句：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

## 十二、权限电荷资源系统

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
没有足够电荷时，不能打出对应卡牌，也不能发动对应技能。
```

费用口径：

```text
游侠打出费用默认等于当前星级。
物品卡费用按物品强度单独标注。
节点技能默认花费1电荷，且节点必须在线。
游侠入场效果包含在该游侠的打出费用中，不额外支付技能费用。
```
