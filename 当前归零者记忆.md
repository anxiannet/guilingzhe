## 一、项目定位

```text
归零者 / ZERO ACCESS
```

定位：

```text
一款关于系统弃民、越权装备与城市权限夺取的独立战术卡牌项目。
```

核心口号：

```text
从零权限，到根权限。
From Zero Access to Root Access.
```

玩家身份：

```text
玩家扮演一支零权限小队 / 归零者小队的首领。
小队成员是被城市权限系统删除的人。
目标不是重新申请权限，而是夺取 Root Access。
```

核心幻想：

```text
系统不给你权限，你就夺取 Root Access。
```

## 六、游侠等级与装备设定

游侠不是正规军，不是官方探员，也不是普通雇佣兵。

```text
游侠，是被城市权限系统删除后，脱离官方身份，在城市权限边缘行动的人。
他们靠捡、抢、改、拼装备逐步武装自己，并通过越权接入、潜入、接管和小队协作争夺系统权限。
```

战力来源：

```text
1. 捡来的装备。
2. 抢来的武器。
3. 拼出来的护甲。
4. 改出来的模块。
5. 越权接入的系统资源。
```

1星至5星装备层级：

```text
1星游侠：只有匕首类冷兵器和简易工具，几乎无护甲。
2星游侠：拥有手枪、自制枪械或土改武器，弹药和稳定性不足。
3星游侠：拥有残缺或低配制式装备，品质不高，弹药不足。
4星游侠：拥有全套制式装备，头盔、防弹衣齐全，弹药充足，装备精良。
5星游侠：拥有轻型动力机甲或高度集成外骨骼装备，接近小队核心战力。
```

说明：

```text
游侠星级代表当前装备状态，不作为游侠中文代号命名依据。
游侠插画必须同时参考技能、人物气质 / 代号和当前星级装备状态。
```

详细设定来源：

```text
14-游侠等级与装备设定.md
```

## 十、当前视觉母设定

归零者卡牌不是普通印刷插画卡，而是：

```text
高科技二维封存体卡牌。
```

核心定义：

```text
《归零者》中的卡牌，是将游侠、越权模块与权限节点压缩封存后的二维权限载体。
调用一张卡，不是“打出一张牌”，而是“释放一个被封存的战术实体”。
```

视觉关键词：

```text
高科技封存载体
二维压缩
透明数据膜
卡内压缩空间
空间折叠边缘
深色金属卡框
半透明技能面板
收藏卡质感
真实存储卡
唯一卡号
系统域标签
冷白青蓝 Root Access 光
明确空间层次
特写 / 近景 / 中景 / 远景
立体战术实体
卡内压缩空间感
```

插画空间层次硬标准：

```text
插画必须具备明确空间层次与立体感。
应避免单层平铺、平面海报化、主体悬空无空间关系的处理。
画面应尽量建立特写、近景、中景、远景中的多层结构。
通过遮挡、透视、体块、景深、封存边缘和空间折叠关系，表现“高科技二维封存体”的卡内压缩空间感。
```

层次理解：

```text
特写：身份锚点 / 核心装置 / 武器或模块细节。
近景：主体本体 / 当前动作 / 节点核心结构。
中景：战术环境 / 相关设施 / 接入线路 / 空间平台。
远景：系统背景 / 所属系统域环境 / 城市权限结构纵深。
```

禁止方向：

```text
不要普通赛博朋克霓虹街头。
不要普通黑客键盘敲代码感。
不要普通雇佣兵枪战。
不要大面积白底说明卡。
不要脏旧废土感。
不要过度刮痕和复杂花纹。
不要让卡面像教学说明页。
不要让 AI 生成文字直接进入印刷终稿。
不要把封存卡号做成后台制作备注感。
不要把系统域标签做成旁白说明感。
不要制造第二套卡号。

不要用中英文直译腔做正式卡名。

不要把游侠画成传统武侠或弓箭手职业。
不要做成单层平铺概念图。
```
