## 一、项目定位

```text
归零者 / ZERO ACCESS
```

定位：

```text
一款关于系统弃民、越权装备与城市权限夺取的战术卡牌主题。
```

核心口号：

```text
从零权限，到根权限。
From Zero Access to Root Access.
```

玩家身份：

```text
玩家扮演一支零权限小队 / 归零者小队的首领。
小队成员是被城市权限系统删除的人。
目标不是重新申请权限，而是夺取 Root Access。
```

## 四、首版系统域与权限节点

首版固定 4 个系统域，每个系统域 3 个权限节点。

| 系统域 | 节点1 | 节点2 | 节点3 |
|---|---|---|---|
| 门禁域 Access Domain | 高塔门禁 Tower Gate | 电梯栈 Elevator Stack | 封锁线闸门 Lockdown Gate |
| 医疗域 Medical Domain | 纳米诊所 Nano Clinic | 复苏舱 Revival Pod | 生物档案库 Bio Archive |
| 无人机域 Drone Domain | 蜂鸟巢 Hummingbird Hive | 空域中继 Sky Relay | 目标网格 Target Grid |
| 信用域 Credit Domain | 黑钱包 Black Wallet | 信用铸机 Credit Mint | 债务库 Debt Vault |

`身份域 Identity Domain` 暂不进入首版，作为扩展候选。

## 五、归零者专属变体规则

当前只在归零者中测试权限节点技能：

```text
在你的回合中，你可以使用1次你已控制权限节点的节点技能。
节点技能不占用主要行动。
每名玩家每回合最多使用1次节点技能。
即使你控制多个权限节点，每回合也只能选择其中1座权限节点发动技能。
中央区无主权限节点不能发动节点技能。
没有合法目标时，不能发动对应节点技能。
```

节点技能卡面统一写法：

```text
控制此节点：……
```

注意：该规则只属于归零者变体，不进入《抢地盘》正式规则。

## 六、当前视觉母设定

归零者卡牌不是普通印刷插画卡，而是：

```text
高科技二维封存体卡牌。
```

核心定义：

```text
《归零者》中的卡牌，是将执行者、越权模块与权限节点压缩封存后的二维权限载体。
调用一张卡，不是“打出一张牌”，而是“释放一个被封存的战术实体”。
```

视觉关键词：

```text
高科技封存载体
二维压缩
透明数据膜
卡内压缩空间
空间折叠边缘
深色金属卡框
半透明技能面板
收藏卡质感
冷白青蓝 Root Access 光
```

禁止方向：

```text
不要普通赛博朋克霓虹街头。
不要普通黑客键盘敲代码感。
不要普通雇佣兵枪战。
不要大面积白底说明卡。
不要脏旧废土感。
不要过度刮痕和复杂花纹。
不要让卡面像教学说明页。
```

## 七、卡面信息原则

### 执行者 / 越权模块

只保留四类信息：

```text
1. 卡名
2. 技能描述
3. 底边附近编号
4. 底边附近 Logo
```

### 权限节点

必须保留五类信息：

```text
1. 卡名
2. 区域信息 / 系统域信息
3. 节点技能描述
4. 节点编号
5. 底边附近 Logo
```

蜂鸟巢当前卡面信息：

```text
蜂鸟巢
无人机域

控制此节点：查看节点牌库顶1张，可置顶或置底。

D1 / DN-D1
归零者 / ZERO ACCESS
```

区域信息要求：

```text
短小、嵌入式、徽章化。
不要写系统域长说明。
不要写“控制无人机域3个节点以获得ROOT ACCESS”。
```

## 九、当前工作流决定

后续不再直接生成完整成卡来赌文字与细节。

推荐流程：

```text
1. 固定卡框感觉。
2. 单独生成无文字、无卡框的插图。
3. 用设计工具合成卡框、插图和人工文字。
4. 技能文案、编号、区域信息、Logo全部人工重排。
```

AI 不应负责最终文字：

```text
技能文案
DN-D1
D1
无人机域
归零者 / ZERO ACCESS
```

这些必须在设计工具中人工排版。
