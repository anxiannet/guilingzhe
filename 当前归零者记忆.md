## 四、权限节点在线 / 离线 / 保护状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
竖置：保护状态。
```

在线权限节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线权限节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

保护状态：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被接管。
被保护节点不能被技能变为离线。
```

离线节点隐藏信息只包括：

```text
节点正面信息。
```

当前胜利句：

```text
控制同一系统域的3座在线权限节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Access Nodes in one System Domain to gain Root Access and win.
```

## 五、节点技能设计原则

```text
每个节点技能都不一样。
每个节点技能只做一件事。
节点技能要简单、易懂。
节点技能必须符合节点世界观。
节点技能不写连锁效果。
节点技能不写条件限制。
节点技能卡面不使用“若 / 当 / 如果 / 然后再”结构。
```

保护关键词：

```text
保护1座权限节点。
```

说明：

```text
保护的持续时间和效果写在规则书中，不塞进卡面技能文案。
```

## 七、可部署实体与攻防生命

```text
游侠、装备、设备、武器都视为可部署实体。
每个可部署实体都有攻击力和生命值。
每个可部署实体都可以部署在权限节点上。
在线节点和离线节点都可以部署可部署实体。
```

部署规则：

```text
打出一个可部署实体时，选择1座权限节点，将该实体部署到该权限节点上。
部署在权限节点上的实体仍由其控制者控制。
部署实体不必然等于控制该节点。
```

数值：

```text
攻击力 / ATK：用于接管权限节点、压制敌方实体或参与节点争夺。
生命值 / HP：用于承受攻击、清理、回滚或系统压制。
生命值降到0的实体被处理，置入来源手牌弃牌区。
```

锚点：

```text
锚点是低成本部署实体，用于占位、侦察、连接节点或作为战术落点。
```

## 八、权限电荷资源系统

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
没有足够电荷时，不能打出对应卡牌，也不能发动对应技能。
```

费用口径：

```text
游侠打出费用默认等于当前星级。
越权模块费用按模块强度单独标注。
权限节点技能默认花费1电荷，且节点必须在线。
游侠入场效果包含在该游侠的打出费用中，不额外支付技能费用。
```
