## 三、牌库模式

### 4人共用牌库模式

```text
玩家共用1个手牌牌库。
玩家共用1个节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢查看节点牌库顶1张。
```

### 双人对战模式

```text
双方各自使用1个手牌牌库。
双人对战模式不使用节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢若在双人对战规则中启用，需要另行定义；当前默认按4人模式节点牌库口径记录。
```

非节点卡卡面原则：

```text
只有节点卡可以在卡面直接提及节点牌库。
非节点卡不得在卡面直接提及节点牌库。
```

G-006 算法口径：

```text
输入：game_mode、actor_player_id、target_deck_id。
规则引擎根据 game_mode 生成 available_deck_ids。
target_deck_id 必须属于 available_deck_ids。
效果：查看 target_deck_id 顶1张牌。
双人对战模式：available_deck_ids = [actor_hand_deck, opponent_hand_deck]。
```

## 十、节点在线 / 离线 / 保护状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
竖置：保护状态。
```

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

保护状态：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被接管。
被保护节点不能被技能变为离线。
```

胜利句：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

## 十一、12座固定节点技能草案

节点技能默认费用：

```text
花费1电荷。
只有已控制且在线的节点可以发动节点技能。
离线节点不能发动节点技能。
```

12座节点按4组设计，每组3个节点技能。

|---|---|---|---|---|

| 门禁域 | 封锁线闸门 | 节点状态组 | 硬隔离闸门系统，切断节点对外权限接口。 | 使1个节点离线。 |
| 医疗域 | 纳米诊所 | 节点控制组 | 纳米屏障诊所，临时封护一个节点，阻止接管与断线。 | 保护1个节点1回合。 |
| 医疗域 | 复苏舱 | 节点控制组 | 伤员转运复苏舱，把己方游侠送往需要支援的节点。 | 移动1个己方游侠到1个节点。 |

| 无人机域 | 蜂鸟巢 | 游侠强化组 | 蜂鸟无人机群为己方游侠提供近距火力校准。 | 选择1个己方游侠，本回合攻击力+1。 |
| 无人机域 | 空域中继 | 游侠强化组 | 空域护航中继为己方游侠投放临时防护层。 | 选择1个己方游侠，本回合生命值+1。 |
