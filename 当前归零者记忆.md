## 二、中文名称规范

```text
名称必须简单、直观，符合中文语境。
名称还必须符合《归零者》的时代背景、技术水平和世界观逻辑。
玩家看到名称后，应能直接判断它是什么，或大致理解它的用途。
名称不应依赖额外解释才能成立。
优先使用常见、明确、可识别的中文词组。
卡牌名称通常控制在2至5个汉字。
```

命名先说明对象是什么，再体现核心功能或结构，然后检查是否符合未来城市权限系统、零权限者、高科技封存载体及相关企业技术体系，最后才考虑风格修饰。

每个新名称必须同时通过：

```text
中文语境审核。
对象识别审核。
功能直觉审核。
世界观审核。
时代感审核。
```

禁止：

```text
生造难懂词。
翻译腔。
只追求气势的抽象名称。
与世界观技术体系无关的普通现代软件名。
虽然功能准确，但不像《归零者》世界中真实存在的人物、设备、物品或节点名称。
```

若名称必须经过一段解释才能理解，或与项目时代背景不符，应重新命名。

## 三、卡牌类型术语

```text
一级卡牌类型：游侠、物品卡、节点。
物品卡分为：装备、消耗品。
原术语“一次性物品”已废止，统一改称“消耗品”。
```

消耗品规则口径：

```text
消耗品打出后立即结算效果。
结算后进入公共弃牌区，除非卡面另有说明。
消耗品不作为场上实体存在。
消耗品只显示费用，不显示ATK与HP。
代码内部类型标识统一使用 consumable，不再使用 one_shot。
```

## 四、标准版基础动作池

标准版有效手牌基本动作当前为13种：

```text
1. 获得其他玩家手牌。
2. 获得手牌牌库。
3. 获得公共弃牌区。
4. 获得场上游侠。
5. 获得场上装备。
6. 调整手牌。
7. 调整手牌牌库。
8. 调整公共弃牌区。
9. 调整游侠。
10. 调整装备。
11. 消除手牌。
12. 消除游侠。
13. 消除装备。
```

当前审核结论：

```text
查看是合法底层动作，但不计入13种有效动作。
消耗品是卡牌类型限制，不是独立基础动作对象。
获得消耗品、调整消耗品、消除消耗品已删除。
消除公共弃牌区已删除，因为公共弃牌区已经是消除结果区域。
```

扩展包动作：

```text
消除手牌牌库：将手牌牌库中的卡牌直接置入公共弃牌区。
该动作会直接减少公共手牌牌库牌数。
由于标准版牌库规模有限，不进入标准版动作池。
仅允许扩展包卡牌或扩展规则使用。
使用前必须补齐牌库耗尽、空牌库抽牌和弃牌区是否重洗等规则。
```

旧编号换算：

```text
原10. 调整游侠 → 现9. 调整游侠。
原11. 调整装备 → 现10. 调整装备。
原13. 消除手牌 → 现11. 消除手牌。
原14. 消除手牌牌库 → 扩展动作：消除手牌牌库。
原16. 消除游侠 → 现12. 消除游侠。
原17. 消除装备 → 现13. 消除装备。
```

后续标准版卡表、审核文档与测试脚本必须使用当前13种编号和名称。扩展包卡牌单独使用“扩展动作：消除手牌牌库”标记。

## 七、卡面三维信息

```text
三维统一指：费用、ATK、HP。
游侠与可部署装备必须在正式卡面显示完整三维。
三维数字必须独立可读，并使用同类型固定位置。
不得使用星级替代ATK。
消耗品只显示费用，不显示ATK与HP，也不显示“不适用”占位。
节点只显示节点技能费用，不套用游侠与装备的ATK、HP三维。
```

游侠正式卡面信息：

```text
卡名、费用、ATK、HP、技能描述、编号、Logo。
```

可部署装备正式卡面信息：

```text
卡名、费用、ATK、HP、技能／功能／规则限制文本、编号、Logo。
```

消耗品正式卡面信息：

```text
卡名、费用、技能描述、编号、Logo。
```

节点正式卡面信息：

```text
卡名、系统域标签、节点技能费用、节点技能描述、卡号／节点位址、Logo。
```

## 八、装备分类与移动

```text
节点装备：只能独立部署，占用1个节点部署位。
游侠装备：只能由游侠携带，不占节点部署位。
通用装备：可以独立部署，也可以由游侠携带。
每个游侠最多携带1件装备。
```

```text
游侠执行普通移动，当前支付1电荷。
节点装备执行重新部署，费用与游侠普通移动相同。
重新部署不是移动、打出、部署或重新上场，不触发装备技能。
```
