## 三、牌库模式

### 4人共用牌库模式

```text
玩家共用1个手牌牌库。
玩家共用1个节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢查看节点牌库顶1张。
```

### 双人对战模式

```text
双方各自使用1个手牌牌库。
双人对战模式不使用节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢若在双人对战规则中启用，需要另行定义；当前默认按4人模式节点牌库口径记录。
```

非节点卡卡面原则：

```text
只有节点卡可以在卡面直接提及节点牌库。
非节点卡不得在卡面直接提及节点牌库。
```

G-006 算法口径：

```text
输入：game_mode、actor_player_id、target_deck_id。
规则引擎根据 game_mode 生成 available_deck_ids。
target_deck_id 必须属于 available_deck_ids。
效果：查看 target_deck_id 顶1张牌。
双人对战模式：available_deck_ids = [actor_hand_deck, opponent_hand_deck]。
```

## 十、节点在线 / 离线 / 保护状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
竖置：保护状态。
```

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

保护状态：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被接管。
被保护节点不能被技能变为离线。
```

胜利句：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

## 十二、通用游侠基础技能池

详细来源：

```text
22-游侠基础技能池与脱离接入定义.md
```

当前基础游侠技能池用于 2 人对战与 4 人共用牌库模式。

筛选标准：

```text
不依赖节点卡。
不直接提及节点牌库。
不与12座节点技能重复或类似。
单一基本动作。
同类技能只保留1个代表。
不包含复杂条件限制。
不包含连锁触发。
不包含跨回合长期状态记录。
不把可以通过费用、攻击力、生命值直接调整解决的内容写成技能。
```

当前主池共14个基础游侠技能：

```text
1. 抽1张牌。
2. 查看手牌牌库顶1张。

3. 查看1名对手的手牌。
4. 选择1名对手，其弃1张牌。

5. 对1个对手场上游侠造成1点伤害。
6. 选择1个对手场上游侠，本回合攻击力-1。
7. 选择1个对手场上游侠，本回合不能攻击。
8. 使1个对手场上游侠脱离接入，返回其控制者手牌。

9. 对1个对手场上装备造成1点伤害。
10. 选择1个对手场上装备，本回合攻击力-1。
11. 选择1个对手场上装备，本回合不能攻击。
12. 使1个对手场上装备脱离接入，返回其控制者手牌。

13. 使1个己方场上游侠脱离接入，返回你的手牌。
14. 使1个己方场上装备脱离接入，返回你的手牌。
```

脱离接入定义：

```text
脱离接入，是指将1个场上的可部署实体从当前接入状态中移回其控制者手牌。
当前可被脱离接入的对象：场上游侠、场上装备。
脱离接入不是伤害，不触发受到伤害相关判断。
脱离接入不是处理，不触发被处理相关判断。
脱离接入不是移动，不等同于移动到节点。
脱离接入的可部署实体离开场上时，清除其本回合临时状态。
脱离接入后返回手牌的牌，之后重新打出时，必须重新支付其打出费用。
```
