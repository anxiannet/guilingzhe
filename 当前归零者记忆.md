## 八、中央节点区与节点补充

中央节点区：

```text
中央节点区用于放置当前可被争夺的无主节点。
2到4人共用牌库模式中，中央节点区通常保持3座节点。
中央节点区中的节点初始为在线、无主、无实体。
无主节点不能成为攻击目标，只能通过部署获得节点控制权。
```

获得中央节点控制权：

```text
玩家可以将游侠或装备部署到中央节点区的无主节点。
若将实体部署到中央节点区的无主节点，该玩家获得该节点控制权。
获得控制权后，该节点从中央节点区移入该玩家的控制节点区。
随后从节点牌库翻开节点，补充中央节点区至3座。
若节点牌库为空，则不再补充。
```

节点回流：

```text
若某个已控制节点因实体离场、返回手牌、进入弃牌区或其他效果导致没有任何己方实体，该节点立即变为无主。
在使用节点牌库的模式中，该无主节点放回节点牌库并重洗。
随后从节点牌库补充中央节点区至3座。
节点回流时，清除该节点上的控制者、保护状态与临时状态。
节点回流后，默认恢复为在线、无主、无实体。
```

## 十一、回合流程与资源

当前测试回合流程：

```text
1. 回合开始：清除由当前玩家提供、持续到当前玩家回合开始的保护效果。
2. 获得权限电荷：本回合电荷 = min(当前回合数, 10)。
3. 抽牌：按当前牌库模式抽1张牌。
4. 节点技能窗口：可以使用1次已控制且在线节点的节点技能。
5. 行动阶段：可以按任意顺序打出手牌或攻击节点，只要有足够电荷支付费用。
6. 结束阶段：清除本回合临时状态，未使用电荷清空，手牌超过7张则弃至7张。
```

当前测试行动规则：

```text
每回合不限制打出手牌张数。
每回合不限制攻击节点次数。
每打出1张游侠或物品卡，立即支付其费用。
每派出1个实体攻击节点，立即支付1电荷。
节点技能每回合最多使用1次，消耗电荷。
节点技能只能在节点技能窗口使用。
若玩家在节点技能窗口未使用节点技能，本回合不能在行动阶段补用节点技能。
刚上场的游侠或装备，本回合不能参与攻击节点。
```

费用口径：

```text
游侠打出费用以卡表标注为准。
物品卡费用以卡表标注为准。
节点技能费用以节点卡面费用图标 + 数字标注为准。
节点技能句不重复写“电荷”或“控制此节点”。
游侠、装备的入场效果包含在打出费用中，不额外支付技能费用。
一次性物品结算自身效果，不额外支付技能费用。
攻击节点时，每派出1个己方实体，支付1电荷。
```

## 十二、入场技能限制

```text
当前测试规则采用“先部署，后结算入场技能”。
打出游侠或装备时，先选择节点并部署该实体，再结算该牌的卡面技能。
一次性物品不部署，直接结算卡面技能。
可部署实体的入场技能不得将己方场上实体返回手牌。
游侠不得通过入场技能将己方游侠或己方装备返回手牌。
装备不得通过入场技能将己方游侠或己方装备返回手牌。
己方场上实体返回手牌效果只允许出现在一次性物品上。
```

## 十四、ATK / HP 数值锚点

```text
旧星级不作为规则字段，只作为 ATK 的视觉锚点。
ATK 是主动突破能力。
HP 是防御与驻守能力。
费用是卡牌总价值。
生命值可以在玩家说明中理解为防御力，但正式字段仍写 HP / 生命值。
```

ATK 锚点：

```text
ATK 0：不能主动攻击节点，偏侦察、工具、后勤、无人设备、信标、数据设备。
ATK 1：轻量攻击，视觉参考旧1星游侠感，匕首、撬棍、短刀、旧工具、临时护具。
ATK 2：标准攻击，视觉参考旧2星至3星游侠感，手枪、自制枪械、土改武器、低配制式装备。
ATK 3：强攻击，视觉参考旧4星至5星游侠感，完整战术装备、稳定火力、重火力、外骨骼部件。
```

HP 锚点：

```text
HP 1：脆弱，无护甲或临时护具。
HP 2：基础防御，厚外套、拼接护具、局部护甲。
HP 3：稳定驻守，低配制式护甲、战术背心、头盔。
HP 4：强防御，完整防弹衣、重型护甲、盾牌。
HP 5：重型防御 / 工事，重型装甲、固定工事、防御地堡、外骨骼防护结构。
```

费用锚点：

```text
1费：轻量单位，低 ATK / 低 HP，技能简单。
2费：标准低费牌，可以有基础技能，或中等身材。
3费：中费牌，可以有组合技能、较好留场，或特殊被动。
4费：强牌，要么攻击强，要么防御强，要么技能强。
5费：高价值核心牌，应具备强清除、万能解、强攻坚或关键节奏能力。
```

## 十五、攻击节点与占领结算

攻击节点条件：

```text
攻击方可以从自己控制的任意节点中派出任意数量的己方游侠或装备作为攻方。
攻方不能包含本回合刚上场的实体。
攻方不能包含本回合已攻击过的实体。
攻方不能包含 ATK 为0的实体。
每派出1个攻方实体，支付1电荷。
攻击目标必须是1座由其他玩家控制的节点。
无主节点不能成为攻击目标；无主节点只能通过部署获得节点控制权。
被保护节点不能成为攻击目标。
攻击方不因目标节点最多3个实体而被提前限制为最多派出3个实体。
```

占领节点结算：

```text
若攻击结束时，目标节点没有守方实体，且至少有1个攻方实体存活：存活攻方实体进入目标节点，攻击方获得节点控制权；目标节点最多保留3个存活攻方实体，超出的返回各自来源节点。
若攻击结束时，目标节点仍有守方实体：存活攻方实体返回各自来源节点，目标节点控制权不变。
若攻击结束时，目标节点没有守方实体，且没有攻方实体存活：目标节点变为无主，目标节点上没有任何实体。
攻击节点结算完成后，检查所有来源节点；若某来源节点的控制者已没有己方实体留在该节点，该来源节点变为无主。
若使用节点牌库模式，变为无主的控制节点按节点回流规则处理。
```

## 十六、胜利条件与测试结算

正式胜利条件：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

Root Access 检查时点：

```text
玩家部署到无主节点并获得节点控制权后。
攻击节点成功，占领目标节点后。
节点在线 / 离线状态改变后。
节点变为无主后。
节点回流或中央节点区补充后。
```

测试结算顺序：

```text
1. 控制在线节点数量多者胜。
2. 若相同，比较全部在线节点上己方可部署实体 ATK 总和，较高者胜。
3. 若仍相同，比较自己在单一系统域中控制的最多在线节点数，较高者胜。
4. 若仍相同，比较自己场上可部署实体数量，较多者胜。
5. 若仍相同，视为平局。
```
