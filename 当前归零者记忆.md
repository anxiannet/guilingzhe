## 三、可部署实体与攻防生命

正式术语：

```text
可部署实体
```

可部署实体包括：

```text
1. 游侠。
2. 装备。
3. 设备。
4. 武器。
```

基础规则：

```text
每个可部署实体都有攻击力和生命值。
每个可部署实体都可以部署在在线权限节点上。
```

当前卡牌类型关系：

```text
游侠牌：默认都是可部署实体。
越权模块牌：若表现为装备、设备或武器，也视为可部署实体。
一次性反制 / 指令 / 回滚类越权模块，可以不是可部署实体。
```

部署规则：

```text
打出一个可部署实体时，选择1座在线权限节点，将该实体部署到该权限节点上。
离线权限节点不能部署新实体。
部署在权限节点上的实体仍由其控制者控制。
部署实体不必然等于控制该节点。
```

数值：

```text
攻击力 / ATK：用于接管权限节点、压制敌方实体或参与节点争夺。
生命值 / HP：用于承受攻击、清理、回滚或系统压制。
生命值降到0的实体被处理，置入来源手牌弃牌区。
```

详细规则来源：

```text
17-部署实体与攻防生命系统.md
```

## 四、权限节点在线 / 离线状态

权限节点有两种状态：

```text
正面朝上：在线状态。
背面朝上：离线状态。
```

在线权限节点：

```text
可以被接管。
可以部署可部署实体。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线权限节点：

```text
可以保留控制归属，但不能部署新实体。
不能发动节点技能。
不计入 Root Access 胜利条件。
不能用于满足“控制同一系统域3座在线权限节点”。
```

当前胜利句：

```text
控制同一系统域的3座在线权限节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Access Nodes in one System Domain to gain Root Access and win.
```

## 五、权限电荷资源系统

正式资源名：

```text
权限电荷 / Access Charge
```

卡面短称：

```text
电荷
```

当前测试规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
没有足够电荷时，不能打出对应卡牌，也不能发动对应技能。
```

费用口径：

```text
游侠打出费用默认等于当前星级。
越权模块费用按模块强度单独标注。
权限节点技能默认花费1电荷，且节点必须在线。
游侠入场效果包含在该游侠的打出费用中，不额外支付技能费用。
```

## 六、对战与多人核心规则

```text
双人 / 多人游玩时不使用节点牌库。
双人 / 多人游玩时不从节点卡中随机抽取节点。
权限节点作为场上固定争夺对象存在。
权限节点正面朝上为在线状态，背面朝上为离线状态。
离线权限节点不能发动节点技能，也不计入 Root Access 胜利条件。
游侠、装备、设备、武器都可以作为可部署实体部署在权限节点上。
每个可部署实体都有攻击力和生命值。
游侠不再拥有“查看节点牌库 / 调整节点牌库 / 重洗节点牌库”的技能。
原本与节点牌库相关的效果，转移到固定权限节点自身的节点技能体系中。
```

使用方式：

```text
2 到 4 人共用游玩：所有玩家共用同一副标准手牌牌库与同一个手牌弃牌区。
对战卡组游玩：每名玩家可以各自使用一副标准手牌牌库与对应手牌弃牌区。
```

统一区位处理：

```text
抽牌：从当前模式对应的手牌牌库抽。
弃牌：放入当前模式对应的手牌弃牌区。
处理 / 清理：置入来源手牌弃牌区。
吹回：回到该实体的控制者手牌。
放回牌库：放回来源手牌牌库。
隔离：放入隔离区，不改变控制关系；隔离结束按技能返回。
支援游侠：不是实体牌，不进入任何玩家手牌、手牌牌库或手牌弃牌区。
```

## 八、权限节点三类技能结构

每个系统域固定 3 座权限节点。

每个系统域内，3 座权限节点分别承担：

```text
侦察类
进攻类
防守类
```

分类说明：

```text
侦察类：查看、筛选、预判手牌或信息。
进攻类：强化接管、推进争夺、提升进攻实体。
防守类：保护己方节点、恢复实体、阻止或削弱敌方接管。
```
