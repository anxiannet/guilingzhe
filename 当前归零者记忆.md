## 三、牌库模式

### 4人共用牌库模式

```text
玩家共用1个手牌牌库。
玩家共用1个节点牌库。
蜂鸟无人机查看任意手牌牌库顶1张。
蜂鸟巢查看节点牌库顶1张。
```

### 双人对战模式

```text
双方各自使用1个手牌牌库。
双人对战模式不使用节点牌库。
蜂鸟无人机查看任意手牌牌库顶1张。
蜂鸟巢若在双人对战规则中启用，需要另行定义；当前默认按4人模式节点牌库口径记录。
```

非节点卡卡面原则：

```text
只有节点卡可以在卡面直接提及节点牌库。
非节点卡不得在卡面直接提及节点牌库。
```

G-006 算法口径：

```text
输入：game_mode、actor_player_id、target_deck_id。
规则引擎根据 game_mode 生成 available_deck_ids。
target_deck_id 必须属于 available_deck_ids。
效果：查看 target_deck_id 顶1张牌。
双人对战模式：available_deck_ids = [actor_hand_deck, opponent_hand_deck]。
```

技能必须同时满足：


## 六、标准版基础动作池

详细来源：

```text
22-标准版基础动作池.md
```

当前标准版游侠与物品卡基础动作主池：

```text
1. 抽牌。
2. 查看牌库顶。
3. 调整牌库顶。
4. 返回牌库。
5. 查看手牌。
6. 返回手牌。
7. 消灭。
```

备选动作：

```text
重洗牌库。
```

当前明确移除：

```text
弃牌。
公开手牌。
隔离。
禁用。
费用变化。
攻击变化。
生命变化。
伤害。
治疗。
获得电荷。
己方返回牌库。
己方消灭。
```

按作用对象展开：

```text
手牌牌库：抽牌、查看牌库顶、调整牌库顶、返回牌库；重洗牌库为备选。
对手手牌：查看手牌、返回牌库。
对手场上游侠：返回手牌、返回牌库、消灭。
对手场上装备：返回手牌、返回牌库、消灭。
己方场上游侠：返回手牌。
己方场上装备：返回手牌。
```

## 七、卡名与技能直觉原则

```text
卡名必须让玩家能直觉联想到技能。
侦察类名字对应查看。
信使、抽取、广播类名字对应抽牌。
筛选、预读、算盘、洗牌类名字对应牌库调整。
```

## 八、三维复核原则

基础动作池不设计费用、攻击、生命类技能，但卡牌本身仍需要费用、ATK、HP。

本轮数值原则：

```text
信息类技能费用最低。
抽牌类技能略高于单纯查看。
返回手牌类技能为中等强度。
返回牌库强于返回手牌，费用更高。
消灭类技能为高强度。
多目标技能高于单目标技能。
组合技能高于单动作技能。
有被动回手能力的游侠，基础身材压低。
一次性物品无 ATK / HP。
装备拥有 ATK / HP，并可部署在节点上。
```

## 十一、节点在线 / 离线 / 保护状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
竖置：保护状态。
```

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

保护状态：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被接管。
被保护节点不能被技能变为离线。
```

胜利句：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

## 十二、节点卡费用标注规则

节点卡必须在卡面标明节点技能费用。

```text
节点技能费用不是隐藏规则。
节点技能费用不能只写在规则书或说明文档中。
每张节点卡的技能区或费用标识区必须显示该节点技能费用。
节点技能费用允许按技能强度不同而变化。
```

推荐卡面短写：

```text
费用：1电荷
```

或：

```text
技能费用：1电荷
```
