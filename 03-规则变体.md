## 当前建议

《归零者 / ZERO ACCESS》当前已经作为独立项目维护。

当前规则处理原则：

```text
不主动读取或修改旧的 qiangdipan 仓库。
不把历史来源当成当前规则依赖。
保留当前已迁入的新仓库规则文件作为测试基础。
权限节点技能作为归零者当前重点测试规则。
扩展包并入标准版，形成一套 60 张标准手牌牌库。
一副标准牌既可以支持 2 到 4 人共用手牌牌库游玩，也可以作为单人对战卡组使用。
卡面文案避免使用“你的牌组 / 你的牌库 / 你的手牌牌库”。
双人对战时不使用节点牌库，也不从节点卡中抽取随机节点。
权限节点作为场上固定争夺对象存在。
游侠不再拥有关于节点牌库 / 节点卡抽取的技能。
原本与节点牌库相关的效果，转移到权限节点自身的节点技能体系中。
```

## 二、标准手牌牌库

扩展包并入标准版后，当前标准手牌牌库为 60 张：

```text
游侠：55 张
越权模块：5 张
合计：60 张
```

使用方式：

```text
2 到 4 人共用游玩：所有玩家共用同一副标准手牌牌库与同一个手牌弃牌区。
对战卡组游玩：每名玩家可以各自使用一副标准手牌牌库与对应手牌弃牌区。
```

因此卡面描述采用模式中性写法：

```text
手牌牌库
手牌弃牌区
控制者手牌
来源手牌弃牌区
来源手牌牌库
```

卡面避免写法：

```text
你的牌组
你的牌库
你的手牌牌库
```

说明：

```text
“手牌牌库”在共用游玩中指公共的标准手牌牌库。
“手牌牌库”在对战卡组游玩中指该玩家当前使用的手牌牌库。
卡面不写“你的牌库”，由具体玩法模式决定牌库归属。
```

## 三、区位与归属规则

每名玩家独立拥有：

```text
手牌
已打出游侠
已使用越权模块
已控制权限节点
```

根据玩法模式决定是否共用：

```text
共用游玩：共用手牌牌库与手牌弃牌区。
对战卡组游玩：各自拥有手牌牌库与手牌弃牌区。
```

统一处理：

```text
抽牌：从当前模式对应的手牌牌库抽。
弃牌：放入当前模式对应的手牌弃牌区。
处理 / 清理：置入来源手牌弃牌区。
吹回：回到该游侠的控制者手牌。
放回牌库：放回来源手牌牌库。
隔离：放入隔离区，不改变控制关系；隔离结束按技能返回。
支援游侠：不是实体牌，不进入任何玩家手牌、手牌牌库或手牌弃牌区。
```

禁用旧词：

```text
公共牌库
弃牌堆
节点牌库
你的牌库
你的牌组
你的手牌牌库
```

## 五、权限节点技能测试规则

本规则为 `归零者 / ZERO ACCESS` 当前重点测试规则。

双人 / 多人游玩时，权限节点不作为随机抽取的节点卡使用，而是作为场上固定争夺对象使用。

### 使用时机

```text
在你的回合中，你可以使用1次你已控制权限节点的节点技能。
```

### 使用限制

```text
每名玩家每回合最多使用1次节点技能。
即使你控制多个权限节点，每回合也只能选择其中1座权限节点发动技能。
你只能发动自己已控制权限节点的技能。
中央区无主权限节点不能发动节点技能。
没有合法目标时，不能发动对应节点技能。
```

### 与主要行动的关系

```text
节点技能不占用主要行动。
```

### 卡面写法

权限节点卡面技能默认表示：

```text
控制此节点时，你可以使用下列节点技能。
```

因此卡面不再重复写固定前缀“控制此节点：”，只写具体效果。

示例卡面写法：

```text
查看手牌牌库顶1张，可置顶或置底。
```

```text

接管权限节点
```

## 九、玩家对抗理由

每个系统域同一时间只能存在一个 Root Access。

一名玩家完成同一系统域三座权限节点控制后，即获得该系统域的最高权限。其他玩家不只是输掉一个地点，而是被该系统域降权、封锁或排除。

对抗理由：

```text
同一个系统域只承认一个最高权限。
别人先获得 Root Access，你就会被系统踢出去。
```

## 十、可选规则变体候选

以下内容仅作未来独立规则候选，暂不进入当前首版定稿。

### 权限警报轨

```text
每次使用带有“警报”图标的越权模块，权限警报 +1。
警报达到上限时，系统清场，未获得 Root Access 的玩家进入结算或失败。
```

### 身份域扩展

```text
新增身份域 Identity Domain。
控制身份域后，可恢复被删除的游侠，或封锁敌方游侠的权限。
```

### Root Token

```text
控制完整系统域后，获得 Root Token。
Root Token 可作为实体指示物，表示该系统域已被接管。
```
