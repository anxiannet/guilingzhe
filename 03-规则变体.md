```text
扩展包并入标准版，形成一套62张标准手牌牌库。
一副标准牌既可以支持2到4人共用手牌牌库游玩，也可以作为单人对战卡组使用。
4人共用牌库模式保留节点牌库。
双人对战模式不使用节点牌库，双方各自使用手牌牌库。

“权限节点”简化为“节点”。
节点正面朝上为在线状态，背面朝上为离线状态。
离线节点仍可部署、攻击、占领、控制。

12座节点的技能必须各不相同。
节点技能必须符合节点世界观。
节点技能要求简单、易懂、不连锁。
打出游侠、打出物品卡、发动节点技能都需要消耗权限电荷。
权限电荷不能保留，按回合递增，最高10电荷。
每回合不限制打出手牌张数，只限制权限电荷资源。
节点技能不占打出手牌次数，但每名玩家每回合最多使用1次节点技能。
```

## 二、牌库模式

### 4人共用牌库模式

```text
玩家共用1个手牌牌库。
玩家共用1个公共弃牌区。
玩家共用1个节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢查看节点牌库顶1张。
```

说明：

```text
4人共用牌库模式下，所有玩家从同一个标准手牌牌库抽牌。
一次性物品结算后进入公共弃牌区。
被消灭的游侠或装备进入公共弃牌区。
返回牌库类效果将目标返回共用手牌牌库的指定位置。
公共手牌牌库耗尽后，当前测试规则不自动重洗公共弃牌区。
```

### 3人共用牌库测试模式

```text
3人测试时，默认沿用4人共用牌库模式。
玩家共用1个手牌牌库。
玩家共用1个公共弃牌区。
玩家共用1个节点牌库。
```

### 双人对战模式

```text
双方各自使用1个手牌牌库。
双方各自拥有自己的弃牌区。
双人对战模式不使用节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢若在双人对战规则中启用，需要另行定义；当前默认按4人模式节点牌库口径记录。
```

## 三、G-006 算法解释

G-006 蜂鸟无人机卡面只写：

```text
查看任意牌库顶1张。
```

算法口径：

```text
输入：game_mode、actor_player_id、target_deck_id。
规则引擎根据 game_mode 生成 available_deck_ids。
target_deck_id 必须属于 available_deck_ids。
效果：查看 target_deck_id 顶1张牌。
```

4人 / 3人共用牌库模式：

```text
available_deck_ids = [shared_hand_deck]
若启用节点牌库，节点牌库只能被节点卡直接指定。
非节点卡不在卡面直接写节点牌库。
```

双人对战模式：

```text
available_deck_ids = [actor_hand_deck, opponent_hand_deck]
```

## 四、节点状态

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署可部署实体。
可以被攻击。
可以被占领。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署可部署实体。
可以被攻击。
可以被占领。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

保护节点：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被占领。
被保护节点不能被技能变为离线。
被保护节点仍可以部署实体。
```

保护持续口径：

```text
当玩家发动保护节点效果后，记录 protected_by = 该玩家。
在该玩家的下一个回合开始时，清除由该玩家提供的保护。
多人局中，不在整轮统一清除保护。
```

## 五、权限电荷

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试资源规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
没有足够电荷时，不能打出对应卡牌，也不能发动对应技能。
```

费用口径：

```text
游侠打出费用以卡表标注为准。
物品卡费用按物品强度单独标注，以卡表标注为准。
节点技能费用以节点卡面标注为准，不再全部默认1电荷。
游侠入场效果包含在该游侠的打出费用中，不额外支付技能费用。
装备入场效果包含在该装备的打出费用中，不额外支付技能费用。
一次性物品结算自身效果，不额外支付技能费用。
```

## 六、回合流程与打牌限制

每名玩家的回合依次执行：

```text
1. 回合开始：清除由当前玩家提供、持续到当前玩家回合开始的保护效果。
2. 获得权限电荷：本回合电荷 = min(当前回合数, 10)。
3. 抽牌：按当前牌库模式抽1张牌。
4. 节点技能窗口：可以使用1次已控制且在线节点的节点技能。节点技能不占打出手牌张数。
5. 打牌阶段：可以打出任意张手牌，只要有足够电荷支付费用。
6. 结束阶段：未使用电荷清空。
```

当前测试打牌规则：

```text
每回合不限制打出手牌张数。
每打出1张游侠或物品卡，立即支付其费用。
没有足够电荷时，不能打出该牌。
玩家可以在仍有电荷时继续打出手牌。
玩家可以主动停止打牌。
节点技能每回合最多使用1次，不占打出手牌张数，但消耗电荷。
```

## 七、可部署实体、部署与入场技能

```text
游侠默认是可部署实体。
装备也是可部署实体。
一次性物品不是可部署实体。
每个可部署实体都有攻击力和生命值。
每个可部署实体都可以部署在节点上。
在线节点和离线节点都可以部署可部署实体。
```

部署定义：

```text
打出一个可部署实体时，选择1个节点，将该实体部署到该节点上。
```

入场技能时点：

```text
当前测试规则采用“先部署，后结算入场技能”。
也就是说，打出游侠或装备时，先选择节点并部署该实体，再结算该牌的卡面技能。
一次性物品不部署，直接结算卡面技能。
```

## 八、当前测试控制权算法审核

当前测试版继续采用自动控制权算法，作为平衡模拟与程序测试口径。

```text
每当有可部署实体进入或离开节点、节点状态变化、实体 ATK 变化时，立即重新检查节点控制权。
对每座节点，分别计算每名玩家在该节点上的可部署实体 ATK 总和。
ATK 总和最高且大于0的玩家获得该节点控制权。
若最高 ATK 并列，节点控制权不变。
若节点无控制者且最高 ATK 并列，则该节点保持无主。
被保护节点不能因控制权检查改变控制者。
离线节点可以被攻击、占领并保留控制归属，但不计入 Root Access 胜利条件。
```

审核结论：

```text
该算法适合当前自动测试，因为它简单、稳定、可复现，能直接把部署、返回手牌、返回牌库、消灭、移动、强化转化为节点控制变化。
该算法暂时不需要引入单独“占领节点”行动。
```

需要注意的风险：

```text
1. 因为不限制打出手牌张数，资源足够时，玩家可以在同一回合连续部署多个实体，连续触发多次控制权检查。
2. 因此 Root Access 必须在每次控制权检查后立即检查，而不是只在回合结束检查。
3. 被保护节点允许部署实体，但不能改变控制者；保护解除后，下一次控制权检查才会根据当前 ATK 总和更新控制权。
4. 如果后续测试发现连续部署导致胜利过快，优先调整费用曲线和复制数，而不是恢复每回合1张手牌限制。
```

## 九、Root Access 检查时点

```text
以下时点必须立即检查 Root Access：
1. 有实体部署到节点后。
2. 有实体离开节点后。
3. 有实体 ATK 改变后。
4. 节点在线 / 离线状态改变后。
5. 保护状态解除并发生下一次控制权检查后。
```

说明：

```text
一旦有玩家控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
不等待回合结束。
不允许继续打出更多手牌改变已经达成的胜利结果。
```

## 十、胜利条件与测试结算

正式胜利条件：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

牌库耗尽与测试结算：

```text
牌库耗尽后，不自动重洗弃牌区。
需要抽牌时，若对应牌库为空，则跳过抽牌。
玩家仍可使用手牌中的卡牌。
玩家仍可通过场上实体和节点技能改变节点控制。
```

若达到模拟最大回合仍无人获得 Root Access，进入测试结算：

```text
1. 控制在线节点数量多者胜。
2. 若相同，比较全部在线节点上己方可部署实体 ATK 总和，较高者胜。
3. 若仍相同，比较自己在单一系统域中控制的最多在线节点数，较高者胜。
4. 若仍相同，比较自己场上可部署实体数量，较多者胜。
5. 若仍相同，视为平局。
```
