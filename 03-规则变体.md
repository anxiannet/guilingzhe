## 当前建议

```text
扩展包并入标准版，形成一套 60 张标准手牌牌库。
一副标准牌既可以支持 2 到 4 人共用手牌牌库游玩，也可以作为单人对战卡组使用。
双人 / 多人游玩时不使用节点牌库，也不从节点卡中抽取随机节点。
权限节点作为场上固定争夺对象存在。
权限节点正面朝上为在线状态，背面朝上为离线状态。
离线权限节点仍可部署、争夺、控制。
离线权限节点不能发动节点技能，节点信息不公开，也不计入 Root Access 胜利条件。
每个系统域的3座权限节点，分别承担侦察类、进攻类、防守类技能。
侦察类查看离线节点隐藏信息。
进攻类使节点离线。
防守类使节点在线。
打出游侠、打出越权模块、发动权限节点技能都需要消耗权限电荷。
权限电荷不能保留，按回合递增，最高10电荷。
```

## 二、权限节点在线 / 离线状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
```

在线权限节点：

```text
节点信息公开。
可以部署可部署实体。
可以发起争夺。
可以被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线权限节点：

```text
节点信息不公开。
可以部署可部署实体。
可以发起争夺。
可以被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

核心区别：

```text
离线只关闭节点技能与公开信息，不关闭部署和争夺。
```

离线节点隐藏信息包括：

```text
节点正面卡名。
系统域。
节点技能。
当前控制者。
该节点上的部署实体。
```

说明：

```text
当前主规则不使用节点牌库。
因此“查看离线节点信息”只查看场上固定权限节点的隐藏信息，不重新引入节点牌库。
查看后，该节点仍保持离线，不自动公开给所有玩家。
```

## 三、胜利条件

中文胜利句：

```text
控制同一系统域的3座在线权限节点，立即获得 Root Access 并获胜。
```

英文胜利句：

```text
Control all 3 online Access Nodes in one System Domain to gain Root Access and win.
```

## 四、权限电荷

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试资源规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
没有足够电荷时，不能打出对应卡牌，也不能发动对应技能。
```

费用口径：

```text
游侠打出费用默认等于当前星级。
越权模块费用按模块强度单独标注。
权限节点技能默认花费1电荷，且节点必须在线。
游侠入场效果包含在该游侠的打出费用中，不额外支付技能费用。
```

## 五、可部署实体与部署

```text
游侠、装备、设备、武器都视为可部署实体。
每个可部署实体都有攻击力和生命值。
每个可部署实体都可以部署在权限节点上。
在线节点和离线节点都可以部署可部署实体。
```

部署定义：

```text
打出一个可部署实体时，选择1座权限节点，将该实体部署到该权限节点上。
```

## 六、权限节点技能测试规则

### 使用时机

```text
在你的回合中，你可以花费1电荷，使用1次你已控制且在线权限节点的节点技能。
```

### 使用限制

```text
每名玩家每回合最多使用1次节点技能。
即使你控制多个在线权限节点，每回合也只能选择其中1座权限节点发动技能。
你只能发动自己已控制且在线权限节点的技能。
离线权限节点不能发动节点技能。
没有合法目标时，不能发动对应节点技能。
没有足够权限电荷时，不能发动节点技能。
```

## 七、权限节点技能分类规则

每个系统域固定包含3座权限节点。

每个系统域内，3座权限节点分别承担：

```text
侦察类：查看任意1座离线权限节点的隐藏信息。
进攻类：选择1座权限节点，使其离线。
防守类：选择1座权限节点，使其在线。
```

设计说明：

```text
侦察类用于破解离线隐藏信息。
进攻类用于关闭节点技能与公开信息，延缓 Root Access。
防守类用于恢复节点在线，使其重新可用技能并重新计入胜利条件。
```
