## 三、权限节点技能测试规则

本规则为 `归零者 / ZERO ACCESS` 当前重点测试规则。

### 使用时机

```text
在你的回合中，你可以使用1次你已控制权限节点的节点技能。
```

### 使用限制

```text
每名玩家每回合最多使用1次节点技能。
即使你控制多个权限节点，每回合也只能选择其中1座权限节点发动技能。
你只能发动自己已控制权限节点的技能。
中央区无主权限节点不能发动节点技能。
没有合法目标时，不能发动对应节点技能。
```

### 与主要行动的关系

```text
节点技能不占用主要行动。
```

一名玩家同一回合可以：

```text
使用1次节点技能
+
执行1个主要行动
```

### 卡面写法

权限节点卡面技能默认表示：

```text
控制此节点时，你可以使用下列节点技能。
```

因此卡面不再重复写固定前缀“控制此节点：”，只写具体效果。

示例卡面写法：

```text
查看节点牌库顶1张，可置顶或置底。
```

## 六、玩家对抗理由

每个系统域同一时间只能存在一个 Root Access。

一名玩家完成同一系统域三座权限节点控制后，即获得该系统域的最高权限。其他玩家不只是输掉一个地点，而是被该系统域降权、封锁或排除。

对抗理由：

```text
同一个系统域只承认一个最高权限。
别人先获得 Root Access，你就会被系统踢出去。
```

## 七、可选规则变体候选

以下内容仅作未来独立规则候选，暂不进入当前首版定稿。

### 权限警报轨

```text
每次使用带有“警报”图标的越权模块，权限警报 +1。
警报达到上限时，系统清场，未获得 Root Access 的玩家进入结算或失败。
```

### 身份域扩展

```text
新增身份域 Identity Domain。
控制身份域后，可恢复被删除的执行者，或封锁敌方执行者的权限。
```

### Root Token

```text
控制完整系统域后，获得 Root Token。
Root Token 可作为实体指示物，表示该系统域已被接管。
```

## 八、核心体验

玩家不是抢物品，而是抢控制现实的权限。

```text
系统不给你权限，你就夺取 Root Access。
```
