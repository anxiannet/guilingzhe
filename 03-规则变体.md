# 归零者｜规则变体

> 文件定位：本文件记录 `归零者 / ZERO ACCESS` 独立项目当前规则映射、测试规则与未来变体候选。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 当前建议

```text
扩展包并入标准版，形成一套62张标准手牌牌库。
一副标准牌既可以支持2到4人共用手牌牌库游玩，也可以作为单人对战卡组使用。
双人 / 多人游玩时不使用节点牌库，也不从节点卡中抽取随机节点。
节点作为场上固定争夺对象存在。
物品卡 / Gear 是所有非游侠手牌卡的统称。
旧称“越权模块”废止，不再作为卡面主类型。
“权限节点”简化为“节点”。
节点正面朝上为在线状态，背面朝上为离线状态。
离线节点仍可部署、争夺、控制。
离线节点不能发动节点技能，正面信息不公开，也不计入 Root Access 胜利条件。
当前控制者和该节点上的部署实体始终是公开信息。
被保护节点竖置，直到保护者下回合开始。
被保护节点不能被接管，不能被技能变为离线。
所有技能必须 AI 友好、算法友好、可自动测试。
所有技能不得依赖喊名、答应、语音触发或真人反应。
12座节点的技能必须各不相同。
节点技能必须符合节点世界观。
节点技能要求简单、易懂、不连锁、无条件限制。
打出游侠、打出物品卡、发动节点技能都需要消耗权限电荷。
权限电荷不能保留，按回合递增，最高10电荷。
```

## 二、技能自动测试原则

```text
所有技能必须可被规则引擎解析。
所有技能必须可自动测试。
所有技能不得依赖喊名、答应、语音触发或真人反应。
```

技能文本必须明确：

```text
目标。
数量。
区域。
动作。
持续时间。
归属。
```

禁止：

```text
喊一名玩家。
喊一名玩家名字。
对方答应后。
如果对方听见。
如果对方回应。
口头确认。
现实语音触发。
玩家主观判断触发。
```

## 四、节点在线 / 离线状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
竖置：保护状态。
```

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署可部署实体。
可以发起争夺。
可以被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署可部署实体。
可以发起争夺。
可以被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

## 五、节点保护状态

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被接管。
被保护节点不能被技能变为离线。
被保护节点仍可以部署实体。
```


英文：

## 七、权限电荷

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试资源规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
没有足够电荷时，不能打出对应卡牌，也不能发动对应技能。
```

费用口径：

```text
游侠打出费用默认等于当前星级。
物品卡费用按物品强度单独标注。
节点技能默认花费1电荷，且节点必须在线。
游侠入场效果包含在该游侠的打出费用中，不额外支付技能费用。
```

## 八、可部署实体与部署

```text
游侠默认是可部署实体。
物品卡若可以留在场上，也可以是可部署实体。
每个可部署实体都有攻击力和生命值。
每个可部署实体都可以部署在节点上。
在线节点和离线节点都可以部署可部署实体。
```

部署定义：

```text
打出一个可部署实体时，选择1座节点，将该实体部署到该节点上。
```

## 九、节点技能测试规则

```text
在你的回合中，你可以花费1电荷，使用1次你已控制且在线节点的节点技能。
```

限制：

```text
每名玩家每回合最多使用1次节点技能。
即使你控制多个在线节点，每回合也只能选择其中1座节点发动技能。
你只能发动自己已控制且在线节点的技能。
离线节点不能发动节点技能。
没有合法目标时，不能发动对应节点技能。
没有足够权限电荷时，不能发动节点技能。
```
