```text
扩展包并入标准版，形成一套62张标准手牌牌库。
一副标准牌既可以支持2到4人共用手牌牌库游玩，也可以作为单人对战卡组使用。
4人共用牌库模式保留节点牌库。
双人对战模式不使用节点牌库，双方各自使用手牌牌库。
```

```text

蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢若在双人对战规则中启用，需要另行定义；当前默认按4人模式节点牌库口径记录。
```

## 三、G-006 算法解释

G-006 蜂鸟无人机卡面只写：

```text
查看任意牌库顶1张。
```

算法口径：

```text
输入：game_mode、actor_player_id、target_deck_id。
规则引擎根据 game_mode 生成 available_deck_ids。
target_deck_id 必须属于 available_deck_ids。
效果：查看 target_deck_id 顶1张牌。
```

双人对战模式：

```text
available_deck_ids = [actor_hand_deck, opponent_hand_deck]
```

说明：

```text
非节点卡不在卡面直接写节点牌库。
节点牌库作为具体牌库名称，只允许节点卡直接提及。
```

## 四、技能自动测试原则

```text
所有技能必须可被规则引擎解析。
所有技能必须可自动测试。
所有技能不得依赖喊名、答应、语音触发或真人反应。
```

技能文本必须明确：

```text
目标。
数量。
区域。
```

```text
喊一名玩家。
喊一名玩家名字。
对方答应后。
如果对方听见。
如果对方回应。
口头确认。
现实语音触发。
```

## 六、节点在线 / 离线状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
竖置：保护状态。
```

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署可部署实体。
可以发起争夺。
可以被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署可部署实体。
可以发起争夺。
可以被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

## 七、节点保护状态

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被接管。
被保护节点不能被技能变为离线。
被保护节点仍可以部署实体。
```

## 九、权限电荷

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试资源规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
没有足够电荷时，不能打出对应卡牌，也不能发动对应技能。
```

费用口径：

```text
游侠打出费用默认等于当前星级。
物品卡费用按物品强度单独标注。
节点技能默认花费1电荷，且节点必须在线。
游侠入场效果包含在该游侠的打出费用中，不额外支付技能费用。
```

## 十、可部署实体与部署

```text
游侠默认是可部署实体。
实体物品也是可部署实体。
一次性物品不是可部署实体。
每个可部署实体都有攻击力和生命值。
每个可部署实体都可以部署在节点上。
在线节点和离线节点都可以部署可部署实体。
```

部署定义：

```text
打出一个可部署实体时，选择1个节点，将该实体部署到该节点上。
```
