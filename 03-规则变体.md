# 归零者｜规则变体

> 文件定位：本文件记录 `归零者 / ZERO ACCESS` 独立项目当前规则映射、测试规则与未来变体候选。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`
>
> 说明：本文件可参考旧项目《夕妖：抢地盘》的公共牌库、回合流程、保护持续与结算思路，但《归零者 / ZERO ACCESS》仍是独立项目，当前规则以本仓库为准。

## 当前建议

```text
扩展包并入标准版，形成一套62张标准手牌牌库。
一副标准牌既可以支持2到4人共用手牌牌库游玩，也可以作为单人对战卡组使用。
4人共用牌库模式保留节点牌库。
双人对战模式不使用节点牌库，双方各自使用手牌牌库。
只有节点卡可以在卡面直接提及节点牌库。
非节点卡不得在卡面直接提及节点牌库。
G-006 蜂鸟无人机卡面写作“查看任意牌库顶1张”。
节点作为场上固定争夺对象存在。
物品卡 / Gear 是所有非游侠手牌卡的统称。
装备是可以作为可部署实体进入节点的物品卡。
旧称“越权模块”废止，不再作为卡面主类型。
“权限节点”简化为“节点”。
节点正面朝上为在线状态，背面朝上为离线状态。
离线节点仍可部署、争夺、控制。
离线节点不能发动节点技能，正面信息不公开，也不计入 Root Access 胜利条件。
当前控制者和该节点上的部署实体始终是公开信息。
被保护节点竖置，直到保护者下回合开始。
被保护节点不能被接管，不能被技能变为离线。
所有技能必须 AI 友好、算法友好、可自动测试。
所有技能不得依赖喊名、答应、语音触发或真人反应。
12座节点的技能必须各不相同。
节点技能必须符合节点世界观。
节点技能要求简单、易懂、不连锁。
打出游侠、打出物品卡、发动节点技能都需要消耗权限电荷。
权限电荷不能保留，按回合递增，最高10电荷。
每名玩家每回合最多执行1个主要行动。
节点技能不占主要行动，但每名玩家每回合最多使用1次节点技能。
```

## 二、牌库模式

### 4人共用牌库模式

```text
玩家共用1个手牌牌库。
玩家共用1个公共弃牌区。
玩家共用1个节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢查看节点牌库顶1张。
```

说明：

```text
4人共用牌库模式下，所有玩家从同一个标准手牌牌库抽牌。
一次性物品结算后进入公共弃牌区。
被消灭的游侠或装备进入公共弃牌区。
返回牌库类效果将目标返回共用手牌牌库的指定位置。
公共手牌牌库耗尽后，当前测试规则不自动重洗公共弃牌区。
```

### 3人共用牌库测试模式

```text
3人测试时，默认沿用4人共用牌库模式。
玩家共用1个手牌牌库。
玩家共用1个公共弃牌区。
玩家共用1个节点牌库。
```

说明：

```text
3人不是当前主规格，但模拟器需要支持。
3人局用于压力测试多人争夺，不作为印刷规则优先目标。
```

### 双人对战模式

```text
双方各自使用1个手牌牌库。
双方各自拥有自己的弃牌区。
双人对战模式不使用节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢若在双人对战规则中启用，需要另行定义；当前默认按4人模式节点牌库口径记录。
```

## 三、G-006 算法解释

G-006 蜂鸟无人机卡面只写：

```text
查看任意牌库顶1张。
```

算法口径：

```text
输入：game_mode、actor_player_id、target_deck_id。
规则引擎根据 game_mode 生成 available_deck_ids。
target_deck_id 必须属于 available_deck_ids。
效果：查看 target_deck_id 顶1张牌。
```

4人 / 3人共用牌库模式：

```text
available_deck_ids = [shared_hand_deck]
若启用节点牌库，节点牌库只能被节点卡直接指定。
非节点卡不在卡面直接写节点牌库。
```

双人对战模式：

```text
available_deck_ids = [actor_hand_deck, opponent_hand_deck]
```

说明：

```text
非节点卡不在卡面直接写节点牌库。
节点牌库作为具体牌库名称，只允许节点卡直接提及。
```

## 六、节点在线 / 离线状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
竖置：保护状态。
```

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署可部署实体。
可以发起争夺。
可以被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署可部署实体。
可以发起争夺。
可以被接管。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

---

## 七、节点保护状态

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被接管。
被保护节点不能被技能变为离线。
被保护节点仍可以部署实体。
```

保护持续口径：

```text
当玩家发动保护节点效果后，记录 protected_by = 该玩家。
在该玩家的下一个回合开始时，清除由该玩家提供的保护。
多人局中，不在整轮统一清除保护。
```

## 九、权限电荷

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试资源规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
没有足够电荷时，不能打出对应卡牌，也不能发动对应技能。
```

费用口径：

```text
游侠打出费用以卡表标注为准。
物品卡费用按物品强度单独标注，以卡表标注为准。
节点技能费用以节点卡面标注为准，不再全部默认1电荷。
游侠入场效果包含在该游侠的打出费用中，不额外支付技能费用。
装备入场效果包含在该装备的打出费用中，不额外支付技能费用。
一次性物品结算自身效果，不额外支付技能费用。
```

## 十、回合流程与主要行动

每名玩家的回合依次执行：

```text
1. 回合开始：清除由当前玩家提供、持续到当前玩家回合开始的保护效果。
2. 获得权限电荷：本回合电荷 = min(当前回合数, 10)。
3. 抽牌：按当前牌库模式抽1张牌。
4. 节点技能窗口：可以使用1次已控制且在线节点的节点技能。节点技能不占主要行动。
5. 主要行动：选择并执行1个主要行动。
6. 结束阶段：未使用电荷清空。
```

当前测试主要行动：

```text
打出1张游侠。
打出1张物品卡。
放弃行动。
```

说明：

```text
每名玩家每回合最多执行1个主要行动。
每名玩家每回合最多使用1次节点技能。
当前测试版暂不设置独立“接管节点”行动按钮；节点控制在可部署实体变化后按接管算法自动检查。
```

## 十一、可部署实体与部署

```text
游侠默认是可部署实体。
装备也是可部署实体。
一次性物品不是可部署实体。
每个可部署实体都有攻击力和生命值。
每个可部署实体都可以部署在节点上。
在线节点和离线节点都可以部署可部署实体。
```

部署定义：

```text
打出一个可部署实体时，选择1个节点，将该实体部署到该节点上。
```

入场技能时点：

```text
当前测试规则采用“先部署，后结算入场技能”。
也就是说，打出游侠或装备时，先选择节点并部署该实体，再结算该牌的卡面技能。
一次性物品不部署，直接结算卡面技能。
```

## 十二、当前测试接管算法

当前测试版采用自动接管算法，作为平衡模拟与程序测试口径。

```text
每当有可部署实体进入或离开节点、节点状态变化、实体 ATK 变化时，立即重新检查节点控制权。
对每座节点，分别计算每名玩家在该节点上的可部署实体 ATK 总和。
ATK 总和最高且大于0的玩家控制该节点。
若最高 ATK 并列，节点控制权不变。
若节点无控制者且最高 ATK 并列，则该节点保持无主。
被保护节点不能因接管检查改变控制者。
离线节点可以被接管和保留控制归属，但不计入 Root Access 胜利条件。
```

说明：

```text
该接管算法是当前测试算法。
后续若改为显式“接管节点”主要行动，必须同步更新测试代码。
```

## 十三、牌库耗尽与测试结算

参考旧项目公共牌库耗尽处理，本项目当前测试口径为：

```text
牌库耗尽后，不自动重洗弃牌区。
需要抽牌时，若对应牌库为空，则跳过抽牌。
玩家仍可使用手牌中的卡牌。
玩家仍可通过场上实体和节点技能改变节点控制。
```

若达到模拟最大回合仍无人获得 Root Access，进入测试结算：

```text
1. 控制在线节点数量多者胜。
2. 若相同，比较全部在线节点上己方可部署实体 ATK 总和，较高者胜。
3. 若仍相同，比较自己在单一系统域中控制的最多在线节点数，较高者胜。
4. 若仍相同，比较自己场上可部署实体数量，较多者胜。
5. 若仍相同，视为平局。
```

说明：

```text
测试结算只用于模拟器超时判定。
正式胜利条件仍优先是控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```
