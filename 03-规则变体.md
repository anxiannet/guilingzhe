## 二、双人对战区位规则

双人对战时，每名玩家各自使用一副手牌牌库。

```text
手牌牌库 = 游侠牌 + 越权模块牌组成的牌库。
```

每名玩家独立拥有：

```text
自己的手牌牌库
自己的手牌弃牌区
自己的手牌
自己的已打出游侠
自己的越权模块
```

双方共用或竞争：

```text
12 座固定权限节点
Root Access / 系统域状态
```

双人对战不使用：

```text
节点牌库
随机抽取节点卡
从节点牌库查看 / 置顶 / 置底 / 重洗节点卡
```

防混牌原则：

```text
每张游侠牌与越权模块牌始终归属于其拥有者。
任何技能都不能改变卡牌拥有者。
抽牌只从自己的手牌牌库抽。
弃牌只放入自己的手牌弃牌区。
处理 / 清理目标时，牌进入其拥有者手牌弃牌区。
吹回目标时，牌回到其拥有者手牌。
放回牌库时，牌放回其拥有者手牌牌库。
```

禁用旧词：

```text
公共牌库
弃牌堆
节点牌库
```

统一改为：

```text
手牌牌库
手牌弃牌区
固定权限节点
拥有者手牌
拥有者手牌弃牌区
拥有者手牌牌库
```

## 四、权限节点技能测试规则

本规则为 `归零者 / ZERO ACCESS` 当前重点测试规则。

双人对战时，权限节点不作为随机抽取的节点卡使用，而是作为场上固定争夺对象使用。

### 使用时机

```text
在你的回合中，你可以使用1次你已控制权限节点的节点技能。
```

### 使用限制

```text
每名玩家每回合最多使用1次节点技能。
即使你控制多个权限节点，每回合也只能选择其中1座权限节点发动技能。
你只能发动自己已控制权限节点的技能。
中央区无主权限节点不能发动节点技能。
没有合法目标时，不能发动对应节点技能。
```

### 与主要行动的关系

```text
节点技能不占用主要行动。
```

一名玩家同一回合可以：

```text
使用1次节点技能
+
执行1个主要行动
```

### 卡面写法

权限节点卡面技能默认表示：

```text
控制此节点时，你可以使用下列节点技能。
```

因此卡面不再重复写固定前缀“控制此节点：”，只写具体效果。

示例卡面写法：

```text
查看你的手牌牌库顶1张，可置顶或置底。
```

## 七、游侠技能与节点技能边界

游侠技能可以操作：

```text
手牌牌库
手牌
手牌弃牌区
场上游侠
接管权限节点
越权模块
非实体支援游侠
```

游侠技能不再操作：

```text
节点牌库

节点牌库重洗
随机节点卡抽取
```

权限节点技能可以操作：

```text
固定权限节点状态
接管加成
手牌牌库筛选
节点间移动
防守限制
医疗恢复
```

## 八、混牌风险处理原则

### 处理 / 清理

```text
处理或清理游侠时，该游侠进入其拥有者手牌弃牌区。
```

### 吹回

```text
吹回游侠时，该游侠回到其拥有者手牌。
```

### 隔离

```text
隔离不改变拥有者。
被隔离的游侠放入其拥有者隔离区。
隔离结束后，返回原权限节点或按技能说明处理。
```

### 非实体支援

```text
部分技能可产生“支援游侠”。
支援游侠不是实体牌，不进入任何玩家手牌、牌库或弃牌区。
```

## 九、玩家对抗理由

每个系统域同一时间只能存在一个 Root Access。

一名玩家完成同一系统域三座权限节点控制后，即获得该系统域的最高权限。其他玩家不只是输掉一个地点，而是被该系统域降权、封锁或排除。

对抗理由：

```text
同一个系统域只承认一个最高权限。
别人先获得 Root Access，你就会被系统踢出去。
```

## 十、可选规则变体候选

以下内容仅作未来独立规则候选，暂不进入当前首版定稿。

### 权限警报轨

```text
每次使用带有“警报”图标的越权模块，权限警报 +1。
警报达到上限时，系统清场，未获得 Root Access 的玩家进入结算或失败。
```

### 身份域扩展

```text
新增身份域 Identity Domain。
控制身份域后，可恢复被删除的游侠，或封锁敌方游侠的权限。
```

### Root Token

```text
控制完整系统域后，获得 Root Token。
Root Token 可作为实体指示物，表示该系统域已被接管。
```
