# 归零者｜规则变体

> 文件定位：本文件记录 `归零者 / ZERO ACCESS` 独立项目当前规则映射、测试规则与未来变体候选。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`
>
> 说明：本文件可参考旧项目《夕妖：抢地盘》的公共牌库、回合流程、保护持续与结算思路，但《归零者 / ZERO ACCESS》仍是独立项目，当前规则以本仓库为准。

---

## 当前建议

```text
扩展包并入标准版，形成一套62张标准手牌牌库。
一副标准牌既可以支持2到4人共用手牌牌库游玩，也可以作为单人对战卡组使用。
4人共用牌库模式保留节点牌库。
双人对战模式不使用节点牌库，双方各自使用手牌牌库。
节点作为场上固定争夺对象存在。
物品卡 / Gear 是所有非游侠手牌卡的统称。
装备是可以作为可部署实体进入节点的物品卡。
旧称“越权模块”废止，不再作为卡面主类型。
“权限节点”简化为“节点”。
节点只属于一个玩家，或处于无主状态。
一个节点上不能同时存在多个玩家的实体。
无主节点上不能存在任何实体。
每座节点最多有3个实体。
刚上场的游侠或装备，本回合不能参与攻击节点。
每回合不限制打出手牌张数，只限制权限电荷资源。
```

| 行动单位 / 人物牌 | 游侠 / Operative |
| 非游侠手牌卡 | 物品卡 / Gear |
| 物品卡形态 | 装备、一次性物品 |
| 可部署实体 | 游侠、装备 |
| 攻击力 | ATK |
| 生命值 | HP |

| 固定场地 | 12座节点 |

禁用术语：

```text

不再使用“被接管”。
不再使用“接管算法”。

胜利条件统一写“控制节点”。
玩家行为可写“攻击节点”或“占领节点”。
```

## 二、牌库模式

### 4人共用牌库模式

```text
玩家共用1个手牌牌库。
玩家共用1个公共弃牌区。
玩家共用1个节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢查看节点牌库顶1张。
```

### 3人共用牌库测试模式

```text
3人测试时，默认沿用4人共用牌库模式。
玩家共用1个手牌牌库。
玩家共用1个公共弃牌区。
玩家共用1个节点牌库。
```

### 双人对战模式

```text
双方各自使用1个手牌牌库。
双方各自拥有自己的弃牌区。
双人对战模式不使用节点牌库。
蜂鸟无人机查看任意牌库顶1张。
```

说明：

```text
公共手牌牌库耗尽后，当前测试规则不自动重洗公共弃牌区。
需要抽牌时，若对应牌库为空，则跳过抽牌。
```

## 三、节点状态

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署可部署实体。
可以被攻击。
可以被占领。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署可部署实体。
可以被攻击。
可以被占领。
可以保留控制归属。
不能发动节点技能。
不计入 Root Access 胜利条件。
```

保护节点：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能成为攻击节点的目标。
被保护节点不能被技能变为离线。
被保护节点仍可以由控制者部署实体增援。
无主节点不能被保护。
```

保护持续口径：

```text
当玩家发动保护节点效果后，记录 protected_by = 该玩家。
在该玩家的下一个回合开始时，清除由该玩家提供的保护。
多人局中，不在整轮统一清除保护。
```

## 四、权限电荷

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试资源规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
没有足够电荷时，不能打出对应卡牌、攻击节点或发动对应技能。
```

费用口径：

```text
游侠打出费用以卡表标注为准。
物品卡费用以卡表标注为准。
节点技能费用以节点卡面标注为准。
游侠、装备的入场效果包含在打出费用中，不额外支付技能费用。
一次性物品结算自身效果，不额外支付技能费用。
攻击节点时，每派出1个己方实体，支付1电荷。
```

说明：

```text
当前测试版先采用“每派出1个实体攻击，支付1电荷”。
如果后续测试发现攻击过强或过弱，再调整攻击费用，而不是限制攻击实体数量。
```

## 五、回合流程与行动限制

每名玩家的回合依次执行：

```text
1. 回合开始：清除由当前玩家提供、持续到当前玩家回合开始的保护效果。
2. 获得权限电荷：本回合电荷 = min(当前回合数, 10)。
3. 抽牌：按当前牌库模式抽1张牌。
4. 节点技能窗口：可以使用1次已控制且在线节点的节点技能。
5. 行动阶段：可以打出任意张手牌，也可以攻击节点，只要有足够电荷支付费用。
6. 结束阶段：清除本回合临时状态，未使用电荷清空。
```

当前测试行动规则：

```text
每回合不限制打出手牌张数。
每回合不限制攻击节点次数。
每打出1张游侠或物品卡，立即支付其费用。
每派出1个实体攻击节点，立即支付1电荷。
没有足够电荷时，不能执行对应行动。
玩家可以在仍有电荷时继续打牌或继续攻击。
玩家可以主动停止行动。
节点技能每回合最多使用1次，消耗电荷。
刚上场的游侠或装备，本回合不能参与攻击节点。
```

## 六、可部署实体、驻守上限与节点归属

```text
游侠默认是可部署实体。
装备也是可部署实体。
一次性物品不是可部署实体。
每个可部署实体都有攻击力和生命值。
```

驻守上限：

```text
每座节点最多有3个实体。
一个节点上的实体必须全部属于该节点控制者。
一个节点上不能同时存在多个玩家的实体。
无主节点上不能存在任何实体。
```

节点归属：

```text
一座节点最多只属于一个玩家。
节点也可以处于无主状态。
控制节点的玩家称为该节点控制者。
控制节点的必要条件：该节点上至少有1个该玩家的己方实体。
若节点控制者在该节点上没有任何己方实体，该节点立即变为无主。
```

部署定义：

```text
打出游侠或装备时，选择1座无主节点或你已控制的节点，将该实体部署到该节点上。
不能从手牌直接部署到其他玩家控制的节点。
若目标节点已有3个实体，不能选择该节点。
若将实体部署到无主节点，你获得该节点控制权。
若将实体部署到你已控制的节点，视为增援节点。
```

入场限制：

```text
刚上场的实体，指本回合打出并部署到节点上的游侠或装备。
刚上场的实体可以留在节点上提供驻守压力。
刚上场的实体本回合不能被选择为攻击节点的攻方。
从下一次其控制者回合开始起，该实体才可以参与攻击节点。
移动不是上场；通过节点技能移动到新节点的旧实体，如果本回合尚未攻击，可以参与攻击节点。
```

入场技能时点：

```text
当前测试规则采用“先部署，后结算入场技能”。
打出游侠或装备时，先选择节点并部署该实体，再结算该牌的卡面技能。
一次性物品不部署，直接结算卡面技能。
```

## 七、攻击节点条件

攻击节点必须满足：

```text
1. 攻击方选择任意数量的己方游侠或装备作为攻方。
2. 被选择的攻方必须来自攻击方已控制的节点。
3. 被选择的攻方不能是本回合刚上场的实体。
4. 攻方可以来自一个或多个自己控制的节点。
5. 每派出1个攻方实体，支付1电荷。
6. 攻击方不能派出超过当前可支付电荷数量的实体。
7. 攻击方不能选择已经参与过本回合攻击的实体。
8. 攻击目标必须是1座由其他玩家控制的节点。
9. 无主节点不能成为攻击目标；无主节点只能通过部署获得节点控制权。
10. 被保护节点不能成为攻击目标。
11. ATK 为0的实体不能参与攻击节点。
```

说明：

```text
攻击方可以从自己控制的任意节点中派出任意数量的游侠和装备攻击。
实际派出数量只受剩余电荷、可用实体数量、目标节点容量、本回合是否已攻击、是否刚上场、ATK是否大于0限制。
```

目标节点容量：

```text
如果攻击成功，存活攻方会进入目标节点。
因此，攻击方选择攻方数量时，必须保证成功后目标节点实体数量不会超过3个。
```

## 八、攻击节点结算流程

完整流程：

```text
1. 攻击方选择1座由其他玩家控制且未被保护的目标节点。
2. 攻击方从自己控制的任意节点中选择任意数量的己方游侠或装备作为攻方。
3. 攻方不能包含本回合刚上场的实体，不能包含本回合已攻击过的实体，不能包含 ATK 为0的实体。
4. 攻击方为每个攻方实体支付1电荷。
5. 攻击方决定攻方实体的攻击顺序。
6. 目标节点控制者作为守方。
7. 守方每次选择1个守方实体迎战。
8. 当前攻方实体与当前守方实体同时造成等于各自 ATK 的伤害。
9. 受到伤害大于等于 HP 的实体被消灭，进入对应弃牌区。
10. 每个攻方实体最多攻击1次。
11. 重复结算，直到攻方实体都攻击过、攻方全部被消灭，或目标节点没有守方实体。
```

伤害口径：

```text
战斗中的伤害只在本次攻击节点期间记录。
攻击节点结束后，所有存活实体清除本次战斗中受到的伤害。
战斗伤害不跨攻击保留。
```

## 九、占领节点结算

### 占领成功

```text
若攻击结束时，目标节点没有守方实体，且至少有1个攻方实体存活：
存活攻方实体进入目标节点。
攻击方获得节点控制权。
目标节点上的实体只保留这些存活攻方实体。
```

### 占领失败

```text
若攻击结束时，目标节点仍有守方实体：
存活攻方实体返回各自来源节点。
目标节点控制权不变。
```

### 双方清空

```text
若攻击结束时，目标节点没有守方实体，且没有攻方实体存活：
目标节点变为无主。
目标节点上没有任何实体。
```

来源节点结算：

```text
攻击节点期间，来源节点控制权暂不结算。
攻击节点结算完成后，检查所有来源节点。
若某来源节点的控制者已没有己方实体留在该节点，该来源节点变为无主。
```

说明：

```text
节点只属于一个玩家。
一个节点上不能同时存在多个玩家实体。
获得节点控制权只在部署到无主节点或攻击节点结算成功时发生。
不再使用“节点上 ATK 总和最高者自动获得控制权”的旧测试算法。
```

## 十、Root Access 检查时点

以下时点必须立即检查 Root Access：

```text
1. 玩家部署到无主节点并获得节点控制权后。
2. 攻击节点成功，占领目标节点后。
3. 节点在线 / 离线状态改变后。
4. 节点变为无主后。
```

说明：

```text
一旦有玩家控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
不等待回合结束。
不允许继续打出更多手牌或继续攻击改变已经达成的胜利结果。
```

## 十一、胜利条件与测试结算

正式胜利条件：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

若达到模拟最大回合仍无人获得 Root Access，进入测试结算：

```text
1. 控制在线节点数量多者胜。
2. 若相同，比较全部在线节点上己方可部署实体 ATK 总和，较高者胜。
3. 若仍相同，比较自己在单一系统域中控制的最多在线节点数，较高者胜。
4. 若仍相同，比较自己场上可部署实体数量，较多者胜。
5. 若仍相同，视为平局。
```

说明：

```text
测试结算只用于模拟器超时判定。
正式胜利条件仍优先是控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

## 十二、临时状态

```text
本回合 ATK / HP 增益在当前玩家回合结束时清除。
实体离场、返回手牌、返回牌库或进入弃牌区时，清除该实体所有临时状态。
刚上场状态只用于限制本回合不能攻击。
战斗伤害只在本次攻击节点期间记录，攻击结束后清除。
```
