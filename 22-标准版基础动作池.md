# 归零者｜标准版基础动作池

> 文件定位：本文件用于记录《归零者 / ZERO ACCESS》标准版中游侠与物品卡技能可使用的基础动作池。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、当前结论

标准版游侠与物品卡技能必须遵守：

```text
标准版游侠技能，只能由本文件中的基础动作进行变体或组合形成。
标准版物品卡技能，只能由本文件中的基础动作进行变体或组合形成。
不在基础动作池中的效果，不进入标准版游侠与物品卡技能。
```

说明：

```text
基础动作是技能设计元素，不等于最终卡面技能。
实际卡牌技能可以在基础动作上增加目标、数量、星级、阵营、区域、次数、时机等限制。
实际卡牌技能也可以由多个基础动作组合形成。
```

---

## 二、基础动作池与节点动作池边界

```text
基础动作池只限制游侠与物品卡技能。
节点技能使用独立节点动作池，包括查看节点、上线、离线、保护、移动、临时强化、从弃牌区回收。
节点技能不得反向扩展为标准版手牌技能。
```

说明：

```text
攻击变化、生命变化、上线、离线、保护、移动、临时强化、从弃牌区回收等效果，不能进入标准版游侠与物品卡技能。
上述效果只允许作为节点技能存在，并以 03-规则变体.md 与 04-卡表.md 中的节点规则为准。
```

---

## 三、标准版基础动作筛选原则

标准版基础动作要求：

```text
1. 规则直观。
2. 易于自动测试。
3. 不依赖真人反应。
4. 不制造长期状态记录。
5. 不直接围绕卡牌三维设计。
6. 不与节点技能创建平行体系。
7. 后续技能可通过变体和限制条件扩展。
```

标准版基础动作池不收录直接围绕卡牌三维的动作：

```text
费用。
攻击。
生命。
```

因此以下动作不进入标准版基础动作池：

```text
获得电荷。
费用减少。
攻击力增加。
攻击力减少。
不能攻击。
生命值增加。
恢复生命。
造成伤害。
```

说明：

```text
造成伤害虽然不直接写“生命值-1”，但本质影响生命维度，因此不进入标准版基础动作池。
不能攻击虽然不直接修改攻击力，但围绕攻击维度，因此不进入标准版基础动作池。
```

## 四、当前明确移除的动作

以下动作不进入标准版基础动作池：

```text
弃牌。
公开手牌。
隔离。
禁用。
费用变化。
攻击变化。
生命变化。
伤害。
治疗。
获得电荷。
己方返回牌库。
己方消灭。
```

移除理由：

| 动作 | 移除理由 |
|---|---|
| 弃牌 | 标准版牌量有限，不使用直接弃牌压制。 |
| 公开手牌 | 对玩家收益不大，信息价值弱于查看手牌。 |
| 隔离 | 需要额外隔离区与返回时机，不是基础动作。 |
| 禁用 | 与返回手牌重叠，用返回手牌替代。 |
| 费用变化 | 涉及卡牌三维之一：费用。 |
| 攻击变化 | 涉及卡牌三维之一：攻击。 |
| 生命变化 | 涉及卡牌三维之一：生命。 |
| 伤害 | 本质影响生命维度。 |
| 治疗 | 本质影响生命维度。 |
| 获得电荷 | 等价于费用节奏变化。 |
| 己方返回牌库 | 收益不直观，增加判断成本。 |
| 己方消灭 | 收益不直观，容易变成连锁触发素材。 |

## 五、标准版基础动作主池

当前标准版基础动作主池为 7 个：

```text
1. 抽牌。
2. 查看牌库顶。
3. 调整牌库顶。
4. 返回牌库。
5. 查看手牌。
6. 返回手牌。
7. 消灭。
```

备选动作：

```text
重洗牌库。
```

说明：

```text
重洗牌库有明确规则意义，但体感较弱，暂列备选，不默认进入标准版主池。
```

## 六、基础动作定义

### 1. 抽牌

```text
从手牌牌库抽取牌，加入对应玩家手牌。
```

常见变体：

```text
抽1张牌。
抽2张牌。
选择1名玩家，其抽1张牌。
```

### 2. 查看牌库顶

```text
查看指定手牌牌库顶部的牌，不改变牌库顺序。
```

常见变体：

```text
查看手牌牌库顶1张。
查看你的手牌牌库顶2张。
查看任意手牌牌库顶1张。
```

### 3. 调整牌库顶

```text
改变指定手牌牌库顶部牌的顺序或位置。
```

常见变体：

```text
将手牌牌库顶1张置底。
查看手牌牌库顶2张，任意排序后放回。
查看手牌牌库顶3张，任意排序后放回。
```

说明：

```text
“查看后置底”属于组合动作：查看牌库顶 + 返回牌库。
基础动作池中将其拆开记录。
```

### 4. 返回牌库

```text
将目标牌放回其控制者或拥有者的手牌牌库。
```

返回位置包括：

```text
牌库顶。
牌库底。
洗入牌库。
```

常见变体：

```text
选择1名对手，其将1张手牌放到手牌牌库底。
将1个对手场上游侠返回其控制者手牌牌库底。
将1个对手场上装备返回其控制者手牌牌库底。
将手牌牌库顶1张置底。
```

限制：

```text
标准版基础池不保留己方场上游侠返回己方牌库。
标准版基础池不保留己方场上装备返回己方牌库。
```

### 5. 查看手牌

```text
查看指定玩家的手牌。
```

常见变体：

```text
查看1名对手的手牌。
查看1名玩家的手牌。
```

限制：

```text
公开手牌不进入基础动作池。
弃牌不进入基础动作池。
```

### 6. 返回手牌

```text
将1个场上的可部署实体从当前场上部署状态移回其控制者手牌。
```

可返回手牌的对象：

```text
场上游侠。
场上装备。
```

不可返回手牌的对象：

```text
节点。
一次性物品。
手牌中的牌。
牌库中的牌。
弃牌区中的牌。
已经被消灭、隔离或不在场上的牌。
```

常见变体：

```text
将1个对手场上游侠返回其控制者手牌。
将1个对手场上装备返回其控制者手牌。
将1个己方场上游侠返回你的手牌。
将1个己方场上装备返回你的手牌。
```

规则说明：

```text
返回手牌不是伤害，不触发受到伤害相关判断。
返回手牌不是消灭，不触发被消灭相关判断。
返回手牌不是移动，不等同于移动到节点。
返回手牌的可部署实体离开场上时，清除其本回合临时状态。
返回手牌后的牌，之后重新打出时，必须重新支付其打出费用。
```

世界观解释：

```text
“返回手牌”类效果的内部主题名可以叫“权限驱逐”。
卡面不写“权限驱逐”。
卡面不写“吹回”。
卡面只写“将……返回……手牌”。
```

### 7. 消灭

```text
将1个场上的可部署实体移入其控制者或拥有者的弃牌区。
```

可消灭的对象：

```text
对手场上游侠。
对手场上装备。
```

当前不保留：

```text
消灭1个己方场上游侠。
消灭1个己方场上装备。
```

常见变体：

```text
消灭1个对手场上游侠。
消灭1个对手场上装备。
```

说明：

```text
“消灭”替代旧词“处理”。
“处理”不作为标准版卡面动作使用。
```

## 七、按作用对象展开

### 1. 手牌牌库

保留动作：

```text
抽牌。
查看牌库顶。
调整牌库顶。
返回牌库。
```

备选动作：

```text
重洗牌库。
```

### 2. 对手手牌

保留动作：

```text
查看手牌。
返回牌库。
```

不保留动作：

```text
弃牌。
公开手牌。
```

### 3. 场上游侠

保留动作：

```text
返回手牌。
返回牌库。
消灭。
```

不保留动作：

```text
攻击变化。
生命变化。
治疗。
伤害。
己方消灭。
己方返回牌库。
```

### 4. 场上装备

保留动作：

```text
返回手牌。
返回牌库。
消灭。
```

不保留动作：

```text
攻击变化。
生命变化。
治疗。
伤害。
己方消灭。
己方返回牌库。
```

### 5. 节点

标准版游侠与物品卡不直接操作节点状态。

```text
上线、离线、保护、移动、临时强化、从弃牌区回收属于节点动作池，不属于标准版手牌基础动作池。
```

---

## 八、卡面写法原则

```text
卡面只写最终技能，不写基础动作拆解。
技能句必须明确目标、数量、区域、动作。
不使用“处理”“禁用”“隔离”等非标准动作词。
节点技能费用由节点卡面图标与数字表示，不写入游侠与物品卡技能句。
```
