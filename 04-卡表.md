## 二、权限节点在线 / 离线状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
```

在线权限节点：

```text
可以被接管。
可以在已控制时发动节点技能。
可以计入 Root Access 胜利条件。
```

离线权限节点：

```text
可以保留控制归属，但不能发动节点技能。
不计入 Root Access 胜利条件。
不能用于满足“控制同一系统域3座在线权限节点”。
```

胜利句：

```text
控制同一系统域的3座在线权限节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Access Nodes in one System Domain to gain Root Access and win.
```

## 三、权限电荷与费用口径

正式资源名：

```text
权限电荷 / Access Charge
```

卡面短称：

```text
电荷
```

当前测试资源规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
```

当前费用口径：

```text
打出游侠：费用等于当前星级。
打出越权模块：费用按模块强度单独标注。
发动权限节点技能：默认花费1电荷，且节点必须在线。
```


卡面避免使用：

```text

你的牌库
你的手牌牌库
```

## 六、权限节点技能结构

每个系统域固定 3 座权限节点。

每个系统域内，3 座权限节点分别承担：

```text
侦察类
进攻类
防守类
```

分类说明：

```text
侦察类：查看、筛选、预判手牌或信息。
进攻类：强化接管、推进争夺、提升进攻游侠星级。
防守类：保护己方节点、恢复游侠、阻止或削弱敌方接管。
```
