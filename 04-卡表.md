## 三、节点在线 / 离线 / 保护状态

```text
正面朝上：在线状态。
背面朝上：离线状态。
竖置：保护状态。
```

在线节点：

```text
正面信息公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
可以使用节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开。
当前控制者公开。
该节点上的部署实体公开。
可以部署实体。
可以争夺和被接管。
不能使用节点技能。
不计入 Root Access 胜利条件。
```

保护状态：

```text
被保护节点竖置。
保护持续到保护者下回合开始。
被保护节点不能被接管。
被保护节点不能被技能变为离线。
```

胜利句：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

## 五、权限电荷与费用口径

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试资源规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
```

当前费用口径：

```text
打出游侠：费用等于当前星级。
打出物品卡：费用按物品强度单独标注。
发动节点技能：默认花费1电荷，且节点必须在线。
```

## 六、节点技能原则

```text
每个节点技能都不一样。
每个节点技能只做一件事。
节点技能要简单、易懂。
节点技能必须符合节点世界观。
节点技能不写连锁效果。
节点技能不写条件限制。
节点技能卡面不使用“若 / 当 / 如果 / 然后再”结构。
节点技能不得依赖喊名、答应、语音触发或真人反应。
```

保护关键词：

