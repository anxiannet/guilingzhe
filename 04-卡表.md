## 三、牌库模式

### 4人共用牌库模式

```text
玩家共用1个手牌牌库。
玩家共用1个节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢查看节点牌库顶1张。
```

### 双人对战模式

```text
双方各自使用1个手牌牌库。
双人对战模式不使用节点牌库。
蜂鸟无人机查看任意牌库顶1张。
蜂鸟巢若在双人对战规则中启用，需要另行定义；当前默认按4人模式节点牌库口径记录。
```

非节点卡卡面原则：

```text
只有节点卡可以在卡面直接提及节点牌库。
非节点卡不得在卡面直接提及节点牌库。
```

## 四、G-006 算法解释

G-006 蜂鸟无人机卡面写作：

```text
查看任意牌库顶1张。
```

算法口径：

```text
输入：game_mode、actor_player_id、target_deck_id。
规则引擎根据 game_mode 生成 available_deck_ids。
target_deck_id 必须属于 available_deck_ids。
效果：查看 target_deck_id 顶1张牌。
```

双人对战模式：

```text
available_deck_ids = [actor_hand_deck, opponent_hand_deck]
```

## 八、权限电荷与费用口径

```text
权限电荷 / Access Charge
卡面短称：电荷
```

当前测试资源规则：

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷。
本回合电荷 = 当前回合数，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用的权限电荷全部清空。
```

当前费用口径：

```text
打出游侠：费用等于当前星级。
打出物品卡：费用按物品强度单独标注。
发动节点技能：默认花费1电荷，且节点必须在线。
```

12座节点按4组设计，每组3个节点技能。

```text

只有已控制且在线的节点可以发动节点技能。
离线节点不能发动节点技能。
12座节点的技能必须各不相同。
```
