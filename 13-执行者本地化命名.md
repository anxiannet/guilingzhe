## 一、命名原则

```text
游侠名称不参考英文名称。
游侠名称不参考星级。
游侠名称不绑定装备等级。
游侠名称优先根据技能气质重新命名。
```

关键修正：

```text
技能是命名依据，但卡名不是技能说明。
卡名要像代号，不要像规则动作。
代号不必全是动物。
```

命名锚点：

```text
鹰眼
大兵
扳手
黄毛
毒牙
独眼
```

这些名字代表当前方向：

```text
短。
有识别度。
有街头感。
有行动气质。
像小队内部外号。
玩家能直接喊出来。
不是英文直译。
不是星级称号。
不是装备等级称号。
不是技能说明书。
```

命名流程：

```text
1. 先看技能。
2. 提炼这个技能的行为气质。
3. 再给出一个像人物外号、装备绰号或行动代号的中文名。
```

错误方向：

```text
探点客
倒手客
洗库客
送点客
```

这些虽然能看出技能，但太像功能说明，不够像游侠代号。

错误方向二：

```text
所有卡都用动物代号。
```

动物只是一类可用锚点，不是统一格式。

## 二、代号锚点库

代号可以来自以下方向：

```text
动物感：鹰眼、猎豹、白狐、毒牙。
人物外号：大兵、黄毛、独眼、老炮。
工具 / 装备：扳手、铁钩、喇叭、算盘。
街头绰号：扒手、倒爷、清道夫、中间人。
行动气质：回魂、续火、逆鳞、沉石。
```

选择原则：

```text
能对应技能气质。
能被玩家记住。
能像桌上喊人的代号。
不要像规则说明。
不要为了统一形式牺牲自然感。
```

## 四、命名检查

通过：

```text
1. 新名称不依赖英文名。
2. 新名称不依赖星级。
3. 新名称不依赖当前装备等级。
4. 新名称由技能气质推导。
5. 新名称更像游侠代号，而不是功能说明。
6. 新名称不强行统一成动物体系。
```

避免：

```text
不要用“底层、核心、高危、Root级”做游侠名称。
不要用“重装、堡垒、密钥官、贵族”等固定身份做游侠名称。
不要为了英文名对应而硬译。
不要因为当前星级高低而命名。
不要把技能直接写成卡名。
不要所有卡都动物化。
```
