## 二、视觉情绪

不是传统赛博朋克的霓虹混乱，而是“冷静、干净、残酷的系统秩序”。

高层区域：白、银、蓝，秩序、洁净、权限充足。  
底层区域：黑、灰、红色拒绝提示，临时线路、废弃服务器、手工改装装备。

核心对比：

```text
系统很干净
人被排除在外
越权装备有粗粝生命力
Root Access 的光是冷白与青蓝
卡牌本体必须有收藏卡的厚重质感
权限节点卡像真实存在的高科技二维封存存储卡
```

当前修正方向：

```text
不要大面积纯白留白
不要过度刮痕和脏纹理
不要轻飘UI说明卡
需要深色金属框、净面材质、半透明深色技能区、清晰文字
权限节点卡必须有系统域标签，但只能短小、嵌入、徽章化
权限节点技能只写具体效果，不重复固定前缀
权限节点卡号必须表现为世界观内真实封存卡号
```

## 三、卡牌框架方向

### 执行者卡

```text
尺寸：63 × 88 mm 竖版
主体：人物占画面70%以上
卡面文字：卡名、技能描述、底边附近编号、底边附近Logo
不放职业标签长文
不放类型说明长文
不放右上角品牌图标
品牌：仅在底边附近保留一个“归零者 / ZERO ACCESS”Logo
```

### 越权模块卡

```text
尺寸：63 × 88 mm 竖版
主体：装备占画面60%以上
背景：系统HUD、警报框、数据流
卡面文字：卡名、技能描述、底边附近编号、底边附近Logo
不放职业标签长文
不放类型说明长文
不放右上角品牌图标
品牌：仅在底边附近保留一个“归零者 / ZERO ACCESS”Logo
```

### 权限节点卡

```text
尺寸：88 × 63 mm 横版
主体：城市系统设施 / 服务器 / 设备节点
12 张权限节点共用同一套大外框结构
卡面文字：卡名、系统域标签、技能描述、封存卡号 / 节点位址、底边附近Logo
系统域标签：必须出现，如“无人机域 / 门禁域 / 医疗域 / 信用域”
系统域标签位置：卡名附近或左上小徽章区
系统域标签形式：短词 + 小型系统域徽章
技能描述：只写具体效果，不重复“控制此节点：”固定前缀
封存卡号 / 节点位址：如 N-007 / DN-D1
不放类型说明长文
不放系统域长说明
不放右上角品牌图标
品牌：仅在底边附近保留一个“归零者 / ZERO ACCESS”Logo
```

视觉重点：

```text
节点必须像可被接管的城市基础设施。
不能画成普通战场据点。
玩家先通过统一横版外框识别“权限节点封存卡”。
再通过系统域标签、域代号、域色光条、域图标、插画主题和节点位址识别所属系统域。
这些识别内容不是辅助 UI，而是存储卡本身的正式信息层。
```

## 四、卡面文案极简原则

### 执行者 / 越权模块

```text
1. 卡名
2. 技能描述
3. 底边附近编号
4. 底边附近Logo
```

### 权限节点

```text
1. 卡名
2. 系统域标签
3. 节点技能描述
4. 封存卡号 / 节点位址
5. 底边附近Logo
```

权限节点示例：

```text
蜂鸟巢
无人机域

查看节点牌库顶1张，可置顶或置底。

N-007 / DN-D1
归零者 / ZERO ACCESS
```

说明：

```text
“控制此节点”作为权限节点技能规则前提写入规则说明，不在每张权限节点卡面重复。
N-007 / DN-D1 是世界观内真实封存卡号与节点位址。
无人机域是世界观内正式系统域标签。
版本号进入资产文件名和样卡审核记录，不作为正式卡面世界观信息。
```

不再在卡面上保留：

```text
类型说明长文
职业标签长文
系统域说明长文
防御型 / 执行者 / 越权模块等重复说明
连续图标技能链
右上角归零者品牌图标
过多英文装饰句
说明性副标题
权限节点固定前缀“控制此节点：”
制作版本号作为正式信息
```

## 五、系统域识别标准

系统域识别属于世界观内正式信息。

首版4个系统域的识别信息如下：

| 系统域 | 中文域标签 | 英文域标签 | 域代号 | 建议图形 | 识别色 |
|---|---|---|---|---|---|
| 门禁域 | 门禁域 | ACCESS DOMAIN | AD | 门锁 / 闸机 / 门形芯片 | 冷白 + 蓝 |
| 医疗域 | 医疗域 | MEDICAL DOMAIN | MD | 十字 / 医疗舱 / 生命曲线 | 白 + 青绿 |
| 无人机域 | 无人机域 | DRONE DOMAIN | DN | 蜂鸟无人机 / 飞行器 | 深蓝 + 青蓝 |
| 信用域 | 信用域 | CREDIT DOMAIN | CD | 芯片钱包 / 账本 / 黑金芯片 | 黑金 + 紫 |

系统域识别要求：

```text
1. 必须在权限节点卡上出现。
2. 位置不能抢卡名。
3. 不能占用技能区主体空间。
4. 不能写成长句解释。
5. 优先做成嵌入式小徽章或金属压印。
6. 系统域颜色只做局部光条或边缘识别，不铺满整张卡。
7. 域代号必须与节点位址一致，如 DN-D1 必须对应无人机域。
```

## 六、封存卡号与版本

权限节点卡号：

```text
全局存储序列 / 系统域节点位址
```

蜂鸟巢示例：

```text
N-007 / DN-D1
```

世界观含义：

```text
N-007 = 首版节点封存卡体系中的第 7 号存储卡。
DN-D1 = 无人机域第 1 号节点位址。
```

版本记录：

```text
CN-v1.0、EN-v1.0、FRAME-v1.0、ILLUS-v1.0 等版本号进入资产文件名和审核记录。
正式卡面主信息不显示版本号。
测试样卡可在极小位置临时标注版本号，用于审核。
```

## 七、技能区与语言版式

正式卡面改为：

```text
中文主版：技能区留够 1 行中文规则。
英文版：使用同一套边框，技能区留够 2 行英文规则。
```

技能区原则：

```text
1. 不做大图标解释链。
2. 不在技能区堆多个说明图标。
3. 规则文字必须独立可读。
4. 中文规则尽量压缩为1行。
5. 英文规则允许2行。
6. 技能区不放类型图标链。
7. 权限节点技能不重复固定前缀。
```

示例：

```text
中文版：查看节点牌库顶1张，可置顶或置底。
英文版：Look at the top card of the Node Deck. Put it on top or bottom.
```

## 八、颜色逻辑

| 状态 | 颜色方向 | 用途 |
|---|---|---|
| 零权限 | 红色 / 暗红 / 黑灰 | Access Denied、被拒绝、系统警报 |
| 越权中 | 橙黄 / 警告黄 | Override、破解过程、风险状态 |
| 已授权 | 青蓝 / 冷白 | Access Granted、节点接管 |
| Root Access | 白金 / 青蓝高亮 | 胜利、最高权限、系统接管 |

---

## 九、系统域视觉区分

| 系统域 | 视觉元素 | 建议主色 |
|---|---|---|
| 门禁域 | 门、闸机、电梯、封锁线 | 冷白 + 蓝 |
| 医疗域 | 医疗舱、纳米液、生命曲线 | 白 + 青绿 |
| 无人机域 | 蜂鸟无人机、空域网格、机群 | 深蓝 + 青蓝 |
| 信用域 | 芯片钱包、账本、债务库、铸机 | 黑金 + 紫 |

## 十、禁止方向

```text
不要做普通赛博朋克霓虹街头
不要做现实黑客键盘敲代码感
不要做普通雇佣兵枪战
不要做现实政治或现实国家符号
不要让角色只像战术士兵，必须体现权限夺取主题
不要大面积纯白留白
不要过度刮痕、污渍和脏旧纹理
不要把卡面做成教学说明页
不要让图标链抢占技能区
不要在卡面堆职业、类型、系统域等说明文字
不要在右上角重复放归零者品牌图标
不要省略权限节点卡的系统域标签
不要在权限节点技能前重复写“控制此节点：”
不要给 4 个系统域做 4 套完全不同的大外框
不要把封存卡号做成临时制作备注感
不要把系统域标签做成旁白说明感
```
