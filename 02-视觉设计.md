## 二、视觉情绪

不是传统赛博朋克的霓虹混乱，而是“冷静、干净、残酷的系统秩序”。

高层区域：白、银、蓝，秩序、洁净、权限充足。  
底层区域：黑、灰、红色拒绝提示，临时线路、废弃服务器、手工改装装备。

核心对比：

```text
系统很干净
人被排除在外
物品装备有粗粝生命力
Root Access 的光是冷白与青蓝
卡牌本体必须有收藏卡的厚重质感
节点卡像真实存在的高科技二维封存存储卡
```

当前修正方向：

```text
不要大面积纯白留白
不要过度刮痕和脏纹理
不要轻飘UI说明卡
需要深色金属框、净面材质、半透明深色技能区、清晰文字
节点卡必须有系统域标签，但只能短小、嵌入、徽章化
节点技能只写具体效果，不重复固定前缀
节点卡号必须表现为世界观内真实封存卡号
```

## 三、卡牌框架方向

### 游侠卡

```text
尺寸：63 × 88 mm 竖版
主体：人物占画面70%以上
卡面文字：卡名、技能描述、底边附近编号、底边附近Logo
不放职业标签长文
不放类型说明长文
不放右上角品牌图标
品牌：仅在底边附近保留一个“归零者 / ZERO ACCESS”Logo
```

### 物品卡

```text
尺寸：63 × 88 mm 竖版
主体：装备占画面60%以上
背景：系统HUD、警报框、数据流
卡面文字：卡名、技能描述、底边附近编号、底边附近Logo
不放职业标签长文
不放类型说明长文
不放右上角品牌图标
品牌：仅在底边附近保留一个“归零者 / ZERO ACCESS”Logo
```

### 节点卡

```text
尺寸：88 × 63 mm 横版
主体：城市系统设施 / 服务器 / 设备节点 / 机构地标
12 张节点共用同一套大外框结构
卡面文字：卡名、系统域标签、技能描述、封存卡号 / 节点位址、底边附近Logo
系统域标签：必须出现，如“行政域 / 安防域 / 生化域 / 科研域”
系统域标签位置：卡名附近或左上小徽章区
系统域标签形式：短词 + 小型系统域徽章
技能描述：只写具体效果，不重复“控制此节点：”固定前缀
封存卡号 / 节点位址：如 N-007 / BIO-B1
不放类型说明长文
不放系统域长说明
不放右上角品牌图标
品牌：仅在底边附近保留一个“归零者 / ZERO ACCESS”Logo
```

视觉重点：

```text
节点必须像可被接管的城市基础设施、系统机构或权限地标。
不能画成普通战场据点。
玩家先通过统一横版外框识别“节点封存卡”。
再通过系统域标签、域代号、域色光条、域图标、插画主题和节点位址识别所属系统域。
这些识别内容不是辅助 UI，而是存储卡本身的正式信息层。
```

## 四、卡面文案极简原则

### 游侠 / 物品卡

```text
1. 卡名
2. 技能描述
3. 底边附近编号
4. 底边附近Logo
```

### 节点

```text
1. 卡名
2. 系统域标签
3. 节点技能描述
4. 封存卡号 / 节点位址
5. 底边附近Logo
```

节点示例：

```text
蜂鸟巢
生化域

选择1个己方游侠，本回合攻击力+1。

N-007 / BIO-B1
归零者 / ZERO ACCESS
```

说明：

```text
“控制此节点”作为节点技能规则前提写入规则说明，不在每张节点卡面重复。
N-007 / BIO-B1 是世界观内真实封存卡号与节点位址。
生化域是世界观内正式系统域标签。
版本号进入资产文件名和样卡审核记录，不作为正式卡面世界观信息。
```

不再在卡面上保留：

```text
类型说明长文
职业标签长文
系统域说明长文
防御型 / 游侠 / 物品卡等重复说明
连续图标技能链
右上角归零者品牌图标
过多英文装饰句
说明性副标题
节点固定前缀“控制此节点：”
制作版本号作为正式信息
```

## 五、系统域识别标准

系统域识别属于世界观内正式信息。

首版4个系统域的识别信息如下：

| 系统域 | 中文域标签 | 英文域标签 | 域代号 | 建议图形 | 识别色 |
|---|---|---|---|---|---|
| 行政域 | 行政域 | ADMINISTRATIVE DOMAIN | AD | 档案柜 / 印章 / 节点索引芯片 | 冷白 + 行政蓝 |
| 安防域 | 安防域 | SECURITY DOMAIN | DF | 盾牌 / 防火墙 / 安检门 / 锁形线路 | 深蓝 + 警戒红 |
| 生化域 | 生化域 | BIOTECH DOMAIN | BIO | 蜂鸟 / 纳米粒子 / 生命曲线 / 复苏舱 | 青绿 + 冷白 |
| 科研域 | 科研域 | RESEARCH DOMAIN | RD | 试管 / 机械臂 / 研究所标识 / 回收环 | 紫蓝 + 工业橙 |

系统域识别要求：

```text
1. 必须在节点卡上出现。
2. 位置不能抢卡名。
3. 不能占用技能区主体空间。
4. 不能写成长句解释。
5. 优先做成嵌入式小徽章或金属压印。
6. 系统域颜色只做局部光条或边缘识别，不铺满整张卡。
7. 域代号必须与节点位址一致，如 BIO-B1 必须对应生化域。
```

## 六、封存卡号与版本

节点卡号：

```text
全局存储序列 / 系统域节点位址
```

蜂鸟巢示例：

```text
N-007 / BIO-B1
```

世界观含义：

```text
N-007 = 首版节点封存卡体系中的第 7 号存储卡。
BIO-B1 = 生化域第 1 号节点位址。
```

版本记录：

```text
CN-v1.0、EN-v1.0、FRAME-v1.0、ILLUS-v1.0 等版本号进入资产文件名和审核记录。
正式卡面主信息不显示版本号。
测试样卡可在极小位置临时标注版本号，用于审核。
```

## 七、技能区与语言版式

正式卡面改为：

```text
中文版：技能区留够 1 行中文规则。
英文版：使用同一套边框，技能区留够 2 行英文规则。
```

技能区原则：

```text
1. 不做大图标解释链。
2. 不在技能区堆多个说明图标。
3. 规则文字必须独立可读。
4. 中文规则尽量压缩为1行。
5. 英文规则允许2行。
6. 技能区不放类型图标链。
7. 节点技能不重复固定前缀。
```

示例：

```text
中文版：选择1个己方游侠，本回合攻击力+1。
英文版：Choose 1 friendly Operative. It gets +1 ATK this turn.
```
