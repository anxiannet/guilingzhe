## 二、成品方向

- 执行者 / 越权模块卡：63 × 88 mm 竖版。
- 权限节点卡：88 × 63 mm 横版。
- 卡牌不是普通印刷插画，而是二维权限载体。
- 卡内应表现执行者、越权模块或权限节点被压缩封存后的状态。
- 调用卡牌的感觉应接近“释放被封存的战术实体”。
- 权限节点共用同一套大外框结构。
- 系统域通过正式域标签、域代号、域色光条、域图标、插画主题和节点位址区分。

## 三、卡面文案原则

### 执行者 / 越权模块

正式样卡卡面文字只保留：

```text
1. 卡名
2. 技能描述
3. 底边附近编号
4. 底边附近Logo
```

### 权限节点

权限节点卡必须保留五类世界观内正式信息：

```text
1. 卡名
2. 系统域标签
3. 节点技能描述
4. 封存卡号 / 节点位址
5. 底边附近Logo
```

权限节点技能卡面原则：

```text
卡面只写具体效果。
“控制此节点”作为规则前提，不在每张权限节点卡上重复。
```

封存卡号原则：

```text
卡面编号使用：全局存储序列 / 系统域节点位址。
示例：N-007 / DN-D1。
N-007 / DN-D1 是世界观内真实封存卡号与节点位址，不是后台制作编号。
版本号进入资产文件名和审核记录，不作为世界观内正式信息。
```

中文版样卡 v1 文案：

```text
蜂鸟巢
无人机域

查看节点牌库顶1张，可置顶或置底。

N-007 / DN-D1
归零者 / ZERO ACCESS
```

## 六、成品检查重点

```text
1. 是否能看出卡牌本身是高科技封存载体。
2. 是否避免普通插画卡、普通手游卡、普通说明卡感觉。
3. 权限节点卡是否有系统域标签。
4. 是否避免大面积纯白信息区。
5. 是否避免过度刮痕、污渍和脏旧纹理。
6. 是否让执行者、越权模块、权限节点三类卡一眼区分。
7. 中英文版本是否能共用同一套边框与技能区尺寸。
8. AI生成文字是否已在人工排版阶段重排。
9. 资产是否已经分层：卡框、插图、文字、导出图。
10. 是否显示封存卡号 / 节点位址。
11. 是否通过系统域标签、域代号、域色光条、域图标、插画主题识别系统域。
12. 封存卡号是否像世界观内真实存储序列。
```
