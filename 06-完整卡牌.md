## 二、成品方向

- 游侠 / 装备 / 一次性物品卡：63 × 88 mm 竖版。
- 节点卡：88 × 63 mm 横版。
- 卡牌不是普通印刷插画，而是二维权限载体。
- 卡内应表现游侠、装备、一次性物品或节点被压缩封存后的状态。
- 调用卡牌的感觉应接近“释放被封存的战术实体”。
- 节点共用同一套大外框结构。
- 系统域通过正式域标签、域代号、域色光条、域图标、插画主题和节点位址区分。

## 三、卡面信息原则

### 游侠

正式样卡必须保留：

```text
1. 卡名
2. 费用
3. ATK
4. HP
5. 技能描述
6. 底边附近编号
7. 底边附近Logo
```

### 可部署装备

正式样卡必须保留：

```text
1. 卡名
2. 费用
3. ATK
4. HP
5. 技能、功能或规则限制文本
6. 底边附近编号
7. 底边附近Logo
```

### 一次性物品

正式样卡必须保留：

```text
1. 卡名
2. 费用
3. 技能描述
4. 底边附近编号
5. 底边附近Logo
```

一次性物品没有ATK与HP，卡面不显示空占位，也不写“不适用”。

### 节点

节点卡必须保留：

```text
1. 卡名
2. 系统域标签
3. 节点技能费用
4. 节点技能描述
5. 唯一卡号
6. 底边附近Logo
```

节点技能卡面原则：

```text
卡面只写具体效果。
“控制此节点”作为规则前提，不在每张节点卡上重复。
```

三维原则：

```text
费用、ATK、HP统称卡牌三维。
游侠与可部署装备必须显示完整三维。
三维数字必须独立可读，不能只靠颜色或星级表达。
三维使用固定位置，不因插画变化而移动。
一次性物品只显示费用。
节点只显示节点技能费用。
```

卡号原则：

```text
卡号使用：全局存储序列 / 系统域节点位址。
示例：N-007 / DN-D1。
N-007 / DN-D1 是世界观内真实封存卡号与节点位址，不是后台制作编号。
版本记录归属于 N-007 / DN-D1，但不拼进卡号。
```

游侠中文版卡面示例：

```text
鹰眼
费用 5｜ATK 3｜HP 2

消灭1个其他玩家的4费或以下游侠。

R-009
归零者 / ZERO ACCESS
```

一次性物品中文版卡面示例：

```text
正版储存卡
费用 3

选择场上1个游侠或装备，将其返回其控制者手牌。

G-E03
归零者 / ZERO ACCESS
```

节点中文版样卡 v1 文案：

```text
蜂鸟巢
无人机域
节点技能费用 1

查看节点牌库顶1张，可置顶或置底。

N-007 / DN-D1
归零者 / ZERO ACCESS
```

## 六、成品检查重点

```text
1. 是否能看出卡牌本身是高科技封存载体。
2. 是否避免普通插画卡、普通手游卡、普通说明卡感觉。
3. 节点卡是否有系统域标签与节点技能费用。
4. 是否避免大面积纯白信息区。
5. 是否避免过度刮痕、污渍和脏旧纹理。
6. 是否让游侠、装备、一次性物品、节点四类卡一眼区分。
7. 中英文版本是否能共用同一套边框与技能区尺寸。
8. AI生成文字是否已在人工排版阶段重排。
9. 资产是否已经分层：卡框、插图、文字、导出图。
10. 是否显示正确卡号。
11. 是否通过系统域标签、域代号、域色光条、域图标、插画主题识别系统域。
12. 卡号是否像世界观内真实存储序列。
13. 是否避免把版本号拼进卡号。
14. 游侠与可部署装备是否完整显示费用、ATK、HP三维。
15. 三维是否清晰可读且位置固定。
16. 一次性物品是否只显示费用，没有无意义ATK/HP占位。
```
