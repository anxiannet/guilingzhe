## 二、成品方向

- 执行者 / 越权模块卡：63 × 88 mm 竖版。
- 权限节点卡：88 × 63 mm 横版。
- 卡牌不是普通印刷插画，而是二维权限载体。
- 卡内应表现执行者、越权模块或权限节点被压缩封存后的状态。
- 调用卡牌的感觉应接近“释放被封存的战术实体”。

## 三、卡面文案原则

### 执行者 / 越权模块

正式样卡卡面文字只保留：

```text
1. 卡名
2. 技能描述
3. 底边附近编号
4. 底边附近Logo
```

### 权限节点

权限节点卡必须保留区域信息。

```text
1. 卡名
2. 区域信息 / 系统域信息
3. 节点技能描述
4. 节点编号
5. 底边附近Logo
```

示例：

```text
蜂鸟巢
无人机域

控制此节点：查看节点牌库顶1张，可置顶或置底。

D1 / DN-D1
归零者 / ZERO ACCESS
```

## 五、蜂鸟巢强基线精修点

```text
1. 增加区域信息：无人机域。
2. 区域信息应短小、嵌入式、徽章化，不要变成大标签。
3. 右上 D1 保留，但应更像权限芯片 / 金属压印编号 / 封存档案编号。
4. 左上蜂鸟徽章保留，但应更像嵌入式无人机域徽章 / 半透明封装芯片。
5. 透明封存层可再薄一点，像透明数据膜与二维压缩膜。
6. 技能区可再压缩 5%–10%，把更多画面留给封存主体。
7. 右下 Logo 可略缩 5%，避免压过卡名。
8. AI生成文字不能直接进入印刷终稿，后续必须人工重排。
```

## 六、成品检查重点

```text
1. 是否能看出卡牌本身是高科技封存载体。
2. 是否避免普通插画卡、普通手游卡、普通说明卡感觉。
3. 权限节点卡是否有区域信息。
4. 是否避免大面积纯白信息区。
5. 是否避免过度刮痕、污渍和脏旧纹理。
6. 是否让执行者、越权模块、权限节点三类卡一眼区分。
7. 中英文版本是否能共用同一套边框与技能区尺寸。
8. AI生成文字是否已在人工排版阶段重排。
```
