# 归零者｜样卡审核记录

> 文件定位：本文件用于记录 `归零者 / ZERO ACCESS` 第一批样卡的阶段性审核结论。
>
> 本文件只记录皮肤主题探索过程，不替代正式规则、正式卡表、正式技能文案或正式视觉规范。

---

## 一、主题命名当前口径

当前样卡视觉中，中文产品名采用：

```text
归零者
```

英文产品名保留：

```text
ZERO ACCESS
```

说明：

```text
“零权限”更像世界观概念与起点状态。
“归零者”更像产品名、阵营名与包装标题。
```

推荐当前口径：

| 用途 | 中文 | 英文 |
|---|---|---|
| 产品名 / 皮肤名 | 归零者 | ZERO ACCESS |
| 世界观状态 | 零权限 | Zero Access |
| 玩家小队 | 归零者小队 / 零权限小队 | Zero Crew |
| 最高目标 | 根权限 | Root Access |

---

## 二、当前卡面母设定

当前视觉方向已从“普通科幻插画卡”升级为“二维封存体卡牌”。

核心定义：

```text
《归零者》中的卡牌，不是普通印刷卡，
而是将执行者、越权模块与权限节点压缩封存后的二维权限载体。
调用一张卡，不是“打出一张牌”，
而是“释放一个被封存的战术实体”。
```

卡面应表现为：

```text
高科技封存载体
卡内压缩空间
透明封存层
空间折叠边缘
实体从二维介质中展开 / 释放
```

---

## 三、当前卡面文案原则

正式样卡卡面文字只保留四类信息：

```text
1. 卡名
2. 技能描述
3. 底边附近编号
4. 底边附近Logo
```

不再保留：

```text
职业标签长文
类型说明长文
系统域说明长文
连续图标技能链
右上角归零者品牌图标
过多英文装饰句
说明性副标题
```

图标仍可作为“释义徽章 / 识别徽章”存在，但不能回到教学图标链。

## 四、样卡：蜂鸟巢 / Hummingbird Hive

### 当前状态

```text
样卡类型：权限节点 / 横版地盘牌
当前最佳版本：蜂鸟巢-权限节点卡-封存体新版-v5-强基线
审核状态：强基线通过，后续只做精修，不再推翻风格
本地参考路径：用户上传截图 /mnt/data/23DB299B-2308-44C0-A8D7-870182F9050D.jpeg
```

### 通过点

```text
1. 卡面信息控制很好，只保留卡名、技能、DN-D1、Logo、D1进度。
2. 没有多余“无人机域”“节点编号”“ROOT ACCESS长句”等说明文字。
3. 卡框质感明显提升，像高科技实体卡，不像轻UI界面。
4. 插画区域干净，蜂鸟巢主体明确。
5. 透明封存层有效，有“三维节点被压缩进卡里”的感觉。
6. 无人机从封存层边缘穿出，增强了释放感。
7. 底部技能区简洁，规则可读。
8. DN-D1、D1-D2-D3、归零者 Logo 的位置合理。
```

### 后续只做精修

```text
1. 右上 D1 保留，但应更像权限芯片 / 金属压印编号 / 封存档案编号，不要像可点击按钮。
2. 左上蜂鸟徽章保留，但应更像嵌入式无人机域徽章 / 半透明封装芯片。
3. 透明封存层可再薄一点，像透明数据膜与二维压缩膜，少一点厚玻璃展示柜感。
4. 技能区可再压缩 5%–10%，把更多画面留给封存主体。
5. 右下 Logo 可略缩 5%，避免压过卡名。
6. AI生成文字不能直接进入印刷终稿，后续必须人工重排。
```

### 当前禁止

```text
不要再回到大白底说明卡。
不要再增加“无人机域 / 节点编号 / ROOT ACCESS长句”等说明文字。
不要再把卡框做成轻UI或手游界面。
不要再把封存层做成普通玻璃罩。
```

### 建议留档名

```text
蜂鸟巢-权限节点卡-封存体新版-v5-强基线
```

## 五、样卡：跑线侦察员 / Trace Runner

### 当前状态

```text
样卡类型：1星执行者 / 竖版角色牌
当前最佳版本：跑线侦察员执行者卡 v2
审核状态：早期方向通过，但需按新“二维封存体”母设定重做
```

### 通过点

```text
1. 竖版执行者卡形态成立。
2. 角色主体占比足够，符合执行者卡方向。
3. 明亮未来科技风延续了归零者视觉基调。
4. 角色有奔跑、侦察、越权接入的动作感。
```

### 后续重做方向

```text
1. 降低角色高级感：装甲减少，装备更轻便，衣服更像拼装外套。
2. 增加临时线路包、便携终端、廉价护目镜、底层工具感。
3. 强化人物被二维封存 / 从卡内释放的视觉。
4. 卡面文案只保留：卡名、技能描述、底边编号、底边Logo。
5. 不再使用连续图标技能链。
```

### 建议留档名

```text
跑线侦察员-执行者卡-v2-早期方向参考
```


### 仍需微调

```text

3. 左右“使用 / 保护”图标应改成小型无字权限徽章。
4. 减少边框碎细节约20%。
5. 保留中间主体：地堡正在从二维压缩层中展开。
6. AI生成文字不能直接进入印刷终稿，后续必须人工重排。
```

### 建议留档名
