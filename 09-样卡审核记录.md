## 一、主题命名当前口径

当前项目中文产品名采用：

```text
归零者
```

英文产品名保留：

```text
ZERO ACCESS
```

说明：

```text
“零权限”更像世界观概念与起点状态。
“归零者”更像产品名、阵营名与包装标题。
```

推荐当前口径：

| 用途 | 中文 | 英文 |
|---|---|---|
| 产品名 / 项目名 | 归零者 | ZERO ACCESS |
| 世界观状态 | 零权限 | Zero Access |
| 玩家小队 | 归零者小队 / 零权限小队 | Zero Crew |
| 最高目标 | 根权限 | Root Access |

## 二、当前卡面母设定

当前视觉方向已从“普通科幻插画卡”升级为“二维封存体卡牌”。

核心定义：

```text
《归零者》中的卡牌，不是普通印刷卡，
而是将执行者、越权模块与权限节点压缩封存后的二维权限载体。
调用一张卡，不是“打出一张牌”，
而是“释放一个被封存的战术实体”。
```

卡面应表现为：

```text
高科技封存载体
卡内压缩空间
透明封存层
空间折叠边缘
实体从二维介质中展开 / 释放
```

---

## 三、当前卡面文案原则

执行者 / 越权模块卡面文字只保留四类信息：

```text
1. 卡名
2. 技能描述
3. 底边附近编号
4. 底边附近Logo
```

权限节点卡面文字保留五类信息：

```text
1. 卡名
2. 区域信息 / 系统域信息
3. 节点技能描述
4. 节点编号
5. 底边附近Logo
```

权限节点技能文案原则：

```text
卡面只写具体效果。
“控制此节点”作为规则前提，不在每张权限节点卡上重复。
```

不再保留：

```text
职业标签长文
类型说明长文
系统域说明长文
连续图标技能链
右上角归零者品牌图标
过多英文装饰句
说明性副标题
AI 生成文字终稿
```

图标仍可作为“类型 / 系统域 / 状态 / 装饰符号”存在，但不能回到教学图标链。
