# 零权限｜第一批样卡制作清单

> 文件定位：本文件用于记录 `零权限 / Zero Access` 皮肤主题第一批样卡的制作对象、测试目的、插画需求与成品检查项。
>
> 本文件不替代正式规则、正式卡表或正式视觉规范。

---

## 一、样卡目标

第一批样卡不是为了直接完成全套卡牌，而是用于验证：

```text
1. 零权限主题是否一眼成立。
2. 执行者 / 越权模块 / 权限节点三类卡是否能快速区分。
3. 图标化技能是否能在实体卡尺寸下读懂。
4. 角色、装备、节点是否有足够“想抽到 / 想占下”的吸引力。
5. 视觉是否避开普通赛博朋克、普通枪战、普通黑客键盘感。
```

---

## 二、第一批样卡选择

首批选择 5 张，覆盖核心卡牌类型与不同复杂度：

| 序号 | 原版卡牌 | 零权限卡名 | 类型 | 选择理由 |
|---:|---|---|---|---|
| 1 | 小钻风 | 跑线侦察员 / Trace Runner | 1星执行者 | 测试低星执行者卡框、基础图标技能、底层角色气质 |
| 2 | 金鼻白毛老鼠精 | 管线盗权者 / Tunnel Rat | 3星执行者 | 测试偷取类互动、角色记忆点、权限节点驻守表达 |
| 3 | 牛魔王 | 根权限破壁者 / Root Breaker | 5星执行者 | 测试顶级角色气场、Root Access 幻想、强技能排版 |
| 4 | 辟火罩 | 离线地堡 / Offline Bunker | 越权模块 | 测试装备卡、保护效果、地堡高科技幻想 |
| 5 | 蜂鸟巢 | Hummingbird Hive | 权限节点 | 测试横版节点卡、无人机域视觉、系统域区分 |

## 三、样卡 1：跑线侦察员 / Trace Runner

### 基础信息

```text
原版：小钻风
类型：1星执行者
数量：2
定位：底层执行者 / 侦察
```

### 卡面文案

```text
查权限节点牌库顶牌，可置顶或置底。
```

### 图标表达

```text
[入场] [节点牌库] [顶1] → [查看] → [置顶/置底]
```

### 插画方向

```text
底层零权限少年或少女执行者，身穿拼装轻便外套，背着小型信号包，手持临时破解终端。
背景是高塔城市底层的维修通道，墙面投射红色 Access Denied 警告。
角色正在沿着废弃线路奔跑，终端上显示一张即将翻开的权限节点地图。
```

### 视觉测试点

```text
1星卡是否足够清楚但不廉价。
角色是否有“底层但机灵”的感觉。
图标是否能表达查看节点牌库顶牌。
```

---

## 四、样卡 2：管线盗权者 / Tunnel Rat

### 基础信息

```text
原版：金鼻白毛老鼠精
类型：3星执行者
数量：2
定位：核心执行者 / 管线偷权
```

### 卡面文案

```text
选1个1星执行者，加入你的权限节点。
```

### 图标表达

```text
[入场] [1星] [执行者] ×1 → [加入节点]
```

### 插画方向

```text
身形敏捷的管线潜入者，穿着贴身机修潜行服，半蹲在城市基础设施管道中。
一只小型探针正接入敌方权限节点，蓝色数据线把一名低阶执行者的权限牵引到己方节点。
角色外观要聪明、狡黠、技术感强，不做真实盗窃教学感。
```

### 视觉测试点

```text
偷取效果是否能和普通加入手牌效果区分。
“加入你的权限节点”图标是否清楚。
3星角色是否比1星更有职业记忆点。
```

---

## 五、样卡 3：根权限破壁者 / Root Breaker

### 基础信息

```text
原版：牛魔王
类型：5星执行者
数量：1
定位：Root级执行者 / 系统夺权
```

### 卡面文案

```text
立即再打出1张执行者牌。
```

### 图标表达

```text
[入场] → [再打出] [执行者] ×1
```

### 插画方向

```text
传奇级零权限执行者，穿戴重型机修装甲与Root级接入核心，站在巨型权限服务器前。
背景中高塔城市的白色系统墙正在裂开，青蓝Root Access光从裂缝中爆发。
角色不应像普通士兵，应像“能把系统打穿的人”。
可以有机械角形轮廓或厚重肩甲，但避免直接像任何现有IP角色。
```

### 视觉测试点

```text
5星是否具有压迫感与收藏感。
Root Access 是否一眼可见。
再打出执行者的图标是否足够简单。
```

## 六、样卡 4：离线地堡 / Offline Bunker

### 基础信息

```text
原版：辟火罩
类型：越权模块
数量：1
定位：保护你的权限节点1回合
```

### 卡面文案

```text
保护你的权限节点1回合。
```

### 图标表达

```text
你的[权限节点] → [保护] 1回合
```

### 插画方向

```text
一套可折叠的离线防御模块正在权限节点周围展开，形成临时地堡。
外部系统HUD显示 Access Denied，节点内部显示 Offline Safe Mode。
地堡带有小型炮塔、断网屏障、机械支架和青蓝护盾线。
整体要像“离线后反而安全”的高科技防御设备。
```

### 视觉测试点

```text
越权模块卡是否能和执行者卡一眼区分。
保护效果是否足够直观。
地堡是否有“好装备”的吸引力。
```

## 七、样卡 5：蜂鸟巢 / Hummingbird Hive

### 基础信息

```text
原版：地盘牌映射
系统域：无人机域 Drone Domain
类型：权限节点
尺寸：88 × 63 mm 横版
```

### 节点幻想

```text
调用微型侦察无人机。
```

### 插画方向

```text
横版城市节点场景。
画面中心是一座透明蜂巢状无人机停靠站，数十架蜂鸟微型无人机从蜂巢中起飞。
背景是高塔城市低空航线与青蓝空域网格。
节点界面显示 Drone Domain / Hummingbird Hive / Access Node。
整体要有强烈“我想控制这个节点”的感觉。
```

### 视觉测试点

```text
横版节点卡是否与竖版角色 / 装备卡明显区分。
无人机域是否能和门禁域、医疗域、信用域形成视觉差异。
节点是否有被控制后的幻想。
```

## 八、第一批样卡共通检查项

```text
1. 卡名是否清楚、有记忆点。
2. 中文名与英文名是否都适合包装展示。
3. 星级 / 类型 / 图标技能是否能在63×88mm实体尺寸读清。
4. 权限系统主题是否比普通赛博朋克更明显。
5. 是否出现普通枪战、现实军警、现实国家符号等不适合方向。
6. 执行者、越权模块、权限节点三类卡是否一眼可区分。
7. 图标表达是否能减少文字压力。
8. 角色是否有“想抽到”的欲望。
9. 节点是否有“想占下”的欲望。
10. 装备是否有“好东西”的欲望。
```

## 九、后续制作顺序

```text
1. 先做蜂鸟巢，验证横版权限节点视觉。
2. 再做跑线侦察员，验证低星执行者卡框。
3. 再做离线地堡，验证越权模块卡框。
4. 再做管线盗权者，验证中高星角色记忆点。
5. 最后做根权限破壁者，验证5星收藏级气场。
```

## 十、样卡完成后的决策点

完成第一批样卡后，需要判断：

```text
1. 零权限是否值得进入完整60张皮肤生产。
2. 是否需要为零权限单独设计卡背。
3. 是否继续沿用《抢地盘》规则，还是拆成独立产品。
4. 是否将 Identity Domain / 身份域 放入第二扩展。
5. 是否将图标系统推广到其他皮肤主题。
```
