# 归零者｜背景世界观

> 主题定位：系统弃民、权限节点、Root Access 夺权。
>
> 本文件记录《归零者 / ZERO ACCESS》的世界观背景。
>
> “零权限”在本项目中是世界观起点状态，不再作为项目名使用。

---

## 一、核心背景

未来城市不再由法律、现金或武力直接统治，而是由“权限系统”统治。

门禁、交通、医疗、货币、无人机、住房、武器、身份档案，全部接入同一套城市权限网络。拥有权限的人可以进入高塔，调用医疗舱，远程控制设备，清除债务，甚至让巡逻无人机为自己开路。

而失去权限的人，连自己的名字都无法证明。

他们无法开门，无法买药，无法搭乘列车，无法进入住所，也无法通过任何身份验证。系统不会逮捕他们，因为系统已经从档案里删除了他们。

他们被称为：

```text
零权限者
Zero Access
```

“零权限”不是强者称号，而是系统给他们的最低判定。

```text
Access Level: 0
```

但在城市底层，零权限者并没有消失。他们组建小队，改装设备，偷接废弃线路，驯服蜂鸟无人机，穿上机修装甲，在高塔阴影下争夺一个个权限节点。

他们的目标不是重新申请权限。

他们要直接夺取 Root Access。

---

## 二、玩家身份

玩家扮演一支归零者小队 / 零权限小队的首领。

你带领的是一群被系统删除、被高塔拒绝、被城市服务排除在外的人。

他们不再等待系统承认自己。

他们要把系统改写成承认自己。

---

## 三、起点与目标

```text
起点：零权限 / Access Level 0
目标：根权限 / Root Access
爽点：从系统弃民，变成系统主人
```

核心反差：

```text
你从没有任何权限开始。
你的目标是夺取最高权限。
```

---

## 四、玩家为什么对抗

每个系统域同一时间只能存在一个 Root Access。

一支小队夺取 Root Access 后，其他小队会被降权、封锁、踢出节点，甚至重新归零。

因此玩家之间不是为了普通资源互相争斗，而是在争夺“谁能定义这套系统”。

```text
同一个系统域只承认一个最高权限。
你不夺权，别人就会用权限决定你的生存边界。
```

---

## 五、控制后的幻想

每个系统域都由三座关键权限节点构成。控制同一系统域的三座节点，即获得该系统的 Root Access。

### 门禁域 Access Domain

控制后，整座城市的门、电梯、封锁线和安全通道都会为你打开。敌人会被锁在门外。

### 医疗域 Medical Domain

控制后，医疗舱会优先救你的同伴。敌人的急救请求会被延迟、拒绝或降级。

### 无人机域 Drone Domain

控制后，巡逻无人机、蜂鸟侦察机和空中炮塔都会改听你的指令。

### 信用域 Credit Domain

控制后，你可以清除自己的债务，冻结敌人的账户，买下他们曾经站立的街区。

### 身份域 Identity Domain

身份域暂不进入首版，作为扩展候选。控制后，你可以恢复被删除的人，也可以封锁敌方身份权限。

控制同一系统域的三座节点，即可获得 Root Access。
