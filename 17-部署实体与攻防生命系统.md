# 归零者｜部署实体与攻防生命系统

> 文件定位：本文件定义《归零者 / ZERO ACCESS》中可部署实体、攻击力、生命值、部署到权限节点与战斗结算的基础口径。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、当前结论

《归零者》中的手牌牌库不再只包含抽象技能卡。

```text
游侠、装备、设备、武器都视为可部署实体。
每个可部署实体都有攻击力和生命值。
每个可部署实体都可以部署在权限节点上。
```

说明：

```text
越权模块不再只理解为一次性法术效果。
其中的装备、设备、武器可以作为实体部署到权限节点上。
```

---

## 二、可部署实体

正式术语：

```text
可部署实体
```

英文建议：

```text
Deployable Entity
```

包括：

```text
1. 游侠。
2. 装备。
3. 设备。
4. 武器。
```

当前卡牌类型关系：

```text
游侠牌：默认都是可部署实体。
越权模块牌：若表现为装备、设备或武器，也视为可部署实体。
```

暂不作为可部署实体：

```text
一次性反制 / 指令 / 回滚类越权模块。
```

说明：

```text
后续需要为每张越权模块标注“实体模块”或“一次性模块”。
实体模块拥有攻击力和生命值，可以部署在权限节点。
一次性模块只结算效果，不留在节点上。
```

---

## 三、攻击力与生命值

每个可部署实体都有两个基础数值：

```text
攻击力 / ATK
生命值 / HP
```

攻击力代表：

```text
该实体在接管权限节点、压制敌方实体或参与节点争夺时提供的进攻强度。
```

生命值代表：

```text
该实体在权限节点上承受攻击、清理、回滚或系统压制时的生存能力。
```

卡面建议显示：

```text
ATK / HP
```

中文版也可以使用图标：

```text
攻击 / 生命
```

## 四、部署到权限节点

部署定义：

```text
打出一个可部署实体时，选择1座在线权限节点，将该实体部署到该权限节点上。
```

部署限制：

```text
可部署实体只能部署到在线权限节点。
离线权限节点不能部署新实体。
```

控制关系：

```text
部署在权限节点上的实体仍由其控制者控制。
```

节点归属：

```text
权限节点的控制者由当前规则的接管结果决定。
部署实体不必然等于控制该节点。
```

说明：

```text
同一座权限节点上可以存在多个玩家控制的实体。
节点争夺通过接管行动和战斗结算处理。
```

## 五、游侠部署

游侠是最基础的可部署实体。

```text
打出游侠时，支付其电荷费用，然后将其部署到1座在线权限节点。
```

游侠费用：

```text
游侠费用默认等于当前星级。
```

游侠数值：

```text
游侠拥有攻击力和生命值。
星级影响数值上限和装备层级，但不直接替代攻击力和生命值。
```

## 六、装备、设备、武器部署

装备、设备、武器属于越权模块中的实体模块。

```text
打出实体模块时，支付模块费用，然后部署到1座在线权限节点。
```

实体模块类型：

```text
装备：偏向保护、增强、生存。
设备：偏向侦察、干扰、节点控制。
武器：偏向攻击、清理、压制。
```

实体模块数值：

```text
实体模块同样拥有攻击力和生命值。
```

说明：

```text
实体模块可以作为节点上的战术存在，而不是只结算一次效果后进入弃牌区。
```

---

## 七、一次性模块

部分越权模块仍然可以保持一次性效果。

```text
一次性模块没有持续部署状态。
一次性模块可以没有攻击力和生命值。
一次性模块结算后进入手牌弃牌区。
```

当前判断：

```text
G-002 零日芯片：更适合作为一次性反制模块。
G-005 强制回滚扇区：更适合作为一次性大范围模块。
```

说明：

```text
是否所有越权模块都改成实体模块，需要后续逐张审核。
当前先建立“实体模块 / 一次性模块”区分。
```

## 八、节点战斗基础口径

接管权限节点时，比较参与接管的实体战斗能力。

基础方向：

```text
进攻方选择参与接管的实体。
防守方选择该节点上的防守实体。
双方根据攻击力、生命值、技能和节点效果结算。
```

当前暂定：

```text
攻击力用于造成伤害。
生命值降到0的实体被处理，置入来源手牌弃牌区。
```

说明：

```text
具体战斗流程需要后续单独测试，不在本文件一次性定死。
```

## 九、与旧星级系统的关系

星级不废除。

```text
星级继续代表游侠当前装备状态、打出费用和大致强度层级。
```

但星级不再替代战斗数值。

```text
具体战斗由攻击力、生命值、技能、节点状态共同决定。
```

推荐数值方向：

```text
1星：低攻击 / 低生命。
2星：基础攻击 / 基础生命。
3星：稳定攻击 / 稳定生命。
4星：高攻击 / 高生命。
5星：核心攻击 / 核心生命。
```

## 十、卡面信息影响

游侠卡面未来至少需要容纳：

```text
1. 卡名
2. 费用
3. 攻击力
4. 生命值
5. 技能描述
6. 卡号
7. Logo
```

实体模块卡面未来至少需要容纳：

```text
1. 卡名
2. 费用
3. 模块类型：装备 / 设备 / 武器
4. 攻击力
5. 生命值
6. 技能描述
7. 卡号
8. Logo
```

一次性模块卡面可以不显示攻击力和生命值。
