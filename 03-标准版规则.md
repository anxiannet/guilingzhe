# 归零者｜标准版规则

> 文件定位：本文件记录 `归零者 / ZERO ACCESS` 独立项目当前标准版总规则、通用规则、回合流程与结算规则。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`
>
> 说明：具体卡牌费用与技能以当前卡表为准；本文件只维护总规则。

---

## 当前建议

```text
标准流程支持2到4人共用牌库游玩。
节点作为场上争夺对象存在。
节点只属于一个玩家，或处于无主状态。
一个节点上不能同时存在多个玩家的实体。
无主节点上不能存在任何实体。
每座节点最多有3个实体。
每回合不限制打出手牌、移动己方实体或攻击节点的次数，只限制权限电荷与实体状态。
玩家可以支付与派出1个实体攻击节点相同的电荷，移动1个自己的游侠或装备。
```

---

## 一、游戏目标

正式胜利条件：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

系统域、节点名称、编号与技能，测试阶段以 `04-测试版卡表.md` 为准；标准版定稿后以 `05-标准版卡表.md` 为准。

## 二、术语统一

| 概念 | 当前术语 |
|---|---|
| 行动单位 | 游侠 / Operative |
| 非游侠手牌卡 | 物品卡 / Gear |
| 物品卡形态 | 装备、一次性物品 |
| 可部署实体 | 游侠、装备 |
| 攻击力 | ATK |
| 生命值 | HP |
| 游戏资源 | 权限电荷 / Access Charge |
| 争夺对象 | 节点 |
| 区域组 | 系统域 / System Domain |
| 己方位置调整 | 移动实体 |
| 节点争夺行为 | 攻击节点、占领节点 |
| 控制状态变化 | 获得节点控制权 |

禁用术语：

```text
不再使用“接管节点”“被接管”“接管算法”。
规则结算统一写“获得节点控制权”。
胜利条件统一写“控制节点”。
```

## 三、游戏准备

### 2到4人共用牌库模式

```text
1. 将标准手牌洗混，作为公共手牌牌库。
2. 设置1个公共弃牌区。
3. 将标准节点牌洗混，作为节点牌库。
4. 从节点牌库翻开3张节点，放入中央节点区。
5. 中央节点初始为在线、无主、无实体。
6. 每名玩家抽5张起始手牌。
7. 随机决定起始玩家。
8. 从第1回合开始游戏。
```

双人游玩默认仍使用共用牌库模式；独立双人对战规则尚未设定。

---

## 四、牌库模式

```text
玩家共用1个手牌牌库。
玩家共用1个公共弃牌区。
玩家共用1个节点牌库。
中央节点区通常保持3座无主节点。
```

公共弃牌区：

```text
被消灭的游侠与装备进入公共弃牌区。
结算后的一次性物品进入公共弃牌区。
因手牌上限弃掉的牌进入公共弃牌区。
公共手牌牌库耗尽后，当前规则不自动重洗公共弃牌区。
```

---

## 五、中央节点区、节点控制与回流

中央节点区：

```text
中央节点区放置当前可被争夺的无主节点。
无主节点不能成为攻击目标，也不能成为普通移动的目标。
玩家只能通过从手牌部署游侠或装备获得无主节点控制权。
```

部署到无主节点：

```text
玩家将游侠或装备部署到中央节点区的无主节点后，获得该节点控制权。
该节点移入玩家控制节点区。
随后从节点牌库补充中央节点区至3座。
```

节点回流：

```text
若已控制节点因实体离场、移动、返回手牌、进入弃牌区或其他效果而没有任何己方实体，该节点立即变为无主。
在使用节点牌库的模式中，该节点放回节点牌库并重洗。
随后补充中央节点区至3座。
节点回流时清除控制者、保护状态与临时状态，并恢复为在线、无主、无实体。
```

## 六、节点状态

在线节点：

```text
正面信息、控制者与实体公开。
可以部署、移动己方实体进入、被攻击、被占领并发动节点技能。
可以计入 Root Access 胜利条件。
```

离线节点：

```text
正面信息不公开，控制者与实体公开。
可以部署、移动己方实体进入、被攻击和被占领。
不能发动节点技能，不计入 Root Access 胜利条件。
```

保护节点：

```text
被保护节点不能成为攻击目标。
被保护节点不能被技能变为离线。
控制者仍可向其中部署或移动己方实体。
保护持续到保护提供者的下一个回合开始。
```

## 七、权限电荷与费用

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用电荷全部清空。
没有足够电荷时，不能打牌、移动实体、攻击节点或发动技能。
```

费用口径：

```text
游侠和物品卡费用以卡表为准。
节点技能费用以节点卡面为准。
游侠与装备的入场效果包含在打出费用中。
一次性物品不额外支付技能费用。
攻击节点时，每派出1个己方实体，支付1电荷。
普通移动时，每移动1个己方实体，支付与派出1个实体攻击节点相同的电荷；当前为1电荷。
若以后调整单个攻方实体的攻击费用，普通移动单个实体的费用同步调整。
```

## 八、回合流程与行动限制

每名玩家的回合：

```text
1. 回合开始：清除由当前玩家提供、持续至此时的保护效果。
2. 获得权限电荷。
3. 抽1张牌。
4. 节点技能窗口：最多使用1次已控制且在线节点的节点技能。
5. 行动阶段：按任意顺序打出手牌、移动己方实体或攻击节点。
6. 结束阶段：清除本回合临时状态，清空剩余电荷，手牌超过7张则弃至7张。
```

行动阶段可执行：

```text
打出游侠。
打出装备。
打出一次性物品。
移动己方实体。
攻击节点。
主动结束行动阶段。
```

通用限制：

```text
每回合不限制打牌、普通移动或攻击节点的行动次数。
每项行动均须立即支付对应电荷。
刚上场的游侠或装备，本回合不能普通移动，也不能参与攻击节点。
节点技能每回合最多使用1次，且只能在节点技能窗口使用。
```

## 九、打出手牌流程

### 打出游侠或装备

```text
1. 选择手牌中的1张游侠或装备。
2. 支付其费用。
3. 选择中央节点区的无主节点，或你控制且未满3个实体的节点。
4. 将实体部署到目标节点。
5. 若部署到无主节点，你获得该节点控制权并补充中央节点区。
6. 结算该牌技能。
7. 检查 Root Access。
```

### 打出一次性物品

```text
1. 选择1张一次性物品并支付费用。
2. 选择合法目标并结算效果。
3. 结算后进入公共弃牌区。
4. 若节点控制权、在线状态或归属发生变化，立即检查 Root Access。
```

## 十、可部署实体、驻守上限与节点归属

```text
游侠和装备是可部署实体。
一次性物品不是可部署实体。
每座节点最多有3个实体。
同一节点上的实体必须全部属于该节点控制者。
无主节点上不能存在实体。
控制节点的必要条件是该节点上至少有1个该玩家的实体。
```

部署定义：

```text
从手牌打出游侠或装备并放入节点称为部署。
部署到无主节点会获得该节点控制权。
部署到己方节点称为增援。
```

入场限制：

```text
刚上场实体指本回合从手牌打出并部署的游侠或装备。
刚上场实体本回合不能普通移动，也不能参与攻击节点。
从下一次其控制者回合开始，才可普通移动或攻击。
移动不是部署，不会产生新的刚上场状态，也不触发入场技能。
```

入场技能限制：

```text
可部署实体的入场技能不得将己方场上实体返回手牌。
己方场上实体回手只允许出现在一次性物品上。
```

## 十一、普通移动己方实体

普通移动是行动阶段的通用行动，不是卡牌技能，也不是节点技能。

移动条件：

```text
1. 选择自己控制的1个场上游侠或装备。
2. 该实体不能是本回合刚上场的实体。
3. 该实体本回合尚未参与攻击节点。
4. 该实体本回合尚未进行过普通移动。
5. 选择自己控制的另一座节点作为目标节点。
6. 目标节点可以在线或离线。
7. 目标节点必须少于3个实体。
8. 不能移动到当前所在节点、无主节点或其他玩家控制的节点。
9. 支付与派出1个实体攻击节点相同的电荷；当前为1电荷。
10. 将该实体移动到目标节点。
```

结算说明：

```text
普通移动不视为部署，不触发入场技能。
普通移动不改变实体控制者。
每个实体每回合最多进行1次普通移动。
实体完成普通移动后，本回合仍可参与1次攻击节点，只要其ATK大于0且满足攻击条件。
实体一旦在本回合参与过攻击节点，本回合不能再普通移动。
若来源节点因移动而没有任何己方实体，立即结算节点回流。
普通移动完成后，立即检查节点归属与 Root Access。
```

设计口径：

```text
普通移动用于在自己控制的节点之间重新调配驻守与攻击位置。
普通移动不能用于支付1电荷直接占领无主节点。
普通移动不能侵入其他玩家控制的节点。
```

---

## 十二、节点技能与移动技能边界

```text
节点技能可以使用比普通行动更宽的动作集合。
节点技能的具体目标、费用和限制以节点卡表为准。
节点技能移动不自动受“每个实体每回合最多1次普通移动”限制，除非技能另有说明。
节点技能移动不是普通移动，不额外支付普通移动费用。
手牌技能不能因为存在普通移动行动而自动获得任意移动效果。
```

移动类节点技能的一般口径：

```text
未明确写出其他目标时，只能移动己方场上实体。
目标通常必须是自己控制且未满3个实体的节点。
不能移动到无主节点或其他玩家控制的节点，除非技能明确允许。
移动不是部署；旧实体若本回合尚未攻击，仍可参与攻击节点。
```

## 十三、攻击节点条件

```text
1. 选择任意数量的己方游侠或装备作为攻方。
2. 攻方必须来自你控制的节点。
3. 攻方不能是本回合刚上场的实体。
4. 攻方不能是本回合已经参与过攻击的实体。
5. ATK为0的实体不能参与攻击。
6. 每派出1个攻方实体，支付1电荷。
7. 目标必须是其他玩家控制且未被保护的节点。
8. 无主节点不能成为攻击目标。
```

完成普通移动的实体仍可在本回合攻击；已经攻击过的实体不能在攻击后普通移动。

---

## 十四、攻击节点结算

```text
1. 选择合法目标节点与攻方实体。
2. 为每个攻方实体支付攻击费用。
3. 决定攻方实体的攻击顺序。
4. 每个攻方实体攻击时，守方选择1个守方实体迎战。
5. 双方同时造成等于各自ATK的伤害。
6. 受到伤害大于等于HP的实体被消灭并进入公共弃牌区。
7. 每个攻方实体最多攻击1次。
8. 重复结算，直到攻方均已攻击、攻方全部被消灭，或守方没有实体。
```

战斗伤害只在本次攻击节点期间记录，攻击结束后清除。

---

## 十五、占领节点结算

占领成功：

```text
若攻击结束时目标节点没有守方实体，且至少有1个攻方实体存活，存活攻方进入目标节点，攻击方获得节点控制权。
目标节点最多保留3个存活攻方；超出的存活攻方返回各自来源节点。
```

占领失败：

```text
若目标节点仍有守方实体，存活攻方返回各自来源节点，控制权不变。
```

双方清空：

```text
若双方都没有存活实体，目标节点变为无主。
```

攻击结算完成后检查所有来源节点；没有己方实体的来源节点立即变为无主并按节点回流处理。

## 十六、Root Access 检查时点

以下时点必须立即检查：

```text
1. 部署到无主节点并获得控制权后。
2. 攻击成功并占领节点后。
3. 普通移动或技能移动完成后。
4. 节点在线或离线状态改变后。
5. 节点变为无主后。
6. 节点回流或中央节点区补充后。
```

一旦玩家控制同一系统域的3座在线节点，立即获得 Root Access 并获胜，不等待回合结束。

---

## 十七、牌库耗尽与测试结算

```text
公共手牌牌库耗尽后不自动重洗公共弃牌区。
节点牌库耗尽后不再补充中央节点区。
游戏继续，直到有人获得 Root Access 或触发测试结算。
```

测试结算顺序：

```text
1. 控制在线节点数量多者胜。
2. 若相同，比较在线节点上己方实体ATK总和。
3. 若仍相同，比较单一系统域中控制的最多在线节点数。
4. 若仍相同，比较场上实体数量。
5. 若仍相同，平局。
```

---

## 十八、临时状态

```text
本回合ATK或HP增益在当前玩家回合结束时清除。
实体离场、返回手牌、返回牌库或进入弃牌区时，清除全部临时状态。
刚上场状态限制本回合普通移动与攻击。
普通移动状态用于限制同一实体每回合最多普通移动1次。
战斗伤害在攻击节点结束后清除。
节点回流时清除节点控制者、保护状态与临时状态。
```

## 十九、技能自动测试原则

```text
所有技能必须可被规则引擎解析和自动测试。
所有技能不得依赖喊名、答应、语音触发或真人反应。
技能文本必须明确目标、数量、区域、动作、持续时间与归属。
```
