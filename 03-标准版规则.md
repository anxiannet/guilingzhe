# 归零者｜标准版规则

> 文件定位：本文件记录 `归零者 / ZERO ACCESS` 独立项目当前标准版总规则、通用规则、回合流程与结算规则。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`
>
> 说明：具体卡牌费用与技能以当前卡表为准；本文件只维护总规则。

---

## 一、游戏目标

```text
控制同一系统域的3座节点，立即获得 Root Access 并获胜。
```

---

## 二、卡牌与装备类型

一级卡牌类型：

```text
游侠。
物品卡。
节点。
```

物品卡分为：

```text
装备。
一次性物品。
```

装备按部署方式分为：

```text
节点装备。
游侠装备。
通用装备。
```

### 节点装备

```text
只能作为独立实体部署到节点。
占用节点的1个部署位。
不能由游侠携带。
拥有自己的ATK与HP。
不能通过“移动”行动改变节点，只能执行“重新部署”。
```

### 游侠装备

```text
不能直接部署到节点。
打出时必须装备给同一玩家控制的1个游侠。
不占用节点部署位。
每个游侠最多携带1件游侠装备。
游侠装备不是独立实体，不能单独攻击、被攻击或控制节点。
游侠装备的ATK作为携带者的ATK加成。
游侠装备的HP作为携带者的HP加成；HP为0或卡面明确不提供HP加成的装备除外。
```

### 通用装备

```text
既可以作为节点装备独立部署，也可以作为游侠装备由游侠携带。
独立部署时占用节点1个部署位，并使用自己的ATK与HP。
由游侠携带时不占用节点部署位，其ATK与HP分别加入游侠的ATK与HP；卡面明确不提供某项加成时除外。
每个游侠最多携带1件装备，包括游侠装备或通用装备。
独立部署时按照节点装备规则执行重新部署。
```

## 三、节点部署位与实体

```text
每座节点最多容纳3个部署实体。
计入部署位的只有游侠与独立部署的节点装备或通用装备。
由游侠携带的游侠装备或通用装备不占部署位。
同一节点上的部署实体必须全部属于该节点控制者。
无主节点不能存在任何部署实体或被携带装备。
控制节点的必要条件是节点上至少有1个己方部署实体。
```

装备数量不改变节点的3个部署位上限：

```text
3个游侠，每个各携带1件装备，仍只占3个部署位。
2个游侠加1个独立部署的节点装备，共占3个部署位。
```

## 四、游戏准备

```text
1. 将标准手牌洗混，作为公共手牌牌库。
2. 设置公共弃牌区。
3. 将标准节点牌洗混，作为节点牌库。
4. 翻开3张节点放入中央节点区。
5. 中央节点初始为无主、无实体。
6. 每名玩家抽5张起始手牌。
7. 随机决定起始玩家。
```

---

## 五、权限电荷

```text
每回合开始时，当前玩家获得等同于当前回合数的权限电荷，最高为10。
第10回合起，每回合获得10电荷。
回合结束时，未使用电荷全部清空。
```

费用口径：

```text
打出卡牌支付卡面费用。
发动节点技能支付节点卡面费用。
攻击节点时，每派出1个攻击实体，当前支付1电荷。
游侠普通移动时，每移动1个游侠，当前支付1电荷。
节点装备重新部署时，支付与游侠普通移动相同的费用；当前为1电荷。
同一节点内改变通用装备的部署方式不支付电荷。
```

## 六、回合流程

```text
1. 回合开始。
2. 获得权限电荷。
3. 抽1张牌。
4. 行动阶段。
5. 结束阶段。
```

行动阶段可以按任意顺序执行：

```text
打出游侠。
打出节点装备。
打出游侠装备。
打出通用装备。
打出一次性物品。
移动己方游侠。
重新部署己方节点装备或独立部署的通用装备。
改变同节点内通用装备的部署方式。
攻击节点。
使用节点技能。
主动结束行动阶段。
```

```text
每名玩家每回合最多使用1次节点技能。
节点技能可在行动阶段任意合法时点发动。
```

## 七、打出游侠与装备

### 打出游侠

```text
支付费用，将游侠部署到中央节点区的无主节点，或自己控制且未满3个部署实体的节点。
部署到无主节点后，获得该节点控制权。
```

### 打出节点装备

```text
支付费用，将节点装备独立部署到中央节点区的无主节点，或自己控制且未满3个部署实体的节点。
节点装备占用1个部署位。
```

### 打出游侠装备

```text
支付费用，选择自己控制的1个未携带装备的游侠。
将游侠装备置于该游侠下方或旁边，表示由该游侠携带。
不能将游侠装备直接部署到节点。
打出游侠装备不增加节点部署实体数量。
```

### 打出通用装备

支付费用后选择一种方式：

```text
1. 作为节点装备独立部署，占用1个部署位。
2. 装备给自己控制的1个未携带装备的游侠，不占部署位。
```

刚打出的游侠本回合不能普通移动或参与攻击。刚独立部署的装备本回合不能重新部署或参与攻击。

## 八、携带装备规则

```text
每个游侠最多携带1件游侠装备或通用装备。
携带装备与游侠视为位于同一节点。
携带装备随游侠一起移动、攻击、进入目标节点或返回来源节点。
携带装备不单独支付移动或攻击费用。
携带装备不单独占用攻击实体数量。
```

属性结算：

```text
携带装备的ATK加入游侠ATK。
携带装备的HP加入游侠HP。
若装备HP为0，或卡面明确写明不增加HP，则不提供生命值加成。
驱离枪为HP 0，因此只提供ATK加成，不增加游侠HP。
```

合并战斗结算：

```text
游侠与其携带装备在战斗中视为1个战斗单位。
该战斗单位的最终ATK = 游侠ATK + 携带装备ATK。
该战斗单位的最终HP = 游侠HP + 携带装备HP；HP为0或卡面明确不增加HP的装备除外。
攻击节点时，只选择该游侠作为攻方或守方，不单独选择其携带装备。
携带装备不单独造成伤害，也不单独承受普通战斗伤害。
若该战斗单位受到的伤害大于等于最终HP，该游侠被消灭，其携带装备同时进入公共弃牌区。
```

装备单独处理：

```text
卡牌或节点技能可以单独选择被携带装备进行返回、销毁、夺取或其他处理，只要技能文本允许。
若携带装备在战斗中或伤害结算前离场，立即重新计算该游侠的最终ATK与最终HP。
若重新计算后，该游侠当前受到的伤害大于等于新的最终HP，该游侠立即被消灭。
普通战斗伤害仍在本次攻击节点结束后清除。
```

离场结算：

```text
游侠被消灭时，其携带装备同时进入公共弃牌区，除非装备技能另有说明。
游侠返回手牌时，其携带装备也返回其控制者手牌。
游侠进入牌库或其他区域时，其携带装备跟随进入同一区域，除非效果明确只处理游侠。
若携带装备被效果单独销毁，游侠保留在原位置，并立即重新计算ATK与HP。
```

## 九、通用装备的同节点转换

玩家可以在行动阶段改变自己通用装备在同一节点内的位置：

```text
独立部署的通用装备，可以改由同节点内1个未携带装备的己方游侠携带。
由游侠携带的通用装备，可以卸下并作为独立装备留在同一节点，但该节点必须有空余部署位。
由1个游侠携带的通用装备，可以转交给同节点内另1个未携带装备的己方游侠。
```

转换规则：

```text
同节点内转换不支付电荷。
转换不是打出、部署、移动、重新部署或重新上场。
转换不触发上场技能、部署技能或入场技能。
转换不会清除装备的临时状态。
转换后立即重新计算原携带者和新携带者的ATK与HP。
若转换导致某游侠当前受到的伤害大于等于其新HP，该游侠立即被消灭。
转换后必须继续满足每个游侠最多携带1件装备和节点最多3个部署实体的限制。
```

## 十、游侠普通移动

```text
普通移动只能移动自己控制的游侠。
目标必须是自己控制的另一座未满节点。
不能移动到无主节点或其他玩家节点。
每个游侠每回合最多普通移动1次。
刚上场的游侠本回合不能普通移动。
```

携带关系：

```text
移动游侠时，其携带装备随其移动，不额外支付电荷，也不占目标节点部署位。
游侠装备不能脱离游侠单独移动。
```

移动与攻击：

```text
同一游侠每回合可以普通移动1次并参与攻击1次。
可以先移动后攻击，也可以先攻击后移动。
两项行动分别支付费用。
```

若来源节点因移动而没有任何己方部署实体，立即结算节点回流。

## 十一、节点装备重新部署

```text
重新部署只能选择自己控制的节点装备，或以节点装备方式独立部署的通用装备。
支付与游侠普通移动相同的费用；当前为1电荷。
将其放到自己控制的另一座未满节点。
不能重新部署到无主节点或其他玩家节点。
每个装备实体每回合最多重新部署1次。
刚部署或刚转为独立部署状态的装备，本回合不能重新部署。
```

结算口径：

```text
重新部署不是移动。
重新部署不是打出、部署或重新上场。
重新部署不触发装备的上场技能、部署技能、入场技能或其他以部署为条件的效果。
重新部署不会清除装备的临时状态。
装备卡面明确禁止重新部署时，不能执行重新部署。
```

若来源节点因重新部署而没有任何己方部署实体，立即结算节点回流。

---

## 十二、攻击节点

攻方可以选择：

```text
游侠。
独立部署且ATK大于0、且卡面未禁止参与攻击的节点装备或通用装备。
```

```text
携带装备的游侠使用合并后的最终ATK与最终HP参与战斗。
携带装备不作为独立攻方或守方，也不单独承受普通战斗伤害。
独立部署装备使用自己的ATK与HP参与战斗。
每个部署实体每回合最多参与攻击节点1次。
每派出1个攻击实体，支付1电荷。
```

占领成功后，存活攻方进入目标节点；游侠携带的装备继续由该游侠携带，不额外占用部署位。

---

## 十三、节点回流

```text
若已控制节点上没有任何己方部署实体，该节点立即变为无主。
携带装备不能单独维持节点控制权。
节点回流时，节点放回节点牌库并重洗，随后补充中央节点区。
```

---

## 十四、节点技能与技能移动

```text
发动节点技能时，玩家必须控制该节点。
每名玩家每回合最多使用1次节点技能。
节点技能可以在行动阶段任意合法时点发动。
节点技能的目标、费用和限制以节点卡面为准。
```

涉及装备的技能必须明确处理对象：

```text
“游侠或装备”包括独立部署装备和被游侠携带的装备。
若技能移动被携带装备，默认连同携带它的游侠一起移动，除非技能明确写“卸下”或“转移装备”。
若效果返回、夺取或销毁被携带装备，可以只处理该装备。
技能造成的位置变化不是节点装备的重新部署，按技能文本结算。
```

## 十五、Root Access检查

以下时点立即检查：

```text
获得节点控制权后。
攻击并占领节点后。
游侠普通移动、节点装备重新部署或技能移动后。
装备转换导致节点部署实体数量变化后。
节点技能结算后。
节点变为无主或回流后。
```

## 十六、状态记录

```text
刚上场状态：限制游侠本回合普通移动与攻击；限制独立部署装备本回合重新部署与攻击。
普通移动状态：每个游侠每回合最多普通移动1次。
重新部署状态：每个节点装备或独立部署通用装备每回合最多重新部署1次。
攻击状态：每个部署实体每回合最多参与攻击1次。
携带状态：记录装备当前携带者。
部署状态：记录通用装备当前独立部署或被游侠携带。
携带属性：记录装备当前提供的ATK与HP加成。
战斗伤害状态：记录游侠与携带装备合并后的战斗单位在本次攻击中受到的伤害。
```
