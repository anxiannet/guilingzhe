# 零权限｜主题总纲

> 文件定位：本文件是 `零权限 / Zero Access` 皮肤主题的总纲，用于统一背景、玩家目标、系统域、争夺理由与后续卡表映射方向。
>
> 本主题仍为备选皮肤方案，不修改《夕妖：抢地盘》正式规则、卡表、技能文案或视觉规范。

---

## 一、核心定位

```text
零权限 / Zero Access
```

一句话定位：

```text
一款关于系统弃民、越权装备与城市权限夺取的战术卡牌游戏。
```

英文定位：

```text
A tactical card game of zero-access crews, override gear, and Root Access takeover.
```

核心口号：

```text
从零权限，到根权限。
```

英文：

```text
From Zero Access to Root Access.
```

---

## 二、玩家身份

玩家扮演一支“零权限小队”的首领。

零权限不是强者身份，而是起点。

在这个未来城市中，失去系统权限的人无法开门、无法买药、无法乘车、无法证明身份，也无法获得任何公共系统响应。系统不会逮捕他们，因为系统已经从档案里删除了他们。

他们被称为：

```text
零权限者
Zero Access
```

玩家的目标不是接受任务、拿报酬或帮别人回收物品，而是带领被系统删除的人夺取城市系统的最高权限。

```text
起点：Access Level 0
目标：Root Access
幻想：从系统弃民变成系统主人
```

---

## 三、核心争夺理由

未来城市由“权限系统”统治。

门禁、医疗、无人机、信用、身份、交通、住房、武器与基础服务都接入同一套权限网络。拥有权限的人可以调用城市服务；失去权限的人连存在本身都无法证明。

每个系统域同一时间只能存在一个 Root Access。

一支小队夺取 Root Access 后，其他小队会被降权、封锁、踢出节点，甚至再次归零。

因此玩家对抗的理由不是单纯打架，而是：

```text
同一个系统域只承认一个最高权限。
你不夺权，别人就会用权限定义你的生存边界。
```

## 四、首版系统域

作为《抢地盘》同规则换皮方案，首版先固定 4 个系统域，每个系统域 3 个权限节点，对应原版 4 个妖域、12 块地盘。

```text
1. 门禁域 Access Domain
2. 医疗域 Medical Domain
3. 无人机域 Drone Domain
4. 信用域 Credit Domain
```

`身份域 Identity Domain` 暂不进入首版，作为扩展包或独立规则候选。

## 六、胜利幻想

控制同一系统域的三座权限节点，即获得该系统的 Root Access。

这不是占领一块地，也不是完成外包任务，而是直接接管城市现实的一部分。

示例：

```text
控制门禁域：所有门为你打开，敌人被锁在门外。
控制医疗域：你的同伴被优先救治，敌人的急救请求被拒绝。
控制无人机域：天空中的无人机群成为你的眼睛和手。
控制信用域：你的债务被清零，敌人的账户被冻结。
```

## 九、主题边界

本主题不应变成普通赛博朋克、普通黑客游戏或普通雇佣兵撤离游戏。

必须保留的核心差异：

```text
玩家争夺的不是钱、货物、黑箱、地堡或悬赏。
玩家争夺的是城市系统的最高权限。
```

核心判断句：

```text
系统不给你权限，你就夺取 Root Access。
```
