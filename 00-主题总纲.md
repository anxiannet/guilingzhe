## 三、玩家身份

玩家扮演一支“归零者小队 / 零权限小队”的首领。

零权限不是强者身份，而是起点。

在这个未来城市中，失去系统权限的人无法开门、无法买药、无法乘车、无法证明身份，也无法获得任何公共系统响应。系统不会逮捕他们，因为系统已经从档案里删除了他们。

他们被称为：

```text
零权限者
Zero Access
```

玩家的目标不是接受任务、拿报酬或帮别人回收物品，而是带领被系统删除的人夺取城市系统的最高权限。

```text
起点：Access Level 0
目标：Root Access
幻想：从系统弃民变成系统主人
```

---

## 四、核心争夺理由

未来城市由“权限系统”统治。

节点状态、节点保护、人员调度、装备调度、游侠强化与装备强化都接入同一套权限网络。拥有权限的人可以读取节点、上线节点、断开节点、保护节点、调动游侠、调动装备，并把弃置资源重新纳入小队权限。

每个系统域同一时间只能存在一个 Root Access。

一支小队夺取 Root Access 后，其他小队会被降权、封锁、踢出节点，甚至再次归零。

因此玩家对抗的理由不是单纯打架，而是：

```text
同一个系统域只承认一个最高权限。
你不夺权，别人就会用权限定义你的生存边界。
```

## 五、首版系统域

首版固定 4 个系统域，每个系统域 3 个节点。

区域名对应技能组，节点名对应技能效果。

```text
1. 状态域 Status Domain
2. 调度域 Dispatch Domain
3. 游侠域 Operative Domain
4. 装备域 Gear Domain
```

---

## 六、12 个节点

| 系统域 | 节点 1 | 节点 2 | 节点 3 | 控制幻想 |
|---|---|---|---|---|
| 状态域 Status Domain | 节点档案 Node File | 节点上线 Node Online | 节点断线 Node Offline | 读取、上线或断开节点状态 |
| 调度域 Dispatch Domain | 节点护盾 Node Shield | 游侠调度 Operative Dispatch | 装备调度 Gear Dispatch | 保护节点并调动己方实体 |
| 游侠域 Operative Domain | 游侠火力 Operative Firepower | 游侠护甲 Operative Armor | 游侠召回 Operative Recall | 强化或召回己方游侠 |
| 装备域 Gear Domain | 装备火力 Gear Firepower | 装备护甲 Gear Armor | 装备回收 Gear Recovery | 强化或回收己方装备 |

## 七、胜利幻想

控制同一系统域的三座节点，即获得该系统的 Root Access。

这不是占领一块地，也不是完成外包任务，而是直接接管城市现实的一部分。

示例：

```text
控制状态域：你决定哪些节点被看见、上线或断线。
控制调度域：你保护关键节点，并把游侠和装备调到需要的位置。
控制游侠域：你的游侠获得更强火力、防护与重新入场机会。
控制装备域：你的装备获得更强火力、防护与回收机会。
```

胜利句：

```text
控制同一系统域的3座在线节点，立即获得 Root Access 并获胜。
```

英文：

```text
Control all 3 online Nodes in one System Domain to gain Root Access and win.
```

## 九、主题边界

本主题不应变成普通赛博朋克、普通黑客游戏或普通雇佣兵撤离游戏。

必须保留的核心差异：

```text
玩家争夺的不是钱、货物、黑箱、地堡或悬赏。
玩家争夺的是城市系统的最高权限。
```

核心判断句：

```text
系统不给你权限，你就夺取 Root Access。
```
