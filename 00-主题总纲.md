# 归零者｜主题总纲

> 文件定位：本文件是 `归零者 / ZERO ACCESS` 独立项目的主题总纲，用于统一背景、玩家目标、系统域、争夺理由与卡表映射方向。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、核心定位

```text
归零者 / ZERO ACCESS
```

一句话定位：

```text
一款关于系统弃民、物品装备与城市权限夺取的独立战术卡牌项目。
```

核心口号：

```text
从零权限，到最高权限。
From Zero Access to Root Access.
```

---

## 二、命名口径

```text
归零者：产品名 / 项目名 / 阵营名。
ZERO ACCESS：英文产品名。
零权限：世界观状态 / 起点状态。
最高权限 / Root Access：城市系统最高控制权 / 胜利目标。
```

### 中文名称规范

```text
名称必须简单、直观，符合中文语境。
名称还必须符合《归零者》的时代背景、技术水平和世界观逻辑。
玩家看到名称后，应能直接判断它是什么，或大致理解它的用途。
名称不应依赖额外解释才能成立。
优先使用常见、明确、可识别的中文词组。
卡牌名称通常控制在2至5个汉字。
```

命名优先级：

```text
1. 先说明对象是什么。
2. 再体现最核心的功能、结构或识别特征。
3. 检查名称是否符合未来城市权限系统、零权限者和高科技封存载体的世界观。
4. 最后才考虑企业风格、技术气质和世界观修饰。
```

命名审核必须同时通过：

```text
中文语境审核：像自然中文名称，不是直译词或生造词。
对象识别审核：能看出是人物、装备、一次性物品或节点。
功能直觉审核：名称与主要技能或用途不冲突。
世界观审核：名称应属于《归零者》的城市权限、维相技术、纳米科技或底层改装体系。
时代感审核：不能像古代法器、现代日用品广告或普通网络软件名称，除非卡牌设定明确需要。
```

禁止方向：

```text
需要阅读设定说明后才知道是什么的名称。
只强调气势、等级或抽象概念，却不体现对象类型的名称。
生造难懂词、过度文艺词或翻译腔组合。
虽然功能准确，但与项目世界观和时代技术语境不符的名称。
先确定英文名，再机械翻译成不自然中文名。
```

## 三、玩家身份

玩家扮演一支“归零者小队 / 零权限小队”的首领。

零权限不是强者身份，而是起点。失去系统权限的人无法开门、买药、乘车或证明身份，系统已经从档案里删除了他们。

玩家的目标不是重新申请权限，而是带领被系统删除的人夺取城市系统的最高权限。

```text
起点：Access Level 0
目标：最高权限 / Root Access
幻想：从系统弃民变成系统主人
```

---

## 四、核心争夺理由

未来城市由“权限系统”统治。档案、防火墙、传送、货运、生化改造、纳米医疗、武器研究和回收设施全部接入城市权限网络。

每个系统域同一时间只能存在一个最高权限。

```text
同一个系统域只承认一个最高权限。
你不夺权，别人就会用权限定义你的生存边界。
```

## 五、首版系统域

首版固定4个系统域，每个系统域3个节点。

```text
1. 行政域 Administrative Domain
2. 安防域 Security Domain
3. 生化域 Biotech Domain
4. 科研域 Research Domain
```

---

## 六、12个节点

| 系统域 | 节点1 | 节点2 | 节点3 | 控制幻想 |
|---|---|---|---|---|
| 行政域 | 档案馆 | 中继站 | 屏蔽塔 | 读取、检索或屏蔽节点信息 |
| 安防域 | 防火墙 | 传送站 | 货运站 | 保护关键节点并控制安全通道与物资流转 |
| 生化域 | 蜂鸟巢 | 纳米诊所 | 复苏舱 | 强化、修复或召回己方游侠 |
| 科研域 | 武研所 | 装研所 | 回收中心 | 强化、加固或回收己方装备 |

---

## 七、胜利幻想

控制同一系统域的三座节点，即获得该系统的最高权限。

胜利句：

```text
控制同一系统域的3座节点，立即获得最高权限并获胜。
```

英文：

```text
Control all 3 Nodes in one System Domain to gain Root Access and win.
```

节点不再具有在线或离线状态。胜利条件、节点技能和移动规则均不检查节点在线状态。
