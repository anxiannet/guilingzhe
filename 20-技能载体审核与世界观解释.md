# 归零者｜技能载体审核与世界观解释

> 文件定位：本文件审核《归零者 / ZERO ACCESS》中技能是否放在合适载体上，并记录游侠、物品卡、节点三类载体与世界观的对应关系。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、审核结论

```text
技能必须放在符合世界观的载体上。
技能还必须具备明确规则意义。
标准版游侠与物品卡技能必须由 22-标准版基础动作池.md 中的基础动作变体或组合形成。
```

三类正式载体：

```text
游侠
物品卡
节点
```

判断标准：

```text
游侠技能：必须像“人能做的事”。
物品卡技能：必须像“物品产生的效果”。
节点技能：必须像“城市基础设施提供的系统能力”。
```

---

## 二、标准版基础动作约束

标准版游侠与物品卡技能只能使用以下基础动作：

```text
抽牌。
查看牌库顶。
调整牌库顶。
返回牌库。
查看手牌。
返回手牌。
消灭。
```

备选动作：

```text
重洗牌库。
```

当前明确移除：

```text
弃牌。
公开手牌。
隔离。
禁用。
费用变化。
攻击变化。
生命变化。
伤害。
治疗。
获得电荷。
己方返回牌库。
己方消灭。
```

## 三、卡名与技能直觉原则

```text
卡名必须让玩家能直觉联想到技能。
侦察类名字对应查看。
信使、抽取、广播类名字对应抽牌。
算盘、预读、洗牌类名字对应牌库调整。
弹弓、甩棍、门神、推土机类名字对应返回手牌。
清除者、爆破兵、清道夫类名字对应消灭。
回滚、回收、仓库类名字对应返回牌库或返回手牌。
```

不再保留“名字与效果脱节”的旧技能组合。

## 四、载体判断标准

### 游侠

适合放在游侠上的技能：

```text
观察。
侦察。
读牌。
传信。
贴身查看。
掩护撤回。
打退目标。
清除目标。
回收己方装备。
```

世界观解释：

```text
游侠是被系统删除后仍在城市边缘行动的人。
他们的技能应体现经验、动作、判断、身份、习惯或个人战术风格。
```

### 物品卡

适合放在物品卡上的技能：

```text
扫描。
探测。
排序。
抽取。
牵引。
回滚。
回收。
撤离。
消灭。
```

世界观解释：

```text
物品卡是归零者携带、捡来、抢来或改造的非人物物品。
装备、设备、工具、武器、权限卡、程序芯片和一次性物品都属于物品卡。
```

### 节点

节点技能不受本文件“标准版游侠与物品卡基础动作池”限制。

节点技能应体现该节点本身作为城市系统基础设施能够提供的能力。

## 八、自动测试影响

所有标准版游侠与物品卡技能必须能拆为基础动作字段：

```text
actor_player_id
source_card_id
target_player_id
target_card_id
target_card_type
target_deck_id
zone_from
zone_to
quantity
condition
 timing
```

示例：

```text
侦察员：target_deck_id + look_top_1。
信使：actor_player_id + draw_1。
预读者：target_deck_id + look_top_2 + reorder。
甩棍：target_operative_id + return_to_controller_hand。
清除者：target_operative_id + destroy_to_discard。
回收沙盒：target_operative_id + return_to_controller_deck_bottom。
算盘：target_deck_id + look_top_3 + reorder。
```

## 九、节点技能与世界观解释

节点技能当前仍单独维护，不因本次标准版游侠与物品卡技能重写而自动修改。

后续如需统一节点技能，也必须单独读取并审核节点系统。
