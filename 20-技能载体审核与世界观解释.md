# 归零者｜技能载体审核与世界观解释

> 文件定位：本文件审核《归零者 / ZERO ACCESS》中技能是否放在合适载体上，并记录游侠、物品卡、节点三类载体与世界观的对应关系。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、审核结论

```text
技能必须放在符合世界观的载体上。
```

三类正式载体：

```text
游侠
物品卡
节点
```

判断标准：

```text
游侠技能：必须像“人能做的事”。
物品卡技能：必须像“物品产生的效果”。
节点技能：必须像“城市基础设施提供的系统能力”。
```

---

## 二、载体判断标准

### 游侠

适合放在游侠上的技能：

```text
观察
潜入
移动
招募
维修
压制
冲锋
谈判
偷取信息
临场判断
```

世界观解释：

```text
游侠是被系统删除后仍在城市边缘行动的人。
他们的技能应体现经验、动作、判断、身份、习惯或个人战术风格。
```

### 物品卡

适合放在物品卡上的技能：

```text
扫描
锁定
束缚
隔离
回滚
保护
启动
牵引
固定
排序
反制
部署式侦察
```

世界观解释：

```text
物品卡是归零者携带、捡来、抢来或改造的非人物物品。
装备、设备、工具、武器、权限卡、纳米机器人、电磁手雷、程序芯片和一次性物品都属于物品卡。
```

### 节点

适合放在节点上的技能：

```text
门禁查看
交通调度
封锁保护
医疗复苏
生物档案读取
无人机侦察
空域重连
目标锁定
信用抽取
额度铸造
债务冻结
```

世界观解释：

```text
节点是城市权限系统的控制点。
节点技能应体现该节点本身作为系统基础设施能够提供的能力，而不是某个角色或道具的动作。
```

## 六、自动测试影响

所有转换后的技能仍遵守：

```text
有明确目标。
有明确数量。
有明确区域。
有明确动作。
有明确持续时间。
不依赖喊名、答应、语音触发或真人反应。
```

示例：

```text
G-008 铁锚：target_entity_id + duration_this_turn。
G-009 探针：target_node_id + reveal_front_info。
G-010 算盘：target_deck_id + look_top_3 + reorder。
G-011 打火机：actor_player_id + next_operative_cost_minus_1_this_turn。
G-012 弹弓：target_operative_id + return_to_controller_hand。
```
