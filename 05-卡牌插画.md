## 插画层与完整卡牌层区别

### 插画层

插画层只负责表现被封存的战术实体或系统设施。

```text
游侠插画：表现被封存的游侠。
物品卡插画：表现被封存的装备或模块。
节点插画：表现被封存的城市系统设施 / 机构地标 / 节点本体。
```

插画层不生成以下正式卡面信息：

```text
卡名
系统域标签
技能文字
封存卡号
节点位址
归零者 / ZERO ACCESS Logo
制作版本号
```

说明：

```text
插画不是完整卡牌。
插画不承担规则阅读、卡号识别、系统域识别和品牌标识。
插画只提供被封存实体的视觉主体。
```

### 完整卡牌层

完整卡牌层由以下内容合成：

```text
统一卡框
插画窗口
插画层
卡名
系统域标签
技能文字
封存卡号 / 节点位址
Logo
必要的测试版本标记
```

其中：

```text
系统域标签、封存卡号、节点位址属于世界观内正式卡面信息。
它们必须由设计工具人工排版。
不能由 AI 插画生成。
```

## 插画空间层次硬标准

插画必须具备明确空间层次与立体感。

```text
不能只有单层平铺概念图。
不能只是一个主体悬浮在空背景中。
不能做成普通平面海报感。
不能缺少前后景遮挡和空间纵深。
```

画面应尽量建立以下多层结构：

```text
特写：身份锚点 / 核心装置 / 武器或模块细节。
近景：主体本体 / 当前动作 / 节点核心结构。
中景：战术环境 / 相关设施 / 接入线路 / 空间平台。
远景：系统背景 / 所属系统域环境 / 城市权限结构纵深。
```

表现手段：

```text
遮挡
透视
体块
景深
光层
前后景错位
封存边缘
空间折叠关系
卡内压缩空间感
```

核心要求：

```text
插画应看起来像“被压缩封存在卡牌内部的立体战术实体”。
要通过特写、近景、中景、远景的层次关系，表现高科技二维封存体的卡内空间深度。
```

## 不同系统域的插画差异

不同系统域的插画差异主要体现在：

```text
1. 被封存的设施类型不同。
2. 空间结构不同。
3. 光线与材质气质不同。
4. 背景系统痕迹不同。
5. 动态元素不同。
```

注意：

```text
系统域差异可以通过插画气质体现。
但系统域标签、域代号、封存卡号和节点位址仍属于完整卡牌层，不进入插画层。
```

### 行政域 / Administrative Domain

核心插画对象：

```text
档案馆、中继站、屏蔽塔、行政数据库、节点索引柜、权限档案墙、城市管理终端。
```

空间结构：

```text
高冷行政大厅、服务器档案库、垂直档案墙、天线塔、信号屏蔽阵列。
```

视觉气质：

```text
冷白、行政蓝、银灰。
秩序、审查、记录、节点状态管理、冷酷的城市系统治理。
```

动态元素：

```text
档案抽屉光轨、节点索引线、中继信号束、屏蔽波纹、权限拒绝红框。
```

避免：

```text
不要画成普通政府办公楼。
不要画成现实档案室。
不要画成温暖图书馆。
必须体现权限系统、节点状态与高科技封存感。
```

### 安防域 / Security Domain

核心插画对象：

```text
防火墙、传送站、货运站、安全闸门、隔离通道、扫描门、受控物流轨道。
```

空间结构：

```text
安检大厅、防火墙核心、传送平台、货运栈桥、封闭通道、分层安全门。
```

视觉气质：

```text
深蓝、警戒红、冷灰。
防御、隔离、管制、受控移动、安全流转。
```

动态元素：

```text
防火墙光幕、传送光环、货运轨道光线、红色扫描束、隔离栅格。
```

避免：

```text
不要画成普通车站或普通仓库。
不要画成消防防火墙。
不要让传送站过度魔法化。
必须体现城市权限系统下的安全通道和受控物流。
```

### 生化域 / Biotech Domain

核心插画对象：

```text
蜂鸟巢、纳米诊所、复苏舱、生体强化舱、微型无人机巢、纳米医疗设备。
```

空间结构：

```text
洁净舱体、透明液舱、蜂巢停靠结构、环形医疗平台、生命维持管线。
```

视觉气质：

```text
青绿、冷白、深蓝。
生体强化、纳米修复、复苏、被权限化的医疗资源。
```

动态元素：

```text
蜂鸟微型无人机轨迹、纳米粒子流、生命曲线、透明培养液、复苏光柱。
```

避免：

```text
不要画成普通医院病房。
不要画成恐怖实验室。
不要脏污血腥。
必须体现生化权限系统、纳米医疗与封存科技感。
```

### 科研域 / Research Domain

核心插画对象：

```text
武研所、装研所、回收中心、装备实验台、装甲测试架、残件再造线、机械臂。
```

空间结构：

```text
研发中心、实验室、武器测试舱、装甲加固台、回收输送线、半自动装配平台。
```

视觉气质：

```text
紫蓝、工业橙、暗银。
研发、改造、加固、回收、把废件重新变成可用装备。
```

动态元素：

```text
机械臂焊接光、武器测试靶线、装甲扫描网、回收输送带、再造粒子流。
```

避免：

```text
不要画成普通工厂流水线。
不要画成军事仓库。
不要画成脏旧废土修车铺。
必须体现科研机构、装备研发与权限封存感。
```

## 节点插画审核清单

```text
1. 是否看起来像城市权限系统中的机构 / 建筑 / 地标。
2. 是否不是纯技能按钮、纯UI图标或说明书插图。
3. 是否与所属系统域气质一致。
4. 是否不生成任何卡面文字、卡号、Logo 或系统域标签。
5. 是否具备特写、近景、中景、远景的空间层次。
6. 是否有高科技二维封存体的卡内压缩空间感。
7. 是否没有旧四域残留：门禁域、医疗域、无人机域、信用域。
8. 是否没有过度霓虹、废土脏污、恐怖血腥或普通城市风景感。
```
