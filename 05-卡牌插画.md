## 插画层与完整卡牌层区别

### 插画层

插画层只负责表现被封存的战术实体或系统设施。

```text
执行者插画：表现被封存的执行者。
越权模块插画：表现被封存的装备或模块。
权限节点插画：表现被封存的城市系统设施 / 节点本体。
```

插画层不生成以下正式卡面信息：

```text
卡名
系统域标签
技能文字
封存卡号
节点位址
归零者 / ZERO ACCESS Logo
制作版本号
```

说明：

```text
插画不是完整卡牌。
插画不承担规则阅读、卡号识别、系统域识别和品牌标识。
插画只提供被封存实体的视觉主体。
```

### 完整卡牌层

完整卡牌层由以下内容合成：

```text
统一卡框
插画窗口
插画层
卡名
系统域标签
技能文字
封存卡号 / 节点位址
Logo
必要的测试版本标记
```

其中：

```text
系统域标签、封存卡号、节点位址属于世界观内正式卡面信息。
它们必须由设计工具人工排版。
不能由 AI 插画生成。
```

## 不同系统域的插画差异

不同系统域的插画差异主要体现在：

```text
1. 被封存的设施类型不同。
2. 空间结构不同。
3. 光线与材质气质不同。
4. 背景系统痕迹不同。
5. 动态元素不同。
```

注意：

```text
系统域差异可以通过插画气质体现。
但系统域标签、域代号、封存卡号和节点位址仍属于完整卡牌层，不进入插画层。
```

### 门禁域 / Access Domain

核心插画对象：

```text
门、闸机、电梯井、封锁线、安检塔、机械门阵列。
```

空间结构：

```text
垂直、高塔、通道、闸口、门框、隔离层。
```

视觉气质：

```text
冷白、蓝、银灰。
秩序、封闭、权限检查、通行限制。
```

动态元素：

```text
扫描线、门禁光束、闸门开合、权限拒绝投影、封锁栅格。
```

避免：

```text
不要画成普通城门。
不要画成中世纪门楼。
不要只画一扇普通门。
必须体现城市系统门禁设施。
```

### 医疗域 / Medical Domain

核心插画对象：

```text
医疗舱、纳米诊所、复苏舱、生物档案库、生命维持设备。
```

空间结构：

```text
舱体、圆形医疗舱、透明液舱、实验室隔间、生物数据柜。
```

视觉气质：

```text
白、青绿、冷光。
洁净、生命维持、系统化治疗、被权限化的医疗资源。
```

动态元素：

```text
生命曲线、纳米粒子、医疗扫描光、透明培养液、数据化生物档案。
```

避免：

```text
不要画成普通医院病房。
不要画成恐怖实验室。
不要脏污血腥。
必须体现医疗权限系统与封存科技感。
```

### 无人机域 / Drone Domain

核心插画对象：

```text
蜂鸟无人机、无人机巢、空域中继塔、目标网格、低空航线节点。
```

空间结构：

```text
开放空域、蜂巢停靠结构、环形中继、空中网格、悬浮平台。
```

视觉气质：

```text
深蓝、青蓝、冷白光。
机动、侦察、空域管制、微型无人机集群。
```

动态元素：

```text
蜂鸟微型无人机飞行轨迹、空域网格、目标锁定线、飞行中继光束。
```

避免：

```text
不要画成普通机场。
不要画成大型战斗机基地。
不要画成纯天空风景。
必须体现微型无人机、空域系统与节点设施。
```

### 信用域 / Credit Domain

核心插画对象：

```text
黑钱包、信用铸机、债务库、账本服务器、金融芯片阵列。
```

空间结构：

```text
金库、账本墙、服务器保险柜、铸币机、黑箱交易终端。
```

视觉气质：

```text
黑金、紫、暗银。
冷酷、结算、债务锁定、信用生成、金融权限控制。
```

动态元素：

```text
数字账本流、信用额度铸造光、债务锁链、黑金芯片、交易路径。
```

避免：

```text
不要画成普通金币堆。
不要画成赌场。
不要画成传统银行柜台。
必须体现信用系统、账本服务器和权限化金融控制。
```

## 插画差异与正式域标签的关系

插画需要让玩家在没有文字时也能感到系统域气质不同。

但最终识别仍由完整卡牌层确认：

```text
系统域标签
域代号
封存卡号 / 节点位址
域色光条
域图标
```

因此：

```text
插画负责“气质识别”。
完整卡牌层负责“正式识别”。
```

## 插画方向

- 权限节点：门禁服务器、医疗舱、无人机巢、信用铸机、身份锁。
- 执行者：零权限黑客、蜂鸟无人机手、机修装甲师、Root级装甲执行者。
- 越权模块：零日芯片、离线地堡、机修装甲、轨道钥匙、黑盒终端。

---

## 视觉重点

每张插画要体现“被拒绝的系统”与“强行越权”的反差。

卡牌不是普通插画卡，而是高科技二维封存体卡牌。

```text
执行者、越权模块、权限节点应像被压缩封存在卡牌内部，随时可以从二维权限载体中释放出来。
```

权限节点插画尤其要注意：

```text
插画中可以体现所属系统域的视觉气质，例如无人机、空域网格、蜂巢结构。
但不要在插画内直接生成系统域标签、封存卡号或节点位址。
这些属于完整卡牌层的正式信息。
```
