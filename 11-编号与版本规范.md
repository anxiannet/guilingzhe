# 归零者｜编号与版本规范

> 文件定位：本文件记录 `归零者 / ZERO ACCESS` 的卡号、存储序列、系统域识别与版本记录规范。
>
> 本文件不替代正式卡表或正式规则。

---

## 一、当前结论

```text
所有卡都要记录。
每张正式卡都必须有唯一卡号。
每张正式卡都必须有版本记录文件。
版本记录必须归属到唯一卡号下面，但不能生成第二套卡号。
12 个权限节点已经先建立版本记录。
公共牌库执行者与越权模块需要在卡号体系确定后继续建立版本记录。
```

核心口径：

```text
卡号只有卡面上的一个。
例如：N-007 / DN-D1。
它同时承担世界观识别、玩家认牌、资产归属和版本记录入口。
```

---

## 二、唯一卡号原则

每张正式卡只有一个卡号。

权限节点示例：

```text
N-007 / DN-D1
```

含义：

```text
N-007 = 全局封存卡存储序列。
DN-D1 = 系统域节点位址。
```

这个卡号同时用于：

```text
1. 卡面正式显示。
2. 世界观内封存卡识别。
3. 玩家认牌与收藏整理。
4. 资产目录归属。
5. 样卡版本记录入口。
6. 审核记录索引。
```

禁止：

```text
不要再创造第二套“版本管理完整号”。
不要把 CN-v1.0 拼进卡号。
不要把文件名当作卡号。
不要让同一张卡出现两个不同编号。
```

## 三、卡面信息与版本记录

### 卡面内正式内容

```text
卡名
系统域标签 / 类型识别
节点技能效果 / 卡牌技能效果
唯一卡号
归零者 / ZERO ACCESS 标识
```

这些内容属于世界观内真实信息，也是正式卡面内容。

### 版本记录内容

版本记录是卡号下面的附属记录，不是卡号本身。

```text
卡号：N-007 / DN-D1
语言版本：中文版
样卡版本：CN-v1.0
插图版本：ILLUS-01
卡框版本：FRAME-v1.0
导出版本：EXPORT-v1.0
审核结论：待审核 / 通过 / 修改
```

说明：

```text
版本可以记录，但不能改变卡号。
版本字段归属于卡号。
卡号始终只有一个。
```

## 五、权限节点外框原则

12 张权限节点统一使用：

```text
88 × 63 mm 横版
同一套深色金属外框
同一套插画窗口
同一套标题区
同一套系统域标签位置
同一套技能区
同一套卡号位置
同一套底边 Logo 位置
```

统一目的：

```text
玩家一眼看到横版深色金属封存框，就知道这是权限节点 / Access Node。
世界观内，这些牌都是同一种高科技二维封存体存储卡。
```

禁止：

```text
不要给 4 个系统域做 4 套完全不同的大外框。
不要让系统域差异破坏“权限节点是一类封存卡”的统一识别。
```

## 六、系统域识别层

系统域识别层属于世界观内正式信息。

它不是玩家提示 UI，而是该封存卡自带的系统域标签、域代号和节点位址。

系统域不靠大改外框区分，而靠以下正式识别层：

```text
1. 系统域短词 / 域标签
2. 系统域代号
3. 系统域颜色光条
4. 系统域小图标
5. 插画主题
6. 节点位址编号
```

### 系统域短词 / 域标签

中文版：

```text
门禁域
医疗域
无人机域
信用域
```

英文版：

```text
ACCESS DOMAIN
MEDICAL DOMAIN
DRONE DOMAIN
CREDIT DOMAIN
```

### 系统域代号

```text
AD = Access Domain / 门禁域
MD = Medical Domain / 医疗域
DN = Drone Domain / 无人机域
CD = Credit Domain / 信用域
```

## 七、权限节点卡号结构

权限节点卡号采用两层结构：

```text
全局存储序列 / 系统域节点位址
```

格式：

```text
N-001 / AD-A1
N-002 / AD-A2
N-003 / AD-A3
N-004 / MD-M1
N-005 / MD-M2
N-006 / MD-M3
N-007 / DN-D1
N-008 / DN-D2
N-009 / DN-D3
N-010 / CD-C1
N-011 / CD-C2
N-012 / CD-C3
```

说明：

```text
N = Node / 权限节点封存卡。
001-012 = 首版 12 张权限节点封存卡的全局存储序列。
AD / MD / DN / CD = 系统域代号。
A1 / M1 / D1 / C1 = 该系统域内的节点位址。
```

## 十、文件命名规范

文件名可以包含卡号信息，方便排序和查找，但文件名不是卡号。

推荐文件名：

```text
N-007_DN-D1_hummingbird-hive_cn-card-v1.png
N-007_DN-D1_hummingbird-hive_en-card-v1.png
N-007_DN-D1_hummingbird-hive_frame-v1.png
N-007_DN-D1_hummingbird-hive_illustration-01.png
```

说明：

```text
文件名中的 N-007_DN-D1 只是把卡号转成文件系统可用格式。
真实卡号仍然是卡面上的 N-007 / DN-D1。
不要把文件名误认为第二套卡号。
```

---

## 十一、卡面是否显示版本号

正式销售版卡面只显示唯一卡号：

```text
N-007 / DN-D1
```

测试样卡如果需要版本标记，可以放在极小测试标记位，但不能当作卡号的一部分：

```text
CN-v1.0
```

说明：

```text
版本标记是测试信息。
卡号是正式信息。
二者不能合并成第二套卡号。
```
