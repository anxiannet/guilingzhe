# 归零者｜编号与版本规范

> 文件定位：本文件记录 `归零者 / ZERO ACCESS` 的卡号、流水位、系统域识别与样卡版本记录规范。
>
> 本文件不替代正式卡表或正式规则。

---

## 一、当前结论

```text
12 个权限节点共用同一套权限节点外框。
区域区分依靠系统域识别层，而不是完全不同的外框。
卡号必须包含全局流水位，方便收藏、排版、查错和版本追踪。
版本号主要进入资产文件名和审核记录，不作为规则文本。
```

---

## 二、权限节点外框原则

12 张权限节点统一使用：

```text
88 × 63 mm 横版
同一套深色金属外框
同一套插画窗口
同一套标题区
同一套区域信息位置
同一套技能区
同一套编号区
同一套底边 Logo 位置
```

统一目的：

```text
玩家一眼看到横版深色金属封存框，就知道这是权限节点 / Access Node。
```

禁止：

```text
不要给 4 个系统域做 4 套完全不同的大外框。
不要让系统域差异破坏“权限节点是一类卡”的统一识别。
```

---

## 三、系统域识别层

系统域不靠大改外框区分，而靠以下识别层：

```text
1. 区域短词徽章
2. 系统域颜色光条
3. 系统域小图标
4. 插画主题
5. 编号前缀
```

### 区域短词

中文版：

```text
门禁域
医疗域
无人机域
信用域
```

英文版：

```text
ACCESS DOMAIN
MEDICAL DOMAIN
DRONE DOMAIN
CREDIT DOMAIN
```

### 系统域颜色方向

```text
门禁域：冷白 + 蓝
医疗域：白 + 青绿
无人机域：深蓝 + 青蓝
信用域：黑金 + 紫
```

颜色只做局部识别：

```text
标题栏细线
区域徽章边缘
左侧窄边
技能区上沿光线
底部编号小条
```

不要铺满整张卡。

### 系统域图标

```text
门禁域：门锁 / 闸机 / 门形芯片
医疗域：十字 / 生命曲线 / 医疗舱
无人机域：蜂鸟无人机 / 飞行器
信用域：芯片钱包 / 账本 / 黑金芯片
```

图标只用于识别系统域，不解释技能。

## 四、卡号结构

权限节点卡面编号采用两层结构：

```text
全局流水号 / 系统域编号
```

格式：

```text
N-001 / AD-A1
N-002 / AD-A2
N-003 / AD-A3
N-004 / MD-M1
N-005 / MD-M2
N-006 / MD-M3
N-007 / DN-D1
N-008 / DN-D2
N-009 / DN-D3
N-010 / CD-C1
N-011 / CD-C2
N-012 / CD-C3
```

说明：

```text
N = Node / 权限节点全局流水。
001-012 = 首版 12 张权限节点的全局流水位。
AD = Access Domain / 门禁域。
MD = Medical Domain / 医疗域。
DN = Drone Domain / 无人机域。
CD = Credit Domain / 信用域。
A1/M1/D1/C1 = 系统域内第 1 座节点。
```

## 六、蜂鸟巢当前编号

蜂鸟巢卡面编号改为：

```text
N-007 / DN-D1
```

含义：

```text
N-007 = 首版 12 张权限节点中的第 7 张。
DN-D1 = 无人机域第 1 座权限节点。
```

蜂鸟巢中文版样卡 v1 卡面信息：

```text
蜂鸟巢
无人机域

查看节点牌库顶1张，可置顶或置底。

N-007 / DN-D1
归零者 / ZERO ACCESS
```

## 七、版本记录规范

版本号用于记录资产和样卡迭代，不直接作为规则信息。

推荐格式：

```text
CN-v1.0
EN-v1.0
FRAME-v1.0
ILLUS-v1.0
```

用途：

```text
CN-v1.0 = 中文版合成样卡第 1 个正式测试稿。
EN-v1.0 = 英文版合成样卡第 1 个正式测试稿。
FRAME-v1.0 = 卡框版本。
ILLUS-v1.0 = 插图版本。
```

文件命名示例：

```text
hummingbird-hive-cn-card-v1.png
hummingbird-hive-en-card-v1.png
hummingbird-hive-frame-v1.png
hummingbird-hive-illustration-01.png
```

审核记录中必须记录：

```text
卡名
卡面编号
语言版本
样卡版本
使用插图版本
使用卡框版本
审核结论
是否进入下一版
```

---

## 八、卡面是否显示版本号

正式卡面主信息不显示版本号。

测试样卡可以在极小位置临时标注：

```text
CN-v1.0
```

但它属于样卡审核标记，不属于正式规则信息。

正式印刷导出时可选择删除版本标记，只保留：

```text
N-007 / DN-D1
归零者 / ZERO ACCESS
```
