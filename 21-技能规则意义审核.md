# 归零者｜技能规则意义审核

> 文件定位：本文件用于审核《归零者 / ZERO ACCESS》中技能是否具有明确规则意义，而不只是文字可理解或世界观合理。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、审核结论

```text
技能不仅要 AI 友好、算法友好、自动测试友好，还必须有明确规则意义。
```

规则意义指：

```text
1. 能影响接管节点、部署、抽牌、弃牌、伤害、保护、在线 / 离线、费用、电荷或胜利条件。
2. 能在常见回合中触发，而不是只等待极少数情况。
3. 使用后玩家能明显理解“这张牌为什么有用”。
4. 自动测试不仅能测试状态变化，还能测试该状态对核心流程的影响。
```

---

## 二、低规则意义技能的判断

以下技能虽然能被程序解析，但规则意义偏弱：

```text
只限制很少出现的动作。
只对极少数卡生效。
不影响接管节点、不影响胜利条件、不影响资源循环。
玩家很难判断什么时候该打。
自动测试只能测“状态存在”，不能测“状态有用”。
```

## 三、G-008 铁锚审核

旧技能：

```text
选择1个实体，本回合不能移动。
```

问题：

```text
当前规则中“移动实体”的触发点较少。
主要只有电梯栈等移动效果会被它影响。
如果本回合没有移动效果，铁锚没有实际收益。
该技能过于依赖对手是否使用少数移动技能。
```

规则意义评级：

```text
偏弱。
```

修改后技能：

```text
选择1个实体，本回合不能参与接管节点。
```

修改理由：

```text
接管节点是游戏核心动作。
限制实体参与接管，可以直接影响节点争夺和胜负节奏。
该技能在任何有接管压力的回合都可能有用。
```

世界观解释：

```text
铁锚是权限固定工具。
它不是单纯阻止物理移动，而是把目标实体锚定在当前节点权限层，阻止它进入本回合的接管流程。
```

自动测试意义：

```text
输入：target_entity_id。
效果：target_entity_id 获得“本回合不能参与接管节点”状态。
测试点：当玩家发起接管节点时，target_entity_id 不能被选为参与接管的实体。
持续时间：本回合。
```

## 四、技能规则意义审核表

| 技能类型 | 规则意义 | 审核结论 |
|---|---|---|
| 抽牌 | 影响手牌资源 | 有意义 |
| 弃牌 / 换牌 | 影响手牌质量 | 有意义 |
| 查看牌库顶 | 影响信息与决策 | 有意义 |
| 查看手牌 | 影响隐藏信息 | 有意义 |
| 造成伤害 | 影响实体存活 | 有意义 |
| 吹回手牌 | 影响节点部署与节奏 | 有意义 |
| 部署弃牌区游侠 | 影响场上实体数量 | 有意义 |
| 保护节点 | 直接影响节点争夺 | 有意义 |
| 使节点在线 / 离线 | 影响节点技能与胜利条件 | 有意义 |
| 获得电荷 | 影响资源与行动数量 | 有意义 |
| 费用减少 | 影响出牌节奏 | 有意义 |
| 不能参与接管节点 | 直接影响核心动作 | 有意义 |
| 不能移动 | 当前触发点较少 | 偏弱，需谨慎使用 |

## 五、后续审核原则

之后每个技能都要补充一条规则意义说明：

```text
这个技能影响了哪一个核心系统？
```

核心系统包括：

```text
接管节点。
节点在线 / 离线。
节点保护。
实体部署。
实体存活。
手牌资源。
牌库信息。
电荷费用。
胜利条件。
```

不建议继续保留只影响低频动作的技能，除非该低频动作已经成为明确核心机制。
