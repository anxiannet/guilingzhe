# 归零者｜技能规则意义审核

> 文件定位：本文件用于审核《归零者 / ZERO ACCESS》中技能是否具有明确规则意义，而不只是文字可理解或世界观合理。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`
>
> 说明：本文件只审核技能的规则意义；不重复维护完整游戏规则。
>
> 完整游戏规则以 `03-标准版规则.md` 为准；当前测试卡表数据以 `04-测试版卡表.md` 为准；标准版最终卡表以 `05-标准版卡表.md` 为准；标准版基础动作池以 `22-标准版基础动作池.md` 为准。

---

## 一、审核结论

```text
技能不仅要 AI 友好、算法友好、自动测试友好，还必须有明确规则意义。
标准版游侠与物品卡技能还必须能拆解为 22-标准版基础动作池.md 中的基础动作变体或组合。
标准版当前尚未最终定稿，动作对象组合仍可作为候选进入人工复核。
可部署实体的入场技能不得将己方场上实体返回手牌。
己方场上实体回手效果只允许出现在一次性物品上。
```

标准版游侠与物品卡当前只认可以下基础动作：

```text
抽牌。
查看牌库顶。
调整牌库顶。
返回牌库。
查看手牌。
返回手牌。
消灭。
```

备选动作：

```text
重洗牌库。
```

本文件不重复维护：

```text
牌库模式。
回合流程。
部署规则。
攻击节点流程。
占领节点结算。
节点控制权规则。
节点保护规则。
Root Access 胜利检查。
测试结算。
```

## 二、规则意义判断

规则意义指：

```text
1. 能影响手牌资源、牌库信息、牌库顺序、实体部署、实体留场、场面压力或后续出牌节奏。
2. 能在常见回合中触发，而不是只等待极少数情况。
3. 使用后玩家能明显理解“这张牌为什么有用”。
4. 自动测试不仅能测试状态变化，还能测试该状态对核心流程的影响。
```

低规则意义技能的判断：

```text
只限制很少出现的动作。
只对极少数卡生效。
不影响资源、场面、牌库信息、部署节奏或节点争夺。
玩家很难判断什么时候该打。
自动测试只能测“状态存在”，不能测“状态有用”。
```

## 三、标准版明确移除动作

以下动作不进入标准版游侠与物品卡技能池：

```text
弃牌。
公开手牌。
隔离。
禁用。
费用变化。
攻击变化。
生命变化。
伤害。
治疗。
获得电荷。
己方返回牌库。
己方消灭。
可部署实体回手己方场上实体。
```

移除理由：

| 动作 | 审核结论 | 理由 |
|---|---|---|
| 弃牌 | 移除 | 标准版牌量有限，不使用直接弃牌压制。 |
| 公开手牌 | 移除 | 对玩家收益不大，信息价值弱于查看手牌。 |
| 隔离 | 移除 | 需要额外隔离区和返回时机，不是基础动作。 |
| 禁用 | 移除 | 与返回手牌重叠，用返回手牌替代。 |
| 费用变化 | 移除 | 涉及卡牌三维之一：费用。 |
| 攻击变化 | 移除 | 涉及卡牌三维之一：攻击。 |
| 生命变化 | 移除 | 涉及卡牌三维之一：生命。 |
| 伤害 | 移除 | 本质影响生命维度。 |
| 治疗 | 移除 | 本质影响生命维度。 |
| 获得电荷 | 移除 | 等价于费用节奏变化。 |
| 己方返回牌库 | 移除 | 收益不直观，增加判断成本。 |
| 己方消灭 | 移除 | 收益不直观，容易变成连锁触发素材。 |
| 可部署实体回手己方场上实体 | 移除 | 会产生自己回自己、同名互回、人回收装备、装备回收人的循环风险。 |

说明：

```text
上述移除动作只限制标准版游侠与物品卡。
节点技能可以有临时攻击 / 生命强化等节点能力，但节点技能数据以当前测试卡表或标准版卡表为准，节点规则以 03-标准版规则.md 为准。
己方场上实体返回手牌不是完全移除；该效果只允许出现在一次性物品上。
```

## 四、技能规则意义审核表

| 基础动作 | 规则意义 | 审核结论 |
|---|---|---|
| 抽牌 | 增加手牌资源 | 标准版主池 |
| 查看牌库顶 | 获取牌库信息，辅助决策 | 标准版主池 |
| 调整牌库顶 | 改善后续抽牌质量或改变牌库节奏 | 标准版主池 |
| 返回牌库 | 延缓目标再次进入手牌或场上 | 标准版主池 |
| 查看手牌 | 获取隐藏手牌信息 | 标准版主池 |
| 返回手牌 | 移除当前场上压力，迫使目标重新部署 | 标准版主池，但己方目标只允许一次性物品使用 |
| 消灭 | 永久清除对手场上实体，送入弃牌区 | 标准版主池 |
| 重洗牌库 | 重置牌库顺序，但体感较弱 | 备选动作 |

## 五、按作用对象的规则意义

### 1. 手牌牌库

保留动作：

```text
抽牌。
查看牌库顶。
调整牌库顶。
返回牌库。
重洗牌库。
```

规则意义：

```text
影响手牌资源、牌库信息、牌库顺序和后续抽牌质量。
返回牌库可以用于把指定目标返回其控制者手牌牌库，具体目标对象需按卡表和测试结果进入人工复核。
```

### 2. 对手手牌

候选动作：

```text
查看手牌。
返回牌库。
```

规则意义：

```text
查看手牌影响隐藏信息。
将对手手牌中的牌返回牌库，可以延缓该牌被打出，但会提高操作复杂度和目标选择成本。
该对象组合尚未定稿，如进入标准版，需要单独做强度、体验和自动测试复核。
```

### 3. 对手场上游侠

保留动作：

```text
返回手牌。
返回牌库。
消灭。
```

规则意义：

```text
移除对手当前场上压力，打断部署、攻击节点与占领节点节奏。
返回手牌让目标可重新部署。
返回牌库比返回手牌更强，延缓目标再次进入场上。
消灭会将目标送入弃牌区。
```

### 4. 对手场上装备

保留动作：

```text
返回手牌。
返回牌库。
消灭。
```

规则意义：

```text
移除对手装备压力，打断装备布局。
将对手装备返回牌库可以作为候选效果保留，但需要单独审核其强度、目标数量和费用。
```

### 5. 己方场上游侠

限制保留动作：

```text
返回手牌。仅限一次性物品。
```

规则意义：

```text
撤回己方游侠，保留后续重新部署机会。
实体离开节点后，若其所在节点不再有控制者实体，该节点会按 03-标准版规则.md 变为无主。
该效果不得出现在游侠或装备的入场技能上。
```

### 6. 己方场上装备

限制保留动作：

```text
返回手牌。仅限一次性物品。
```

规则意义：

```text
回收己方装备，保留后续重新部署机会。
实体离开节点后，若其所在节点不再有控制者实体，该节点会按 03-标准版规则.md 变为无主。
该效果不得出现在游侠或装备的入场技能上。
```

## 六、循环风险审核

以下结构不得进入标准版游侠或装备入场技能：

```text
游侠入场后将己方游侠返回手牌。
游侠入场后将己方装备返回手牌。
装备入场后将己方游侠返回手牌。
装备入场后将己方装备返回手牌。
```

禁止原因：

```text
会产生自己回自己。
会产生同名互回。
会产生人回收装备。
会产生装备回收人。
会导致节点反复变为无主并回流，污染节点控制权测试。
会导致模拟器中单卡使用率异常膨胀。
```

允许结构：

```text
一次性物品将1个己方场上游侠返回你的手牌。
一次性物品将1个己方场上装备返回你的手牌。
```

## 七、后续审核原则

之后审核标准版游侠与物品卡技能时，必须先判断：

```text
1. 该技能是否由基础动作池中的动作形成？
2. 如果不是，是否应移除或改写？
3. 如果是组合技能，能否拆解为基础动作 + 限制条件？
4. 是否触碰费用、攻击、生命三维？
5. 是否引入弃牌、公开手牌、隔离、禁用等已移除动作？
6. 卡名是否能让玩家直觉联想到技能？
7. 是否与 04-测试版卡表.md 或 05-标准版卡表.md 中的当前卡牌数据一致？
8. 是否会与 03-标准版规则.md 中的部署、攻击节点、占领节点、节点控制权规则冲突？
9. 如果该技能属于候选对象组合，是否已经单独做强度、体验和自动测试复核？
10. 是否违反“可部署实体入场技能不得回手己方场上实体”的限制？
```

不符合以上标准的技能，不进入标准版游侠与物品卡候选。
