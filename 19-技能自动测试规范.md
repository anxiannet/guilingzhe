# 归零者｜技能自动测试规范

> 文件定位：本文件定义《归零者 / ZERO ACCESS》中技能文本、类人审核与平衡模拟的自动测试规则。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、当前结论

```text
所有技能必须可被规则引擎解析。
所有技能必须可自动测试。
所有技能不得依赖真人口头反应。
标准版游侠与物品卡技能必须能映射到 22-标准版基础动作池.md。
节点技能可以使用独立节点动作，但不得反向开放为手牌技能。
节点不具有在线或离线状态。
平衡模拟必须支持2、3、4人共用牌库测试。
```

当前自动测试脚本：

```text
scripts/human_ai_skill_audit.py
scripts/balance_simulator.py
```

运行方式：

```bash
python scripts/human_ai_skill_audit.py
python scripts/balance_simulator.py --players 2 --games 1000 --seed 7
python scripts/balance_simulator.py --players 3 --games 1000 --seed 7
python scripts/balance_simulator.py --players 4 --games 1000 --seed 7
```

## 二、标准版手牌技能测试

标准版游侠与物品卡技能必须映射到18种基本动作之一，或由多个基本动作组合形成。

```text
1. 获得手牌
2. 获得手牌牌库
3. 获得公共弃牌区
4. 获得游侠
5. 获得装备
6. 获得一次性物品
7. 调整手牌
8. 调整手牌牌库
9. 调整公共弃牌区
10. 调整游侠
11. 调整装备
12. 调整一次性物品
13. 消除手牌
14. 消除手牌牌库
15. 消除公共弃牌区
16. 消除游侠
17. 消除装备
18. 消除一次性物品
```

查看是合法底层动作，但不计入标准版18种有效手牌基本动作。

## 三、节点技能测试

节点技能属于城市基础设施能力，当前允许测试：

```text
查看节点信息。
保护节点。
移动游侠。
移动装备。
临时调整游侠ATK或HP。
临时调整装备ATK或HP。
从公共弃牌区回收游侠或装备。
系统级实体调度与控制权例外。
```

明确禁止继续生成或测试：

```text
使节点在线。
使节点离线。
以在线或离线作为节点技能发动条件。
以在线节点数量作为Root Access胜利条件。
```

中继站与屏蔽塔的旧在线／离线技能已经废止，重新设计前不得进入模拟器有效节点技能池。

## 四、推荐技能字段

```text
目标：哪张牌、哪个实体或哪座节点。
数量：1张、1个、至多2个、全部。
区域：手牌、手牌牌库、公共弃牌区、节点、场上。
动作：获得、调整、消除，或节点专属动作。
结果：进入手牌、返回牌库、移动、消灭、销毁等。
归属：己方、其他玩家、控制者。
时机：打出时、被消灭时、回合内。
费用：卡牌费用或节点技能费用。
ATK / HP：游侠与装备的三维字段。
```

节点状态字段中不得包含 `online`、`offline` 或等价布尔状态。

---

## 五、禁止技能结构

```text
真人语音触发。
玩家主观判断触发。
可部署实体入场后回手己方实体形成循环。
未登记的平行动作。
以已废止的节点在线／离线状态作为目标、条件或结果。
```

## 六、平衡模拟要求

模拟器必须对齐 `03-标准版规则.md`：

```text
2至4名玩家共用标准手牌牌库。
所有玩家共用公共弃牌区。
节点牌库由标准节点构成。
中央节点区保持3座无主节点。
节点只有无主或由某名玩家控制两种归属状态。
节点不记录在线或离线。
控制同一系统域的3座节点即获得Root Access。
每座节点最多容纳3个部署实体。
每名玩家每回合最多使用1次节点技能。
```

模拟输出至少包括：

```text
各座位胜率。
平局率。
平均与中位结束回合。
结束原因。
卡牌使用率。
节点技能使用率。
攻击节点次数。
节点控制权变化次数。
```

## 七、复查要求

每次规则或卡表修改后，必须搜索并拒绝以下残留：

```text
节点在线
节点离线
使节点在线
使节点离线
offline_node
online_node
disable_offline_node
```

“离线地堡”等专有卡名不自动视为节点状态机制，但需要单独审核名称是否仍符合当前世界观。
