# 归零者｜技能自动测试规范

> 文件定位：本文件定义《归零者 / ZERO ACCESS》中技能文本、类人审核与平衡模拟的自动测试规则。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、当前结论

```text
所有技能必须可被规则引擎解析。
所有技能必须可自动测试。
所有技能不得依赖真人口头反应。
标准版游侠与物品卡技能必须能拆解为基础动作池中的动作变体或组合。
卡名、技能和载体必须能通过类人 AI 直觉审核。
整套卡牌平衡性需要通过模拟器进行压力测试。
平衡模拟器必须支持 2、3、4 人测试。
平衡模拟器必须对齐 03-规则变体.md 的当前测试规则。
当前打牌规则为“不限制张数，只限制电荷资源”。
```

当前自动测试脚本：

```text
scripts/human_ai_skill_audit.py
scripts/balance_simulator.py
```

运行方式：

```bash
python scripts/human_ai_skill_audit.py
python scripts/balance_simulator.py --players 2 --games 1000 --seed 7
python scripts/balance_simulator.py --players 3 --games 1000 --seed 7
python scripts/balance_simulator.py --players 4 --games 1000 --seed 7
python scripts/balance_simulator.py --players 4 --deck-mode shared --games 1000 --seed 7
python scripts/balance_simulator.py --players 4 --deck-mode separate --games 1000 --seed 7
```

## 二、基础动作测试

标准版游侠与物品卡技能只能拆解为：

```text
抽牌。
查看牌库顶。
调整牌库顶。
返回牌库。
查看手牌。
返回手牌。
消灭。
```

备选动作：

```text
重洗牌库。
```

脚本会检查以下已移除动作是否残留：

```text
弃牌。
公开手牌。
隔离。
禁用。
费用变化。
攻击变化。
生命变化。
伤害。
治疗。
获得电荷。
己方返回牌库。
己方消灭。
```

## 四、推荐技能字段

技能文本应能拆成以下字段：

```text
目标：选择谁 / 哪张牌 / 哪座节点 / 哪个牌库。
数量：1张 / 1个 / 1座 / 至多2个。
区域：手牌、手牌牌库、弃牌区、节点、场上。
动作：抽、查看、排序、返回牌库、返回手牌、消灭、移动、保护、上线、离线。
持续时间：本回合、回合结束、保护者下回合开始。
归属：控制者、来源、目标玩家。
```

## 五、类人 AI 直觉审核

脚本会模拟玩家直觉检查：

```text
卡名是否能暗示技能。
卡名是否符合载体。
技能是否像该载体能做的事。
技能是否使用基础动作池。
费用是否大致符合技能强度。
装备是否有 ATK / HP。
一次性物品是否没有 ATK / HP。
节点技能费用是否符合当前费用分层。
```

## 六、整套卡牌平衡模拟

平衡模拟脚本：

```text
scripts/balance_simulator.py
```

当前模拟器输出：

```text
每个座位的胜率。
平局率。
平均结束回合。
中位结束回合。
结束原因。
手牌卡使用率 Top / Bottom。
节点技能使用率。
疑似强卡 / 弱卡。
座位胜率是否明显偏离理论均值。
```

当前模拟假设：

```text
支持 2、3、4 人。
2人默认 separate：双方各自使用一副62张临时测试牌库。
3人默认 shared：所有玩家共用一副62张临时测试手牌牌库。
4人默认 shared：所有玩家共用一副62张临时测试手牌牌库。
每回合最多使用1次节点技能。
每回合不限制打出手牌张数，只限制电荷资源。
打出可部署实体时，先部署到节点，再结算入场技能。
一次性物品不部署，直接结算技能。
G-005 强制回滚扇区只影响1座节点中的所有对手游侠。
保护持续到保护者下个回合开始。
牌库耗尽后不自动重洗弃牌区。
节点控制采用各玩家 ATK 总和比较，最高者控制；并列时控制权不变。
每次实体进入 / 离开节点、实体 ATK 改变、节点在线离线变化后，立即检查接管与 Root Access。
控制同一系统域 3 座在线节点立即获胜。
AI 是启发式，不代表真人最优打法。
```

接管算法审核结论：

```text
当前自动接管算法继续保留。
它适合自动测试，因为简单、稳定、可复现。
不再恢复“每回合最多1张手牌”的限制。
如果连续打牌导致胜利过快，优先调整费用曲线、复制数和起手牌，而不是限制打牌张数。
```

## 七、当前脚本检查范围

类人审核脚本检查：

```text
R-001 至 R-023 中正式保留的游侠。
G-001 至 G-012 物品卡。
N-001 至 N-012 节点技能费用。
```

平衡模拟脚本检查：

```text
2、3、4 人对战流程。
shared / separate 两种牌库模式。
当前游侠、物品卡、节点技能费用。
Root Access 胜利速度。
卡牌使用率。
节点技能使用率。
座位胜率偏差。
资源限制下的连续打牌节奏。
```

当前脚本不检查：

```text
图片资产。
最终卡面排版。
真人最优打法。
完整复制数定稿。
完整战斗伤害系统。
```

## 八、通过标准

```text
FAIL 项必须清零。
WARN 项进入人工复核清单。
PASS 项可进入下一步制卡或数值测试。
平衡模拟中使用率过高、过低或胜利速度异常的卡牌进入数值复核清单。
```
