# 归零者｜技能自动测试规范

> 文件定位：本文件定义《归零者 / ZERO ACCESS》中所有技能文本的 AI 友好、算法友好与自动测试规则。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、当前结论

```text
所有技能必须可被规则引擎解析。
所有技能必须可自动测试。
所有技能不得依赖真人口头反应。
```

---

## 二、禁止技能结构

禁止使用以下结构：

```text
喊一名玩家。
喊一名玩家名字。
对方答应后。
如果对方听见。
如果对方回应。
口头确认。
现实语音触发。
玩家主观判断触发。
```

原因：

```text
这类技能依赖真人语音、反应速度、听觉环境和主观争议，无法稳定写入规则引擎，也不利于自动测试。
```

---

## 三、推荐技能结构

技能文本应尽量拆成以下字段：

```text
1. 目标：选择谁 / 哪张牌 / 哪座节点 / 哪个牌库。
2. 数量：1张 / 1个 / 1座 / 至多2个。
3. 区域：手牌、手牌牌库、手牌弃牌区、节点、隔离区。
4. 动作：抽、弃、查看、部署、移动、造成伤害、保护、上线、离线、隔离。
5. 持续时间：本回合、回合结束、保护者下回合开始。
6. 归属：控制者、来源、目标玩家。
```

---

## 四、标准写法

推荐写法：

```text
选择1名玩家。
选择1名敌方游侠。
选择1座节点。
选择1个手牌牌库。
查看手牌牌库顶1张。
抽1张牌。
弃1张牌。
将其部署到1座节点。
将其隔离，回合结束返回原节点。
```

不推荐写法：

```text
喊一名玩家。
对方答应后。
你叫到谁谁抽牌。
被喊的人如果回应。
现场确认后生效。
```

## 五、目标选择规则

所有需要目标的技能，必须能转成明确目标选择：

```text
target_player_id
source_card_id
target_card_id
target_node_id
target_deck_id
zone_from
zone_to
```

示例：

```text
选择1名玩家。你与其各抽1张牌。
```

可转为：

```text
actor_player_id
selected_player_id
draw_count_actor = 1
draw_count_selected = 1
```

## 六、随机与隐藏信息

随机效果可以存在，但必须可被规则引擎处理。

允许：

```text
从目标玩家手牌中随机选择1张。
洗混手牌牌库。
```

要求：

```text
自动测试时必须能固定随机种子。
测试记录必须能还原随机结果。
```

优先建议：

```text
能用“选择”解决的技能，优先使用“选择”。
只有玩法确实需要不确定性时，才使用“随机”。
```

```text
喊一名玩家名字；对方答应后，选择其1名游侠，将其隔离，回合结束返回原节点。

选择1名敌方游侠。将其隔离，回合结束返回原节点。
```

## 八、技能审核标准

每个技能审核时必须回答：

```text
1. 是否有明确目标？
2. 是否有明确数量？
3. 是否有明确区域？
4. 是否有明确动作？
5. 是否有明确持续时间？
6. 是否避免真人口头反应？
7. 是否能写成自动测试用例？
```

通过标准：

```text
上述7项均可明确回答，才算通过技能审核。
```
