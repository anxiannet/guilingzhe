# 归零者｜技能自动测试规范

> 文件定位：本文件定义《归零者 / ZERO ACCESS》中所有技能文本的 AI 友好、算法友好与自动测试规则。
>
> 当前唯一主仓库：`anxiannet/guilingzhe`

---

## 一、当前结论

```text
所有技能必须可被规则引擎解析。
所有技能必须可自动测试。
所有技能不得依赖真人口头反应。
标准版游侠与物品卡技能必须能拆解为基础动作池中的动作变体或组合。
卡名、技能和载体必须能通过类人 AI 直觉审核。
整套卡牌平衡性需要通过模拟器进行压力测试。
平衡模拟器必须支持 2、3、4 人测试。
```

当前自动测试脚本：

```text
scripts/human_ai_skill_audit.py
scripts/balance_simulator.py
```

运行方式：

```bash
python scripts/human_ai_skill_audit.py
python scripts/balance_simulator.py --players 2 --games 1000 --seed 7
python scripts/balance_simulator.py --players 3 --games 1000 --seed 7
python scripts/balance_simulator.py --players 4 --games 1000 --seed 7
```

脚本输出：

```text
human_ai_skill_audit.py：输出 PASS / WARN / FAIL。
balance_simulator.py：输出各座位胜率、平均结束回合、卡牌使用率、节点技能使用率和疑似平衡问题。
```

## 二、禁止技能结构

禁止使用以下结构：

```text
喊一名玩家。
喊一名玩家名字。
对方答应后。
如果对方听见。
如果对方回应。
口头确认。
现实语音触发。
玩家主观判断触发。
```

原因：

```text
这类技能依赖真人语音、反应速度、听觉环境和主观争议，无法稳定写入规则引擎，也不利于自动测试。
```

## 三、标准版基础动作测试

标准版游侠与物品卡技能只能拆解为：

```text
抽牌。
查看牌库顶。
调整牌库顶。
返回牌库。
查看手牌。
返回手牌。
消灭。
```

备选动作：

```text
重洗牌库。
```

脚本会检查以下已移除动作是否残留：

```text
弃牌。
公开手牌。
隔离。
禁用。
费用变化。
攻击变化。
生命变化。
伤害。
治疗。
获得电荷。
己方返回牌库。
己方消灭。
```

## 四、推荐技能结构

技能文本应尽量拆成以下字段：

```text
1. 目标：选择谁 / 哪张牌 / 哪座节点 / 哪个牌库。
2. 数量：1张 / 1个 / 1座 / 至多2个。
3. 区域：手牌、手牌牌库、弃牌区、节点、场上。
4. 动作：抽、查看、排序、返回牌库、返回手牌、消灭、移动、保护、上线、离线。
5. 持续时间：本回合、回合结束、保护者下回合开始。
6. 归属：控制者、来源、目标玩家。
```

## 五、标准写法

推荐写法：

```text
选择1名玩家。
选择1个对手场上游侠。
选择1座节点。
选择1个手牌牌库。
查看手牌牌库顶1张。
抽1张牌。
将1个对手场上游侠返回其控制者手牌。
将1个对手场上游侠返回其控制者手牌牌库底。
消灭1个对手场上游侠。
```

不推荐写法：

```text
喊一名玩家。
对方答应后。
你叫到谁谁抽牌。
被喊的人如果回应。
现场确认后生效。
吹回1个游侠。
处理1个游侠。
权限驱逐1个游侠。
```

## 六、目标选择规则

所有需要目标的技能，必须能转成明确目标选择：

```text
target_player_id
source_card_id
target_card_id
target_node_id
 target_deck_id
zone_from
zone_to
```

示例：

```text
选择1名玩家，其抽1张牌。
```

可转为：

```text
actor_player_id
selected_player_id
draw_count_selected = 1
```

## 七、随机与隐藏信息

随机效果可以存在，但必须可被规则引擎处理。

允许：

```text
从目标玩家手牌中随机选择1张。
洗混手牌牌库。
```

要求：

```text
自动测试时必须能固定随机种子。
测试记录必须能还原随机结果。
```

优先建议：

```text
能用“选择”解决的技能，优先使用“选择”。
只有玩法确实需要不确定性时，才使用“随机”。
```

## 八、类人 AI 直觉审核

除规则字段外，脚本还会模拟玩家直觉检查：

```text
1. 卡名是否能暗示技能。
2. 卡名是否符合载体。
3. 技能是否像该载体能做的事。
4. 技能是否使用基础动作池。
5. 费用是否大致符合技能强度。
6. 装备是否有 ATK / HP。
7. 一次性物品是否没有 ATK / HP。
8. 节点技能费用是否符合当前费用分层。
```

示例规则：

```text
侦察 / 探针 / 无人机：应对应查看类技能。
信使 / 抽取 / 广播：应对应抽牌。
筛选 / 预读 / 算盘：应对应牌库调整。
弹弓 / 甩棍 / 门神 / 推土机：应对应返回手牌。
清除者 / 爆破兵 / 清道夫：应对应消灭。
回滚 / 回收 / 沙盒：应对应返回牌库或返回手牌。
信标 / 撤离 / 烟幕：应对应返回手牌。
```

## 九、整套卡牌平衡模拟

平衡模拟脚本：

```text
scripts/balance_simulator.py
```

运行方式：

```bash
python scripts/balance_simulator.py --players 2 --games 1000 --seed 7
python scripts/balance_simulator.py --players 3 --games 1000 --seed 7
python scripts/balance_simulator.py --players 4 --games 1000 --seed 7
```

当前模拟器输出：

```text
每个座位的胜率。
平局率。
平均结束回合。
中位结束回合。
结束原因。
手牌卡使用率 Top / Bottom。
节点技能使用率。
疑似强卡 / 弱卡。
座位胜率是否明显偏离理论均值。
```

当前模拟假设：

```text
支持 2、3、4 人。
每名玩家使用相同 62 张临时复制数牌库。
30 张不同手牌卡默认各 2 张，侦察员和数据抽取器额外各 1 张。
节点全部开局在线且无主。
节点控制采用各玩家 ATK 总和比较，最高者控制；并列时控制权不变。
控制同一系统域 3 座在线节点立即获胜。
不模拟完整战斗伤害。
AI 是启发式，不代表真人最优打法。
```

注意：

```text
该脚本用于压力测试和平衡预警，不是最终规则引擎。
模拟结果只能提示问题，不能单独作为定稿依据。
```

## 十一、技能审核标准

每个技能审核时必须回答：

```text
1. 是否有明确目标？
2. 是否有明确数量？
3. 是否有明确区域？
4. 是否有明确动作？
5. 是否避免真人口头反应？
6. 是否能写成自动测试用例？
7. 是否由基础动作池变体或组合形成？
8. 卡名是否能直觉联想到技能？
9. 载体是否符合技能来源？
```

通过标准：

```text
FAIL 项必须清零。
WARN 项进入人工复核清单。
PASS 项可进入下一步制卡或数值测试。
平衡模拟中使用率过高、过低或胜利速度异常的卡牌进入数值复核清单。
```
